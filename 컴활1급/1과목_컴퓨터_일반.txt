001 한글 Windows 10의 특징
- 선점형 멀티태스킹(Preemptive Multi-tasking)
    - 운영체제가 각 작업의 CPU 이용 시간을 제어
    - 앱 실행중 문제가 발생하면 해당 앱을 강제 종료, 모든 시스템 자원을 반환하는 멀티태스킹 운영 방식
- 플러그 앤 플레이(PnP, Plug & Play)
    - 컴퓨터 시스템에 하드웨어 설치 시
    - 해당 하드웨어 사용에 필요한 시스템 환경을 운영체제가 자동으로 구성해 주는 것
- OLE(Object Linking and Embedding)
    - 다른 여러 앱에서 작성된 문자나 그림 등의 개체(Object)를 현재 작성중인 문서에 자유롭게 연결(Linking)하거나 삽입(Embedding)해 편집할 수 있게 하는 기능
    - OLE로 연결된 이미지를 원본 앱에서 수정하거나 편집하면 그 내용이 그대로 해당 문서에 반영됨
- 64비트 데이터 처리
    - 완전한 64비트로 데이터 처리
    - 더 많은 양의 데이터를 빠르게 처리할 수 있음
    - 사용자가 좀 더 빠르고 효율적인 시스템을 구축할 수 있게 함
    - 64비트 버전으로 제작된 윈도우10용 앱은 32비트 버전의 윈도우10에서는 작동되지 않음

002 파일 시스템 - NTFS
- 성능, 보안, 디스크 할당, 안정성, 속도 면에서 FAT 파일 시스템에 비해 뛰어난 고급 기능을 제공함
- 시스템 리소스 사용을 최소화함
- 파일 및 폴더에 대한 액세스 제어를 유지하고 제한된 계정을 지원함
- 최대 볼륨 크기는 256TB, 파일 크기는 볼륨 크기에 의해서만 제한됨
- 비교적 큰 오버헤드가 있기 때문에 40MB 이상의 볼륨에서 사용하면 효과적임

003 바로 가기 키(단축키)
- Alt + Esc : 현재 실행중인 앱들을 순서대로 전환함
- Alt + Tab : 현재 실행중인 앱들의 목록을 화면 중앙에 나타냄
- Alt + Enter : 선택된 항목의 속성 대화상자를 실행함
- Alt + Spacebar : 활성창의 바로 가기 메뉴를 표시함
- Alt + F4 : 실행중인 창이나 앱을 종료함, 실행중인 앱이 없으면 Windows 종료 창을 나타냄
- Alt + PrintScrean : 현재 작업중인 활성 창을 클립보드로 복사함
- Ctrl + A : 폴더 및 파일을 모두 선택함
- Ctrl + Esc : 시작 메뉴를 표시함
- Ctrl + Shift + Esx : 작업관리자 대화상자를 실행해 문제가 있는 앱을 강제로 종료함
- Shift + Delete : 폴더나 파일을 휴지통을 거치지 않고 바로 삭제함
- Shift + F10 : 바로 가기 메뉴를 표시함
- Windows키 + E : 파일 탐색기 실행
- Windows키 + F : 피드백 허브 앱 실행 
- Windows키 + L : 컴퓨터를 잠그거나 사용자를 전환함
- Windows키 + M, Windows키 + Shift + M : 열려 있는 모든 창을 최소화/이전 크기로 나타냄
- Windows키 + R : 실행 창 나타냄
- Windows키 + Pause/Break : 설정 -> 시스템 -> 정보 창 나타냄

004 바로 가기 아이콘(단축 아이콘)
- 자주 사용하는 문서나 앱을 빠르게 실행시키기 위한 아이콘
- 원본 파일의 위치 정보를 가지고 있음
- 폴더, 파일, 디스크 드라이브, 다른 컴퓨터, 프린터 등 모든 개체에 대해 작성할 수 있음
- 왼쪽 하단에 화살표 표시가 있어 일반 아이콘과 구별됨
- 원본 파일의 위치와 관계없이 만들 수 있음
- 확장자는 LNK
- 컴퓨터에 여러 개 존재할 수 있음
- 속성 대화상자 : 바로 가기 아이콘의 파일 형식, 설명, 위치, 크기, 만든 날짜, 수정한 날짜, 액세스한 날짜, 연결된 항목의 정보(대상 파일, 형식, 위치 등) 등을 확인할 수 있음

005 작업 표시줄
- 현재 실행되고 있는 앱 단추와 앱을 빠르게 실행하기 위해 등록한 고정 앱 단추 등이 표시되는 곳
- 기본적으로 바탕 화면의 맨 아래쪽에 있음
- 구성 : 시작 단추, 검색 상자, 작업 보기, 앱 단추가 표시되는 부분, 알림 영역(표시기), '바탕 화면 보기' 단추
- 위치를 변경하거나 크기를 조절할 수 있음(화면의 1/2까지만)
- 오른쪽의 알림 영역에 표시할 앱 아이콘과 시스템 아이콘을 설정할 수 있음
- 작업 표시줄 잠금 : 작업 표시줄을 포함해 작업 표시줄에 있는 도구 모음의 크기나 위치를 변경하지 못하도록 함
- 테스크톱/태블릿 모드에서 작업 표시줄 자동 숨기기 : 작업 표시줄이 있는 위치에 마우스를 대면 나타나고 다른 곳으로 이동하면 사라짐
- 작은 작업 표사줄 단추 사용 : 작업 표시줄의 앱 단추들이 작은 아이콘으로 표시됨
- 작업 표시줄 오른쪽 끝 바탕 화면 보기 단추 위에 마우스 포인터를 놓으면 바탕 화면이 일시적으로 표시됨
- 화면에서의 작업 표시줄 위치 : 상하좌우 중 선택

006 가상 데스크톱
- 바탕 화면을 여러 개 만들어 바탕 화면별로 필요한 앱을 실행해 놓고 바탕 화면을 전환하면서 작업할 수 있음
- 가상 데스크톱 생성 시 작업 보기 화면 위쪽에 데스크톱 아이콘이 표시됨
- 데스크톱 아이콘에 마우스를 놓으면 해당 데스크톱에서 현재 작업 중인 앱이 표시됨
- 작업 보기 화면에서 원하는 데스크톱을 선택해 이동할 수 있음
- 작업 보기 화면에서 현재 작업 중인 앱을 드래그해 다른 데스크톱으로 이동할 수 있음
- 제거된 가상 데스크톱에서 작업 중이던 앱은 이전 가상 데스크톱으로 이동됨
- 시스템을 재시작하더라도 가상 데스크톱은 제거되지 않고 남아 있음
- 생성 : Ctrl + Windows키 + D
- 제거 : Ctrl + Windows키 + F4

007 파일 탐색기의 구성 요소
- 즐겨찾기
    - 자주 사용하는 개체를 등록해 해당 개체로 빠르게 이동하기 위해 사용하는 기능
    - 자주 사용하는 폴더나 최근에 사용한 파일이 자동으로 등록됨
- OneDrive : 마이크로소프트 사에서 제공하는 OneDrive와 내 PC의 OneDrive 폴더를 동기화해 파일을 백업하고 보호하는 기능
- 라이브러리
    - 컴퓨터에 흩어져 있는 자료들을 한 곳에서 관리할 수 있게 하는 가상의 폴더
    - 물리적인 파일을 실제 저장하고 있는 것이 아니고 파일이 저장된 폴더를 연결해 보여주는 기능
    - 하나의 라이브러리에서 최대 50개의 폴더를 관리할 수 있음
- 내 PC : 컴퓨터에 설치된 모든 구성 요소를 표시하며, 각 구성 요소를 관리할 수 있는 여러 가지 기능을 제공함
- 네트워크 : 네트워크에 연결된 자원을 확인하거나 공유할 수 있는 기능을 제공함

008 폴더 옵션
- 파일이나 폴더의 보기 형식, 검색 방법 등에 대한 설정 변경
- 대화상자의 탭별 기능
    - 일반
        - 파일 탐색기가 열렸을 때의 기본 위치를 '즐겨찾기'나 '내 PC' 중에서 선택
        - 새로 여는 폴더의 내용을 같은 창에서 열리거나 다른 창에 열리도록 지정
        - 웹을 사용하는 것처럼 바탕 화면이나 파일 탐색기에서도 파일을 한 번 클릭하면 실행되도록 설정
        - 즐겨찾기에 최근에 사용된 파일이나 폴더의 표시 여부 지정
        - 파일 탐색기의 즐겨찾기에 표시된 최근에 사용한 파일 목록을 지울 수 있음
    - 보기
        - 탐색 창에 라이브러리의 표시 여부 지정
        - 메뉴 모음의 항상 표시 여부 지정
        - 숨김 파일이나 폴더의 표시 여부 지정
        - 알려진 파일 형식의 파일 확장명 표시 여부 지정
        - 보호된 운영체제 파일의 숨김 여부 지정
        - 폴더 팁에 파일 크기 정보의 표시 여부 지정
    - 검색
        - 폴더에서 시스템 파일을 검색할 때 색인 사용 여부 지정
        - 색인되지 않은 위치 검색 시 포함할 대상 지정

009 파일과 폴더
- 파일/폴더 선택
    - 연속적인 항목 선택 : 첫 번째 항목 클릭 후 Shift 누른 상태에서 마지막 항목 클릭
    - 비연속적인 항목 선택 : Ctrl을 누른 상태에서 선택할 항목 차례로 클릭
    - 전체 항목 선택 : Ctrl + A
- 파일/폴더 복사 및 이동
    - 같은 드라이브
        - 복사 : Ctrl 누른 상태에서 마우스로 드래그 앤 드롭
        - 이동 : 마우스로 드래그 앤 드롭
    - 다른 드라이브
        - 복사 : 마우스로 드래그 앤 드롭
        - 이동 : Shift 누른 상태에서 마우스로 드래그 앤 드롭

010 파일 탐색기의 검색 상자
- 파일 탐색기에서 찾으려는 내용을 검색 상자에 입력 후 Enter를 누르면 리본 메뉴에 검색 필터를 설정할 수 있는 '검색' 탭이 생성되고 검색이 수행됨
- 파일 탐색기에서 F3나 Ctrl + F를 누르면 검색 상자로 포커스가 옮겨짐
- 기본저긍로 검색 상자에 입력한 내용이 포함된 파일이나 폴더 등에 검색됨
- 내용 앞에 '-'을 붙이면 해당 내용이 포함되지 않은 파일이나 폴더가 검색됨
- 데이터를 검색한 후 검색 기준을 저장할 수 있으며, 저장된 검색 기준을 열면 해당 기준으로 데이터를 검색해 표시함
- 색인 위치를 지정해 더 빠른 속도로 검색할 수 있음
- 수정한 날짜, 크기 등과 같은 속성을 이용해 파일을 검색할 수 있음

011 휴지통
- 삭제된 파일이나 폴더가 임시 보관되는 장소
- 기본적으로 드라이브 용량의 5~10% 범위 내에서 시스템이 자동으로 설정함
- 사용자가 원하는 크기를 MB 단위로 지정할 수 있음
- 아이콘을 이용해 비워짐 유무 구분 가능
- 하드디스크 드라이브마다 한 개씩 만들 수 있음
- 용량을 초과하면 가장 오래 전에 삭제된 파일부터 자동으로 지워짐
- 보관된 파일이나 폴더는 복원할 수 있지만 복원하기 전에는 사용하거나 이름을 변경할 수 없음
- 휴지통에 보관되지 않는 경우
    - DOS 모드(명령 프롬프트), 네트워크 드라이브, USB 메모리에서 삭제된 항목
    - Shift를 누른 채 삭제 명령을 실행한 경우
    - 휴지통 속성에서 '파일을 휴지통에 버리지 않고 삭제할 때 바로 제거'를 선택한 경우
    - 휴지통 속성에서 최대 크기를 0MB로 지정한 경우
    - 같은 이름의 항목을 복사/이동 작업으로 덮어쓴 경우

012 휴지통 속성
- 휴지통의 크기를 드라이브마다 다르게 설정할 수 있고, 모두 같은 크기로 설정할 수도 있음
- 파일이나 폴더를 삭제할 때 휴지통을 거치지 않고 바로 삭제하도록 설정
- 파일이나 폴더가 삭제될 때마다 확인 대화상자가 표시되도록 설정

013 휴지통 복원

014 Windows 보조프로그램
- 메모장
    - 특별한 서식이 필요 없는 간단한 텍스트(ASCII 형식) 파일을 작성할 수 있는 문서 작성 앱
    - 텍스트(.TXT) 형식의 문서만 열거나 저장할 수 있음
    - 문서 전체에 대해서만 글꼴의 종류, 속성, 크기를 변경할 수 있음
    - 그림, 차트 등의 OLE 개체를 삽입할 수 없음
    - ANSI, 유니코드, UTF-8 등의 인코딩 형식으로 저장할 수 있음
    - 문서의 첫 행 맨 왼쪽에 대문자로 .LOG라고 입력하면 메모장을 열 때마다 현재의 시간과 날짜를 문서의 끝에 표시함
    - 주요 메뉴
        - 이동 : 줄(행)을 기준으로 커서를 이동할 수 있지만 '자동 줄 바꿈'이 해제된 상태에서만 사용 가능
        - 시간/날짜 : 커서가 있는 위치에 현재 시간과 날짜 입력
        - 자동 줄 바꿈 : 창 가로 크기에 맞게 텍스트를 표시하고 다음 줄로 넘김
- 그림판
    - 간단한 그림을 작성하거나 수정하기 위한 보조 앱
    - 기본 저장 형식은 PNG이며, EMP, GIF, TIF, JPG, HEIC 형식으로도 저장할 수 있음
    - 그림판에서 편집한 그림을 다른 문서에 붙여넣거나 Windows 바탕 화면의 배경으로 사용할 수 있음
    - Shift를 누른 상태에서는 수평선, 수직선, 45도의 대각선, 정사각형, 정원을 그릴 수 있음
    - 그림의 크기 조정 및 회전, 대칭 이동 등의 편집이 가능함

015 유니버셜 앱
- 계산기
    - 간단한 사칙연산부터, 삼각법, 진법 변환, 날짜 꼐산, 통화 환율 등을 계산할 때 사용하는 앱
    - 종류 : 표준, 공학용, 프로그래머, 그래프, 날짜 계산
    - 변환기 : 통화 환율, 부피, 길이, 무게 및 질량, 온도, 에너지, 면적, 속도, 시간, 일률, 데이터, 입력, 각도 등을 계산
- 빠른 지원
    - 다른 사용자의 컴퓨터에 접속해 원격 지원을 하거나, 내 컴퓨터에 접속한 다른 사용자로부터 원격 지원을 받을 수 있도록 하는 앱
    - 내 컴퓨터의 마우스와 키보드로 다른 사용자 컴퓨터를 제어하는 동안 다른 사용자도 화면을 보면서 마우스와 키보드를 조작할 수 있음
    - 원격 지원을 하는 자는 마이크로소프트 계정으로 로그인 해야하고, 원격 지원을 받는 자는 로그인 하지 않아도 됨
    - '공유 옵션'에는 '모든 권한 가지기'와 '화면 보기'가 있음

016 설정 -> 시스템
- 디스플레이
    - 화면에 표시되는 텍스트나 앱, 아이콘 등의 크기 변경
    - 디스플레이의 해상도나 방향 변경
- 정보
    - 시스템에 연결된 하드웨어 및 Windows 사양 등을 확인하거나 컴퓨터 이름 변경
    - 장치 사양 : 장치 이름, 프로세서(CPU) 종류, 메모리(RAM) 크기, 장치 ID, 제품 ID, 시스템 종류, 펜 및 터치 등
    - Windows 사양 : 에디션, 버전, 설치 날짜, OS 빌드, 경험 등

017 설정 -> 개인 설정
- 배경
    - 바탕 화면의 배경이 표시되는 방식 지정
    - Windows에서 제공하는 이미지나 GIF, BMP, JPEG, PNG 등의 사용자 이미지 중에서 원하는 그림 파일을 선택해 지정
    - 바탕 화면에 놓일 배경 그림의 맞춤 방식 지정
- 잠금 화면
    - 잠금 화면에 표시할 앱이나 배경 지정
    - 잠금 화면에 알림을 표시할 앱 선택
    - 화면 시간 제한 설정 : 정해진 시간 동안 컴퓨터를 사용하지 않으면 화면을 끄거나 절전 모드로 변경되게 설정하는 창으로 이동
    - 화면 보호기 지정
- 테마
    - 컴퓨터의 배경 그림, 색, 소리, 마우스 커서 등 Windows를 구성하는 여러 요소를 하나의 그룹으로 묶어 놓은 것
    - 다른 테마로 변경할 수 있음
- 글꼴
    - 글꼴 폴더에는 OTF나 TTC, TTF, FON 등의 확장자를 갖는 글골 파일이 설치되어 있음
    - 글꼴이 설치되어 있는 폴더의 위치 : C\Windows\Fonts
    - 설치된 글골은 대부분의 앱에서 사용할 수 있음
    - 트루타입(TrueType)과 오픈타입(OpenType) 글꼴을 제공함
- 시작 : 시작 메뉴에 표시되는 앱 목록, 최근에 추가된 앱, 가장 맣이 사용하는 앱 등을 지정하거나 시작 메뉴에 표시할 폴더를 선택할 수 있음

018 연결 프로그램
- 문서, 그림, 사운드 등 특정 데이터 파일을 열 때 자동으로 실행되는 앱
- 파일의 확장자에 의해 연결 프로그램이 결정됨
- 특정 하일을 선택 후 바로 가기 메뉴에서 '연결 프로그램' 선택시 하위 메뉴에 해당 파일을 열 수 있는 앱 목록이 표시됨
- 해당 파일을 열 수 있는 앱이 없을 경우 하위 메뉴가 없으므로 곧바로 하위 앱을 선택할 수 있는 창이 표시됨
- 연결 프로그램을 지정하는 창에서 연결 프로그램을 삭제해도 연결된 데이터 파일은 삭제되지 않음

019 백업 및 복구
- 백업
    - 원본 데이터의 손실에 대비해 중요한 데이터를 외부 저장장치에 저장해 두는 기능
    - 백업 주기와 백업 유지 기간 등을 지정할 수 있음
    - Windows 10은 파일 히스토리를 사용해 파일을 백업함
    - 백업된 데이터 복원 시 전체 또는 원하는 파일을 원래 위치나 원하는 위치로 복원할 수 있음
- 복구
    - PC가 제대로 실행되지 않는 경우 PC를 초기화하거나 이전 버전의 Windows 10으로 되돌릴 때 사용함
    - 이 PC 초기화
        - Windows를 다시 설치해 PC 초기화
        - 사용자가 설치한 앱과 설정 내용이 모두 제거됨
        - 사용자의 개인 파일을 유지하거나 제거할 수 있음

020 설정 -> 장치 -> 마우스/입력
- 설정 -> 장치 : 컴퓨터에 연결된 외부 장치를 확인하거나 추가로 설치할 때 사용함
- 마우스
    - 오른손잡이/왼손잡이에 맞게 마우스 단추 기능 설정
    - 휠을 한 번 돌리면 여러 줄 또는 한 화면이 스크롤 되도록 설정
    - 활성창/비활성창 구분 없이 마우스 포인터가 가리키는 창이 스크롤 되도록 설정
    - '추가 마우스 옵션' 클릭시 실행되는 '마우스 속성' 대화상자에서 세부 기능 설정 가능
- 입력
    - 추천 단어의 표시 여부 설정
    - 틀린 단어 자동 고침 사용 여부 설정
    - 입력 중인 인식 언어 기준으로 텍스트 제안 표시 여부 설정

021 장치 관리자
- 컴퓨터에 설치되어 있는 하드웨어의 종류 및 작동 여부 확인, 속성 변경, 업데이트
- 아래 방향 화살표가 표시된 장치 : 사용되지 않음
- 물음표가 표시된 장치 : 알 수 없는 장치(미설치된 장치)
- 느낌표가 표시된 장치 : 정상적으로 동작하지 않는 장치
- 각 장치의 속성을 이용해 장치의 드라이버 파일이나 IRQ, DMA, I/O 주소, 메모리 주소 등 확인/변경

022 프린터
- Windows 10에서는 대부분의 프린터를 지원 -> 프린터 연결시 자동으로 설치됨
- 설치 : 장치 -> 프린터 및 스캐너 -> 프린터 또는 스캐너 추가
- 여러 개의 프린터를 한 대의 컴퓨터에 설치 가능
- 한 개의 프린터를 네트워크로 공유해 여러 대의 컴퓨터에 설치 가능
- 프린터마다 개별적으로 이름을 붙여 설치 가능
- 이미 설치된 프린터를 다른 이름으로 재설치 가능
- 네트워크 프린터 설치시, 다른 컴퓨터에 연결된 프린터를 내 컴퓨터에 연결된 프린터처럼 사용 가능
- 네트워크 프린터 사용시 프린터의 공유 이름과 프린터가 연결되어 있는 컴퓨터의 이름을 알아야 함
- 로컬 프린터 : 컴퓨터에 직접 연결된 프린터
    - 설치 시 선택할 수 있는 포트 : LPT1, LPT2, LPT3, COM1, COM2, COM3 등
- 네트워크 프린터 : 다른 컴퓨터에 연결되어 있는 프린터
    - 포트 : 자동으로 지정됨
- 기본 프린터
    - 인쇄 명령 수행 시 특정 프린터를 지정하지 않을 경우 자동으로 인쇄 작업이 전달되는 프린터
    - 하나만 지정 가능
    - 현재 기본 프린터 해제하려면 다른 프린터를 기본 프린터로 설정
    - 프린터 이름 이래 '기본값'이라고 표시됨
    - 네트워크 프린터나 추가 설치된 프린터도 기본 프린터로 설정 가능

023 스풀(Spool) 기능
- 저속 출력장치인 프린터를 고속 중앙처리장치(CPU)와 병행처리할 때 컴퓨터 전체의 처리 효율을 높이기 위해 사용하는 기능
- 인쇄할 내용을 하드디스크에 먼저 저장하고 백그라운드 작업으로 CPU 여유시간에 틈틈이 인쇄
- 프린터가 인쇄중이라도 다른 앱을 실행하는 포그라운드 작업 가능
- 문서 전체 또는 일부를 스풀한 다음 인쇄를 시작하도록 설정 가능
- 스풀 사용시 인쇄 속도는 비교적 느려짐

024 인쇄 작업
- 인쇄 중 작업 표시줄에 프린터 아이콘이 표시됨
- 프린터 속성 대화상자 : 프린터 포트, 공유 설정 여부, 최대 해상도, 사용 가능한 용지, 프린터 정보 등 확인 가능
- 인쇄 작업이 시작된 문서도 중간에 강제로 종료시키거나, 잠시 중지시켰다가 다시 인쇄 가능
- 인쇄 대기중인 문서의 문서 이름, 인쇄 상태, 페이지 수, 크기 등 확인 가능
- 인쇄 대기중인 문서 삭제, 순서 임의 조정 가능
- 프린터 -> 모든 문서 취소 :  선택시 스풀러에 저장된 모든 인쇄 작업이 삭제됨
- 문서 -> 취소 : 선택되어 있던 인쇄 작업 삭제됨
- 인쇄 대기열에 대기중인 문서는 다른 프린터로 보낼 수 있음
- 인쇄 중에 있거나 인쇄 중 오류가 발생한 인쇄 작업은 보낼 수 없음
- 인쇄 중 오류 발생 시 해당 문서가 인쇄 대기열에서 없어질 때까지 이후 모든 인쇄 작업이 보류됨

025 드라이브 조각 모음 및 최적화
- 드라이브의 접근 속도 향상 위해 드라이브를 최적화하는 기능
- 드라이브 미디어 유형이 HDD(Hard Disk Drive)인 경우 단편화(Fragmentation)로 인해 여기저기 분산되어 저장된 파일들을 연속된 공간을 최적화시킴
- SSD(Solid State Drive)인 경우 트림(Trim) 기능을 이용해 최적화시킴
- 드라이브에 대한 접근 속도를 향상시기기 위한 것 -> 드라이브 용량 증가와는 관계 없음
- 드라이브 조각 모음 및 최적화가 불가능한 경우
    - HTFS, FAT, FAT32 이외의 파일 시스템으로 포맷된 경우
    - CD/DVD-ROM 드라이브
    - 네트워크 드라이브
    - Windows가 지원하지 않는 형식으로 압축된 드라이브

026 디스크 정리
- 디스크 여유 공간을 확보하기 위해 필요 없는 파일을 삭제하는 기능
- 디스크 정리 대상
    - 다운로드된 프로그램 파일
    - 임시 인터넷 파일
    - Windows 오류 보고서 및 피드백 진단
    - DirectX 셰이더 캐시
    - 전송 최적화 파일
    - 휴지통
    - 임시 파일
    - 미리 보기 사진 등
- 시스템 파일 정리 -> 기타 옵션 : 설치 후 사용하지 않는 앱과 시스템 복원 지점을 제거해 여유 공간 확보 가능

027 작업 관리자
- 컴퓨터에서 현재 실행중인 앱과 프로세스에 대한 정보를 제공하고 응답하지 않는 앱을 종료할 때 사용
- 작업 관리자 대화상자의 탭별 기능
    - 프로세스 : 현재 실행 중인 앱과 프로세스의 상태를 확인하고, 응답하지 않는 앱이나 프로세스를 종료 가능
    - 성능 : CPU, 메모리, 디스크, 이더넷(네트워크), GPU의 자원 사용 현황을 그래프로 표시
    - 앱 기록 : 특정 날짜 이후의 앱별 리소스 사용량을 표시
    - 시작프로그램 : Windows가 시작될 때 자동으로 실행되는 앱의 사용 여부 지정
    - 사용자
        - 현재 컴퓨터에 로그인되어 있는 모든 사용자를 보여줌
        - 특정 사용자에게 메시지를 보내거나 강제로 로그아웃 시킬 수 있음
    - 세부 정보
        - 현재 실행 중인 프로세스에 대해 CPU 및 메모리 사용에 대한 자세한 정보를 표시
        - 현재 실행 중인 프로세스를 선택해 종료 가능
    - 서비스 : 시스템의 서비스 항목을 확인하고 실행 여부 지정

028 레지스트리(Registry)
- 컴퓨터에 설치된 모든 하드웨어와 소프트웨어잇 ㅣㄹ행 정보를 한 군데에 모아 관리하는 계층적인 데이터베이스
- 레지스트리는 하드웨어 자원(IRQ, I/O 주소, DMA 등), 소프트웨어 자원(설치된 프로그램 및 속성 정보) 관리
- 레지스트리 정보는 Windows가 작동하는 동안 지속적으로 참조됨
- 레지스트리의 내용은 기계어로 되어 있어 일반 문서 편집기로 확인할 수 없음
- 수정하려면 REGEDIT와 같은 레지스트리 편입 앱 사용해야 함
- 레지스트리는 시스템과 사용자에 대한 중요한 정보를 가지고 있음 -> 문제 발생 시 시스템이 부팅되지 않을 수도
- 레지스트리의 정보는 삭제할 수 있으나 시스템 이상이 생길 수 있으므로 함부로 삭제하지 않는 것이 좋음
- 레지스트리 편집기 실행 : 실행 창에 '레지스트리 편집기' 또는 'Regedit'입력
- 레지스트리 백업 : 레지스트리 편집기 -> 파일 -> 내보내기 선택 후 내보내기할 파일 이름 지정

029 네트워크 및 인터넷
- 현재 설정되어 있는 기본 네트워크 정보를 확인하거나 네트워크 설정 사항을 변경할 수 있는 다양한 기능 제공
- 상태
    - 네트워크 상태 : 내 컴퓨터가 네트워크에 연결된 상태를 시각적으로 표시함
    - 속성
        - 네트워크 프로필 : 네트워크의 다른 컴퓨터에서 내 컴퓨터의 프린터 및 파일 등을 공유할 수 있도록 허용 여부 설정
        - 데이터 통신 연결 : 데이터 사용량의 제한 여부 설정
        - IP 설정 : IP를 자동(DHCP) 또는 수동으로 할당
        - 속성 : IPv6 주소, IPv5 주소, 물리적 주소(MAC) 등을 표시
    - 데이터 사용량
        - 최근 30일 동안의 앱별 데이터 사용량 및 현재 연결되어 있는 네트워크 표시
        - 데이터 제한 : Windows가 데이터 사용량을 제한할 수 있도록 제한 유형, 요금제 시작일, 데이터 제한 크기(MB, GB)를 설정
    - 사용 가능한 네트워크 표시 : 내 컴퓨터에서 사용 가능한 네트워크를 작업 표시줄 오른쪽의 알림 영역에 표시
    - 고급 네트워크 설정
        - 어댑터 옵션 변경 : 네트워크 어댑터의 연결 설정을 변경할 수 있는 '제어판'의 '네트워크 연결'창이 실행됨
        - 네트워크 및 공유 센터 : 네트워크 정보를 확인하고 설정 사항을 변경할 수 있는 '네트워크 및 공유 센터'창이 실행됨
        - 네트워크 문제 해결사 : 네트워크 문제를 진단하고 해결할 수 있는 'Windows 네트워크 진단' 마법사가 실행됨
    - 하드웨어 및 연결 속성 보기 : 네트워크 이름, 설명, 물리적 주소 등의 네트워크 정보를 표시함
    - Windows 방화벽 : 방화벽을 설정하고 네트워크 및 인터넷 연결에 발생하는 상황을 확인하는 '방화벽 및 네트워크 보호'창이 실행됨
    - 네트워크 초기화 : 네트워크 어댑터를 제거한 후 다시 설치하고 네트워킹 구성 요소를 기본값으로 설정

030 TCP/IP의 구성 요소
- 인터넷 접속을 위해 반드시 지정해야 하는 구성 요소
    - IPv4 : IPv4 주소, 서브넷 마스크, 기본 게이트웨이, DNS 서버 주소
    - IPv6 : IPv6 주소, 서브넷 접두사 길이, 기본 게이트웨이, DNS 서버 주소
- TCP/IP 속성
    - IP 주소 : 인터넷에 연결된 호스트 컴퓨터의 유일한 주소 -> 네트워크 주소 + 호스트 주소
    - 서브넷 접두사 길이 : IPv6 주소의 네트워크 주소와 호스트 주소를 구별하기 위해 IPv6 수신인에게 허용하는 서브넷 마스크 부분의 길이를 비트로 표현한 것
    - 서브넷 마스크 : IPv4 주소의 네트워크 주소와 호스트 주소를 구별하기 위해 IPv4 수신인에게 허용하는 32비트 주소
    - 게이트웨이
        - 다른 네트워크와의 데이터 교환을 위한 출입구 역할
        - LAN에서 다른 네트워크에 데이터를 보내거나 받아들이는 역할을 하는 장치를 지정함
    - DNS 서버 주소
        - DNS 서버 : 문자 형태로 된 도메인 네임을 숫자로 된 IP 주소로 변환해 주는 서버
        - DNS 서버 주소에는 이 서버가 있는 곳의 IP 주소를 지정함
    - DHCP 서버 : 컴퓨터에 IP 주소를 자동으로 할당해 주는 서버

031 문제 해결
- 메모리 용량 문제 해결
    - 불필요한 앱 종료
    - 시작프로그램 폴더 안의 불필요한 앱 삭제
    - 작업관리자의 시작프로그램 탭에서 불필요한 앱의 실행 해제
    - 가상 메모리의 크기를 적절히 설정
- 하드디스크 용량 문제 해결
    - 불필요한 파일은 백업한 다음 하드디스크에서 삭제
    - 디스크 정리를 수행해 불필요한 파일 삭제
- 비정상적인 문제 해결
    - 아전모드로 부팅해 문제 해결 후 정상 모드로 재부팅
    - 시스템 복구 드라이브를 이용해 시스템 복구 수행
    - 시스템 복원 기능을 이용해 정상적으로 부팅되던 시점으로 복원

032 시스템 복구를 해야 하는 경우
- 새 장치를 설치한 후 시스템이 불안정할 때
- 로그온 화면이 나타나지 않고, 운영체제를 시작할 수 없을 때
- 누락되거나 손상된 데이터 파일을 이전 버전으로 되돌릴 때

033 컴퓨터의 분류
- 처리 능력
    - 얼마나 많은 데이터를 얼마나 빠르게 처리할 수 있는가를 기준
    - 종류 : 슈퍼 컴퓨터, 메인 프레임, 미니 컴퓨터, 마이크로 컴퓨터 등
- 데이터 취급
    - 컴퓨터에서 처리하는 데이터의 형태인 디지털형, 아날로그형, 혼합형을 기준
    - 종류 : 디지털 컴퓨터, 아날로그 컴퓨터, 하이브리드 컴퓨터
- 사용 용도
    - 컴퓨터를 어떠한 목적으로 사용하는가를 기준
    - 종류 : 범용 컴퓨터, 전용 컴퓨터

034 컴퓨터의 분류-데이터 취급
- 디지털 데이터 : 시간에 따라 이산적(비연속적)으로 변하는 정보 -> 결과를 숫자나 문자를 조합해 표시
- 아날로그 데이터 : 시간에 따라 크기가 연속적으로 변하는 정보 -> 결과를 곡선이나 그래프로 표시
- 디지털 데이터는 복호화(Decoding) 과정을 통해 아날로그 데이터로, 아날로그 데이터는 부호화(Encoding) 과정을 통해 디지털 데이터로 변환

035 자료 구성의 단위
- 비트(Bit) : 자료(정보) 표현의 최소 단위 -> 2가지 상태를 표시하는 2진수 1자리
- 니블(Nibble) : 1니블 = 4비트
- 바이트(Byte) : 문자를 표현하는 최소 단위 -> 1바이트 = 8비트
- 워드(Word) : CPU가 한 번에 처리할 수 있는 명령 단위 -> 하프워드(2바이트), 풀워드(4바이트), 더블워드(8바이트)
- 필드(Field) : 파일 구성의 최소 단위, 의미 있는 정보를 표현하는 최소 단위
- 레코드(Record) : 하나 이상의 관련된 필드가 모여서 구성됨 -> 컴퓨터 내부의 자료 처리 단위
- 파일(File) : 프로그램 구성의 기본 단위 -> 여러 레코드가 모여 구성됨
- 데이터베이스(Database) : 여러 개의 관련된 파일의 집합

036 자료의 표현 방식-숫자
- 10진 연산
    - 언팩(Unpack) 연산 : 1바이트로 10진수 1자리를 표현
    - 팩(Pack)연산 : 1파이트로 10진수 2자리를 표현
- 2진 연산
    - 2진 정수 데이터의 표현에 사용
    - 표현할 수 있는 범위가 작지만 연산 속도가 빠름
- 부동 소수점 연산
    - 실수 데이터 표현과 연산에 사용
    - 숫자를 부호(1비트), 지수부(7비트), 가수부(소수부)로 나누어 표현함
    - 고정 소수점 연산에 비해 실행 시간이 많이 걸리나 매우 큰 수나 매우 작은 수를 표현하는데 적합함

037 자료의 표현 방식-문자
- BCD 코드(2진화 10진)
    - 하나의 문자를 2개의 Zone 비트와 4개의 Digit 비트로 표현
    - 2^6=64가지의 문자 표현
    - 영문 소문자 표현 불가
- ASCII 코드(미국 표준)
    - 하나의 문자를 3개의 Zone 비트와 4개의 Digit 비트로 표현
    - 2^7=128가지의 문자 표현
    - 데이터 통신용으로 사용
    - 확장 ASCII는 8비트 사용 -> 2^8=256가지의 문자 표현
- EBCDIC 코드(확장 2진화 10진)
    - BCD 코드를 확장한 것
    - 하나의 문자를 4개의 Zone 비트와 4개의 Digit 비트로 표현
    - 2^8=256가지의 문자 표현
    - 대형 컴퓨터에서 사용
- 유니코드(Unicode)
    - 전 세계 모든 문자를 2바이트로 표현할 수 있는 국제 표준 코드 -> 정보 처리/정보 교환용
    - 문자 1개에 부여되는 값을 2바이트(16비트)로 통일 -> 최대 65,536자의 글자 코드화
    - 한글은 조합형, 완성형, 옛글자 모두 표현 가능

038 제어장치(Control Unit)
- 컴퓨터에 있는 모든 장치들의 동작을 지시하고 제어하는 장치
- 프로그램 카운터(PC) : 다음 번에 실행할 명령어의 번지를 기억하는 레지스터
- 명령 레지스터(IR) : 현재 실행중인 명령의 내용을 기억하는 레지스터
- 명령 해독기(Decoder) : 명령 레지스터에 있는 명령어를 해독하는 회로
- 부호기(Encoder) : 해독된 명령에 따라 각 장치로 보낼 제어 신호를 생성하는 회로
- 메모리 주소 레지스터(MAR) : 기억장치를 출입하는 데이터의 번지를 기억하는 레지스터
- 메모리 버퍼 레지스터(MBR) : 기억장치를 출입하는 데이터가 잠시 기억되는 레지스터

039 연산장치(ALU; Arithmetic & Logic Unit)
- 제어장치의 명령에 따라 실제로 연산을 수행하는 장치
- 가산기(Adder) : 2진수의 덧셈을 수행하는 회로
- 보수기(Complementor) : 뺄셈을 위해 입력된 값을 보수로 변환하는 회로
- 누산기(Accumulator) " 연산 결과를 일시적으로 저장하는 레지스터
- 데이터 레지스터 : 연산에 사용될 데이터를 기억하는 레지스터
- 상태 레지스터 : 연산중에 발생하는 여러 가지 상태값을 기억하는 레지스터
- 인덱스 레지스터 : 주소 변경을 위해 사용되는 레지스터

040 레지스터(Register)
- CPU 내부에서 처리할 명령어나 연산의 중간 결과값 등을 일시적으로 기억하는 임시 기억장소
- 플립플롭(Flip-Flop)이나 래치(Latch)들을 연결함
- 메모리 중에서 가장 속도가 빠름
- 레지스터의 크기는 컴퓨터가 한 번에 처리할 수 있는 데이터의 크기를 의미함

041 ROM(Read Only Memory)
- 기억된 내용을 읽을 수만 있는 기억장치 -> 일반적으로 쓰기는 불가능
- 전원이 꺼져도 기억된 내용이 지워지지 않는 비휘발성 메모리
- 기본 입/출력 시스템(BIOS), 글자 폰트, 자가진단 프로그램(POST) 등이 저장됨

042 RAM(Random Access Memory)
- 자유롭게 읽고 쓸 수 있는 메모리
- 전원이 꺼지면 기억된 내용이 모두 사라지는 휘발성 메모리
- 현재 사용중인 프로그램이나 데이터가 저장됨
- 일반적으로 '주기억장치'라고 하면 RAM을 의미함

043 기타 메모리
- 캐시 메모리(Cache Memory)
    - CPU와 주기억장치 사이에서 컴퓨터의 처리 속도를 향상시키는 역할
    - 접근 속도가 빠른 정적 램(SRMA)을 사용하며 용량이 주기억장치보다 작게 구성됨
    - 적중률(Hit Ratio)이 높을수록 시스템의 전체적인 속도가 향상됨
- 가상 메모리(Virtual Memory)
    - 보조기억장치(하드디스크)의 일부를 주기억장치처럼 사용하는 메모리 기법
    - 주기억장치보다 큰 프로그램을 불러와 실행해야 할 때 유용하게 사용됨
- 버퍼 메모리(Buffer Memory) : 두 장치 간에 데이터를 주고받을 때 속도 차이를 해결하기 위한 임시 저장 공간
- 연관 메모리(Associative Memory)
    - 기억장치에 저장된 정보에 접근할 때 주소 대신 기억된 내용의 일부를 이용해 접근하는 장치
    - 정보 검색이 신속함
- 플래시 메모리(Flash Memory)
    - EEPROM의 일종
    - 개인용 정보 단말기, 스마트폰, 디지털 카메라 등에 사용함

044 SSD(Solid State Drive)
- 하드디스크 드라이브(HDD)왑 ㅣ슷하게 동작하면서 HDD와는 달리 기계적 장치가 없음 -> 반도체를 이용해 정보 저장
- 고속으로 데이터를 입/출력할 수 있고, 기계적인 지연이나 실패율이 거의 없음
- 디스크가 아닌 메모리에 데이터를 기록하므로 배드 섹터가 발생하지 않음
- 발열/소음과 전력 소모가 적음
- 소형화/경량화할 수 있음
- 하드디스크에 비해 외부 충격에 강하나 저장 용량당 가격은 더 비쌈

045 표시장치 관련 용어
- 픽셀(Pixel, 화서) : 모니터 화면을 구성하는 가장 작은 단위 -> 수가 많을수록 해상도가 높아짐
- 해상도(Resolution)
    - 모니터 등의 출력장치가 내용을 얼마나 선명하게 표현할 수 있느냐를 나타내는 단위
    - 픽셀 수에 따라 결정됨
- 모니터의 크기 : 대각선의 길이를 센티미터 단위로 표시함
- 재생율(Refresh Rate)
    - 픽셀들이 밝게 빛나는 것을 유지하도록 하기 위한 1초당 재충전 횟수
    - 높을수록 모니터의 깜박임이 줄어듦
- 점 간격(Dot Pitch) : 픽셀들 사이의 공간 -> 가까울수록 해상도가 높음
- 플리커 프리(Flicker Free) : 모니터의 깜빡임 현상인 플리커(Flicker)를 제거해 눈의 피로나 두통 등의 증상을 줄요주는 기술

046 3D 프린터
- 3차원의 입체적인 물품을 만드는 프린터
- 잉크를 종이 표면에 분사해 2D 이미지를 인쇄한는 잉크젯 프린터와 원리가 같음
- 적층형 : 레이어로 쌓아 입체 형상을 만듦
- 절삭형 : 큰 덩어리를 조각하듯이 깎아 만듦
- 의료, 기계, 건축, 예술, 우주 등 많은 문야에서 활용됨
- 출력 속도 단위 : MMS(MilliMeters per Second)

047 프린터 관련 단위
- CPS(Character Per Second) : 1초에 출력되는 글자 수 -> 도트 매트릭스 및 시리얼 프린터
- LPM(Line Per Minute) : 1분에 출력되는 줄(Line) 수 => 라인 프린터
- PPM(Page Per Minute) : 1분에 출력되는 페이지 수 -> 잉크젯 및 레이저 프린터
- MMS(MilliMeters per Second) : 1초에 이동하는 노즐의 거리 -> 3D 프린터
- DPI(Dot Per Inch) : 1인치에 출력되는 점의 수 -> 출력물의 인쇄 품질(해상도)을 나타내는 단위

048 인터럽트(Interrupt)
- 프로그램 실행 도중 예기치 않은 상황이 발생할 경우 현재 작업을 일시 중단하고 발생된 상황을 우선 처리한 후 실행중인 작업으로 복귀해 계속 처리하는 것
- 외부 인터럽트 : 입/출력장치, 타이밍 장치, 전원 등의 외부적인 요인
- 내부 인터럽트 : 잘못된 명령이나 데이터를 사용할 떄 발생 -> 트랩(Trap)
- 소프트웨어 인터럽트
    - 프로그램 처리 중 명령의 요청에 의해 발생
    - 운영체제의 감시 프로그램을 호출하는 SVC(Super Visor Call) 인터럽트 등이 있음

049 채널(Channel)
- 주변장치에 대한 제어 권한을 CPU로부터 넘겨받아 CPU 대신 입/출력을 관리함
- CPU와 입/출력장치 사이의 속도 차이로 인한 문제점을 해결하기 위해 사용됨
- 입/출력만을 목적으로 만든 처리기 -> IOP(Input Output Processor)
- 입/출력 작업이 끝나면 CPU에게 인터럽트 신호를 보냄
- 종류
    - 셀렉터(Selector) 채널 : 고속의 입/출력장치를 제어하는 채널
    - 멀티플렉서(Multiplexer) 채널 : 저속의 입/출력장치를 제어하는 채널
    - 블록 멀티플렉서(Block Multiplexer) 채널 : 셀렉터와 멀티플렉서 채널의 기능이 혼합된 채널

050 마이크로프로세서(Microprocessor)
- 제어장치, 연산장치, 레지스터가 한 개의 반도체 칩(IC)에 내장된 프로세서
- 개인용 컴퓨터에서 CPU로 사용됨
- 클럭 주파수와 내부 버스의 폭(Bandwidth)으로 수로 성능을 평가함
- 기본적인 처리 속도는 트랜지스터의 집적도에 따라 결정됨
- 작은 규모의 임베디드 시스템이나 휴대용 기기에서부터 메인 프레임이나 슈퍼 컴퓨터까지 사용됨
- 설계방식에 따라 RISC와 CISC로 구분됨

051 버스(Bus)
- 컴퓨터에서 데이터를 주고받는 통로
- 사용 용도에 따라 구분
    - 내부 버스 : CPU 내부에서 레지스터 간의 데이터 전송에 사용되는 통로
    - 외부 버스(시스템 버스) : CPU와 주변장치 간의 데이터 전송에 사용되는 통로
        - 제어 버스 : 제어장치의 제어 신호가 각 장치로 전달되는 통로
        - 주소 버스 : 주기억장치의 주소가 각 장치로 전달되는 통로
        - 데이터 버스 : 각 장치별로 필요한 데이터가 전달되는 통로
    - 확장 버스
        - 메인보드에서 지원하는 기능 외에 달느 기능을 지원하는 장치를 연결하는 부분
        - 끼울 수 있는 슬록 형태이기 떄문에 확장 슬록이라고도 함

052 포트(Port)
- USB(범용 직렬 버스)
    - 기존의 직렬, 병렬 PS/2 포트를 통합한 직렬 포트의 일종
    - 마우스, 키보드, 모니터, PC 카메라, 프린터, 디지털 카메라와 같은 주변장치를 최대 127개까지 연결함
    - USB를 지원하는 일부 주변기기는 별도의 전원이 필요함
    - 핫 플러그인(Hot Plug In)과 플러그 앤 플레이(Plug & play) 지원
    - 전송 속도 : USB 1.0(1.5Mbps), 1.1(12Mbps), 2.0(480Mbps), 3.0(5Gbps), 3.1(10Gbps)
    - 연결 단자 색상 : 2.0 이하(검정색 또는 흰색), 3.0(파란색), 3.1(하늘색 또는 빨간색)
- 블루투스(Bluetooth)
    - 스웨덴의 에릭슨에 의해 최초로 개발된 근거리 무선 통신을 가능하게 해주는 통신 방식
    - PANs(Personal Area Networks)의 산업 표준
    - 핸드폰, 노트북과 같은 휴대 가능한 장치들 간의 양방향 정보 전송이 가능함
- HDMI(High Definition Multimedia Interface)
    - 영상과 음향 신호를 압축하지 않고 통합해 전송하는 고선명 멀티미디어 인터페이스
    - S-비디오, 컴포지트 등의 아날로그 케이블보다 고품질의 음향 및 영상을 제공
- DP(Display Port)
    - VESA에서 제정한 디지털 디스플레이 인터페이스
    - 대역폭이 넓고 확장성이 뛰어나 여러 기기에 고품질의 영상 및 음향 신호를 동시 전송할 수 있음 -> HDMI를 대체할 인터페이스

053 바이오스(Bios)
- 컴퓨터의 기본 입/출력 장치나 메모리 등 하드웨어 작동에 필요한 명령을 모아 놓은 프로그램
- 전원이 켜지면 POST(Power On Self Test)를 통해 컴퓨터를 점검한 후 사용 가능한 장치들을 초기화
- 윈도우가 시작될 떄까지 부팅 과정을 이끔
- ROM에 저장되어 있어 ROM-BIOS라고도 함
- 하드웨어와 소프트웨어의 중간 형태인 펌웨어(Firmware)
- 최근에는 플래시 롬(Flash Rom)에 저장됨 -> 칩을 교환하지 않고도 바이오스를 업그레이드할 수 있음
- CMOS 셋업 프로그램을 이용해 일부 BIOS 정보를 설정할 수 있음

054 펌웨어
- 하드웨어의 동작을 지시하는 소프트웨어이지만 하드웨어적으로 구성되어 하드웨어의 일부분으로도 볼 수 있는 제품
- 하드웨어 교체없이 소프트웨어 업그레이드만으로 시스템의 성능을 높이기 위한 목적으로 사용
- 하드웨어와 소프트웨어의 중간적인 성격
- 주로 ROM에 반영구적으로 저장됨
- 하드웨어를 제어/관리하는 역할
- 기계어 처리, 데이터 전송, 부동 소수점 연산, 채널 제어 등의 처리 루틴을 가짐
- 읽기/쓰기가 가능한 플래시 롬에 저장되기 때문에 내용을 쉽게 변경하거나 추가/삭제할 수 있음
- 마이크로 프로그램 : 펌웨어로 만들어져 있는 프로그램

055 RAID(Redundant Array Of Inexpensice Disk)
- 여러 개의 하드디스크를 한 개의 하드디스크처럼 관리하는 관리 기술
- 중요한 자료를 다루는 서버(Server)에서 주요 사용되는 방식
- 하드디스크의 모음뿐만 아니라 자동으로 복제대 백업 정책도 구현함
- 사용시 데이터의 안정성이 높아지고, 데이터 복구가 용이하며, 전송 속도도 빨라짐
- RAID Level : 하드디스크에서 데이터를 저장하는 방식 -> 숫자가 클수록 저장장치의 신뢰성이 높고 효율성이 좋음
- 시스템 장애 시 컴퓨터를 끄지 않고 디스크를 교체할 수 있음
- 미러링과 스트라이핑 기술을 융합해 사용함
    - 미러링(Mirroring) 방식
        - 데이터를 두 개의 디스크에 동일하게 기록하는 방법
        - 한쪽 디스크의 데이터 손상 시 다른 한쪽 디스크를 이용해 복구
    - 스트라이핑(Striping) 방식
        - 데이터를 여러 개의 디스크에 나눠서 기록하는 방법
        - 자료를 읽고 스는 시간을 단축할 수는 있으나, 디스크가 한 개라도 손상되면 데이터를 사용할 수 없게 됨

056 시스템 관리
- 컴퓨터 전원은 사용중인 프로그램을 모두 종료한 후 끔
- 컴퓨터의 설치는 직사광선과 습기가 많은 장소, 자성이 강한 물체가 있는 곳은 피함
- 너무 자주 켜고 끄는 재부팅은 시스템에 충격을 가해 부품의 수명을 단축시키는 행위
- 중요한 데이터는 정기적으로 백업, 가급적 불필요한 프로그램은 설치하지 않음
- 정기적으로 시스템 최적화 프로그램을 사용해 PC를 점검함
- 전원 관리 장치 : 정전, 전압의 불안정 등에 대비해 사용하는 장치
    - 무정전 전원 공급장치(UPS) : 정전되었을 떄 시스템에 일정 시간 동안 전원을 공급해 주는 장치
    - 자동 전압 조절기(AVR) : 입력 전압의 변동에 관계없이 항상 일정한 출력 전압을 유지시켜 주는 장치
    - 정전압 정주파장치(CVCF) : 전압과 주파수를 항상 일정하게 유지시켜 주는 장치
    - 서지 보호기(Surge Protector) : 전압이나 전류의 갑작스런 증가(서지)에 의한 손상을 보호하는 장치

057 업그레이드(Upgrade)
- 소프트웨어적 업그레이드 : 향상된 기능을 가진 새 버전으로 교체
- 하드웨어적 업그레이드 : 컴퓨터 처리 성능의 개선
    - CPU 업그레이드
        - 시스템의 성능을 향상시킬 수 있는 가장 확실한 방법 -> 주로 메인보드와 함께 교체헤 등급을 높임
        - CPU의 핀 수나 크기를 확인한 후 교체
    - RAM 업그레이드
        - 처리 속도가 느려지거나 제대로 동작하지 않을 경우 가장 먼저 고려사항
        - 램을 추가할 떄는 현재 설치되어 있는 램과 핀 수가 같은 램으로 추가해야 함
    - HDD/SDD 업그레이드 : 부족한 하드디스크 공간을 확보하기 위해 하드디스크를 추가하거나 용량이 큰 것으로 교체

058 사용권에 따른 소프트웨어 분류
- 상용 소프트웨어 : 정식으로 대가를 지불하고 사용해야 하는 소프트웨어
- 셰어웨어(Shareware)
    - 기능 혹은 사용 기간에 제한을 두어 배포
    - 무료로 사용 가능
    - 일정 기간 사용해 보고 정식 프로그램을 구입할 수 있음
- 프리웨어(Freeware)
    - 무료로 사용 또는 배포가 가능한 소프트웨어
    - 주로 인터넷을 통해 배포
- 공개 소프트웨어(Open Software)
    - 개발자가 소스를 공개한 소프트웨어
    - 누구나 자유롭게 사용하고 수정 및 재배포할 수 있음
- 데모(Demo) 버전 : 정식 프로그램의 기능을 홍보하기 위해 사용 기간이나 기능을 제한해 배포하는 소프트웨어
- 알파(Alpha) 버전 : 베타테스트 전, 제작 회사 내에서 테스트할 목적으로 제작하는 소프트웨어
- 베타(Beta) 버전 : 정식 프로그램 출시 전, 테스트를 목적으로 일반인에게 공개하는 소프트웨어
- 패치(Patch) 버전 : 이미 제작해 배포된 프로그램의 오류 수정이나 성능 향상을 위해 프로그램의 일부 파일을 변경해주는 소프트웨어
- 애드웨어(Adware) : 프리웨어나 셰어웨어 등에서 광고를 보는 대가로 사용이 허용되는 소프트웨어
- 번들(Bundle) : 특정 하드웨어나 소프트웨어 구입시 무료로 끼워주는 소프트웨어

059 운영체제(OS; Operating System)
- 사용자의 편의를 도모함과 동시에 시스템의 생산성을 높이기 위한 프로그램의 모임
- 사용자와 컴퓨터 사이에서 중재자 역할을 함
- 가장 대표적인 시스템 소프트웨어
- 컴퓨터를 사용하기 위해 기본적으로 필요한 소프트웨어 -> 반드시 설치해야 함
- 컴퓨터가 동작하는 동안 주기억장치에 위치함
- 주요 기능
    - 프로세스, 기억장치, 주변장치, 입/출력장치, 파일 등을 관리
    - 사용자들 간의 하드웨어 공동 사용 및 자원의 스케줄링 등 수행
    - 사용자와 시스템 간의 편리한 인터페이스 제공
    - 데이터 관리, 데이터 및 자원의 공유 기능 제공
- 종류 : Linux, Unix, Windows, MS- DOS 등
- 구성
    - 제어 프로그램
        - 감시 프로그램 : 자원의 할당 및 시스템 전체의 작동 상태를 감시
        - 작업 관리 프로그램 : 작업이 정상적으로 처리될 수 있도록 작업의 순서와 방법을 관리
        - 데이터 관리 프로그램 : 작업에 사용되는 데이터와 파일의 표준적인 처리 및 전송을 관리
    - 처리 프로그램
        - 언어 번역 프로그램 : 사용자가 고급언어로 작성한 원시 프로그램을 기계어 형태의 목적 프로그램으로 변환
        - 서비스 프로그램
            - 사용자가 컴퓨터를 더욱 효율적으로 사용할 수 있도록 제작된 프로그램
            - 연계 편집, 정렬/병합(Sort/Merge), 유틸리티 프로그램 등

060 운영체제의 목적
- 응답 시간 단축, 처리 능력 증대, 신뢰도 향상, 사용 가능도 증대
- 처리 능력(Throughput) : 일정 시간 내에 시스템이 처리하는 일의 양
- 응답 시간(Turn Around Time) : 시스템에 작업을 의뢰한 시간부터 처리가 완료될 때까지 걸린 시간
- 사용 가능도(Availability) : 시스템을 사용할 필요가 있을 때 즉시 사용 가능한 정도
- 신뢰도(Reliability) : 시스템이 주어진 문제를 정확하게 해결하는 정도

061 문서관리의 개념
- 일괄 처리 시스템 : 처리할 데이터를 일정량 또는 일정 기간 동안 모았다가 한꺼번에 처리하는 방식
- 실시간 처리 시스템 : 처리할 데이터가 생겨날 때마다 바로 처리하는 방식
- 다중 프로그래밍 시스템 : 한 대의 CPU로 여러 개의 프로그램을 동시에 처리하는 방식
- 시분할 시스템 : 한 대의 시스템을 여러 사용자가 동시에 사용하는 방식
- 다중 처리 시스템 : 처리 속도를 향상시킬 목적으로 하나의 컴퓨터에 여러 개의 CPU를 설치해 프로그램을 처리하는 방식
- 분산 처리 시스템 : 지역적으로 분산된 여러 대의 컴퓨터를 연결해 작업을 분담해 처리하는 방식
- 임베디드 시스템 : 마이크로프로세서에 특정 기능을 수행하는 응용 프로그램을 탑재해 컴퓨터의 기능을 수행
- 듀얼 시스템 : 두 개의 컴퓨터가 같은 업무를 동시에 처리
- 듀플레그 시스템 : 두 대의 컴퓨터를 설치해 한 쪽 컴퓨터가 가동중일 떄 다른 컴퓨터는 대기

066 망의 구성 형태
- 성형(중앙 집중형) : 모든 노드가 중앙에 1:1로 연결 -> 고장 발견이 쉽고 유지 보수 및 확장 용이
- 링형(루프형) : 인접한 단말기 서로 연결 -> 양방향 데이터 전송 가능 -> 하나 고장나면 전체 영향
- 버스형 : 한 통신 회선에 여러 대의 단말장치가 연결됨
- 계층형(분산형) : 분산 처리 시스템 구성 방식
- 망형 : 모든 지점의 단말장치를 서로 연결 -> 응답시간 빠르고 연결성 높음 -> 단말 추가/제거가 어려움

067 네트워크 관련 장비
- 허브 : 네트워크 구성시 한꺼번에 여러 대의 컴퓨터를 연결하는 장치 -> OSI 계층 중 물리 계층
- 리피터 : 디지털 신호 장거리 전송 위해 수신한 신호를 재생시키거나 출력 전압 높여 전송 -> 물리계층
- 브리지 : 리피터 기능 + 네트워크 분할 통해 트래픽 감소, 물리적으로 다른 네트워크 연결 -> 데이터 링크 계층
- 라우터 : 네트워크 간 연결시 필요, 데이터 전송 시 최적의 경로 설정해 전송 -> 네트워크 계층
- 게이트웨이 : LAN에서 다른 네트워크에 데이터 출입구 역할 -> 전송 계층

068 인트라넷/엑스트라넷
- 인트라넷 : 기업내 정보 시스템
- 엑스트라넷 : 기업간 인트라넷을 서로 연결

069 인터넷 관련 용어
- VolP : 음성 신호를 합축해 IP를 사용하는 인터넷을 통해 전송하는 방식 -> 기존 전화망의 시내전화 요금 수준으로 국제전화 가능
- 모뎀 : 디지털 데이터를 아날로그 신호로 변환(변조) 하고 다시 디지털 데이터로 변환(복조)
- 코덱 : 모뎀과 반대 역할

070 IPv6
- 현재 사용하는 IPv4의 주소 부족 문제 해결
- 16비트씩 8부분, 총 128비트
- 주소 각 부분이 0으로 연속된 경우 0을 생략해'::'로 표시, 한 부분이 0으로 연속된 경우 0을 생략해 ':'로 표시
- IPv4와 호환성이 뛰어남, 비교적 자료 전송 속도가 빠름
- 보안문제 해결 가능, 주소의 확장성/융통성/연동성이 뛰어남

071 DNS/도메인 네임
- DNS(Domain Name System)
    - 문자로 된 도메인 네임을 숫자로 된 IP 주소로 바꾸는 시스템
    - DNS에 등록된 모든 호스트들을 도메인별로 계층화시켜 관리
- 도메인 네임 : 숫자로 된 IP 주소를 사람이 이해하기 쉬운 문자 형태로 표현한 것

072 URL(Uniform Resource Locator)
- 인터넷 상에 존재하는 각종 자원이 있는 위치를 나타내는 표준 주소 체계
- 형식 : 프로토콜://호스트(서버) 주소[:포트 번호][/파일 경로]
    - 프로토콜 : 인터넷 서비스의 종류 -> http(WWW), ftp(FTP), telnet(Telnet), news(Usenet), mailto(E-Mail) 등
    - 서버 주소 : 검색할 정보가 위치한 서버의 호스트 주소
    - 포트 번호 : TCP 접속에 사용되는 포트 번호
    - 파일 경로 : 서비스에 접속한 후 실제 정보가 있는 경로

073 프로토콜(Protocol)
- 네트워크에서 서로 다른 컴퓨터들 간에 정보 교환을 할 수 있게 해주는 통신 규약
- 기능
    - 흐름 제어 : 통신망에 흐르는 패킷 수를 조절 -> 시스템 전체의 안전성 유지
    - 동기화 : 정보 전송 위해 송/수신기가 같은 상태를 유지하게 함
    - 오류 검술 : 데이터의 전송 도중 발생하는 오류 검출

074 OSI 7계층
- 물리 계층 : 전송에 필요한 두 장치 간의 실제 접속과 절단 등 기계적, 전기적, 기능적, 절차적 특성 정의
- 데이터 링크 계층
    - 두 개의 인접한 개방 시스템들 간에 신뢰성 있고 효율적인 정보 전송
    - 프름 제어, 프레임 동기화, 오류 제어, 순서 제어 기능
    - 링크의 확립, 유지, 단결 수단 제공
- 네트워크 계층
    - 개방 시스템들 간의 네트워크 연결 관리, 데이터 교환 및 중계
    - 경로 설정(라우팅), 트래픽 제어, 패킷 정보 전송
- 전송 계층
    - 종단 시스템 간의 신뢰성 있고 투명한 데이터 전송 가능
    - 전송 연결 설정, 데이터 전송, 연결 해제 기능
- 세션 계층
    - 송수신측간의 관련성 유지, 대화 제어
    - 대화 구성 및 동기 제어, 데이터 교환 관리 기능
- 표현 계층
    - 응용 계층으로부터 받은 데이터를 세션 계층에 맞게, 세션 계층에서 받은 데이터는 응용 계층에 맞게 변환
    - 코드 변환, 데이터 암호화, 데이터 압축, 구문 검색, 정보 형식 변환 기능
- 응용 계능
    - 사용자가 OSI 환경에 접근할 수 있도록 서비스 제공
    - 응용 프로세스 간의 정보 교환, 파일 전송 등의 전송 제어 기능

075 TCP/IP
- 인터넷에 연결된 서로 다른 기종의 컴퓨터들 간에 데이터를 주고받을 수 있도록 하는 표준 프로토콜
- TCP
    - 메시지를 송/수신자의 주소와 정보로 묶어 패킷 단위로 나눔
    - 전송 데이터의 흐름 제어, 데이터의 에러 유무 검사
    - 전송 계층
- IP
    - 패킷 주소를 해석하고 경로 설정, 다음 호스트로 전송
    - 네트워크 계층

076 HTTP/ARP/RARP
- HTTP
    - 하이퍼텍스트 문서 전송 위해 사용하는 프로토콜
    - 서비스를 요청하거나 응답하는 프로토콜 구조
    - HTTPS : 보안 강화된 버전
- ARP : IP 주소를 이용해 물리적인 MAC 주소를 찾아주는 프로토콜
- RARP : 물리적 MAC 주소를 IP 주소로 변환하는 프로토콜

078 FTP(파일 전송 프로토콜)
- 컴퓨터와 인터넷 사이에서 파일을 주고받을 수 있도록 하는 원격 파일 전송 프로토콜
- 파일의 전송과 수신, 삭제, 이름 변경 등
- Binary 모드 : 그림 파일, 동영상 파일, 압축된 형태의 파일 전송시
- ASCII 모드 : 텍스트 파일 전송 시

080 ICT 신기술 관련 용어
- 클라우딩 컴퓨팅 : 하드웨어/소프트웨어 등의 컴퓨팅 자원을 자신이 필요한 만큼 빌려 쓰고 사용요금 지급하는 방식의 서비스
- 사물 인터넷(IoT) : 모든 사물을 네트워크로 연결해 언제 어디서나 서로 소통할 수 있게 하는 정보 통신 환경 -> 개인 맞춤형 스마트 서비스 지향
- 위치 기반 서비스(LBS) : 통신 기술, GPS, 데이터베이스를 이용해 위치와 부가 서비스 제공
- 테더링(Tethering) : 인터넷에 연결된 기기를 이용해 다른 기기도 인터넷 사용이 가능하게 함
- 텔레메틱스 : 자동차에 정보 통신 기술과 정보 처리기술을 융합
- 핀테크 : 금육과 기술의 융합을 통한 금융 서비스 및 산업 변화
- LWPA(저전력 광역 통신망) : 저전력, 저비용을 기반으로 소량의 데이터의 장거리 전송과 안정적인 통신 지원 -> IoT에서 무선 광역 네트워크로 사용
- 메타버스 : 현실 세계와 같은 3차원 가상 세계
- 와이파이
- USN : 모든 사물에 부착된 RFID 태그 또는 센서를 통해 탐지된 환경 정보를 네트워크와 연결해 실시간으로 수집/관리하는 네트워크 시스템

082 사운드 카드 관련 용어
- 사운드 카드 : 컴퓨터에서 소리를 재생하거나 녹음하는데 사용되는 장치
- 샘플링 : 음성, 영상 등의 아날로그 신호를 일정 시간 간격으로 검출하는 단계
- 샘플링률 : 아날로그 신호를 디지털 신호로 변환하는 횟수 -> 높을수록 원음에 가까움
- 샘플링 주파수 : 1초 동안 샘플링되는 횟수 -> 클수록 고음역대까지 폭넓게 가능하지만 많은 기억 용량 필요
- 샘플링 비트수 : 표현할 수 있는 서로 다른 음의 종류 -> 많을수록 자연음에 가까움

083 스트리밍 기술
- 웹에서 오디오, 비디오 등의 멀티미디어 데이터를 다운로드하면서 동시에 재생해주는 기술
- 수신한 데이터는 일반적으로 컴퓨터에 저장할 수 없음
- 재생 가능한 데이터 형식 : rma, asf, wmv, asx, wmp, wma 등

084 압축 프로그램
- 압축 : 중복되는 데이터를 이용해 파일 크기를 줄이는 것 -> 디스크 공간 효율적 사용
- 재압축은 파일 크기 변화 없음

085 그래픽 기법
- 디더링 : 제한된 색상을 조합해 복잡한 색이나 새로운 색 만드는 작업
- 렌더링 : 물체의 모형에 명암과 색상을 입혀 사실감을 더해 주는 작업
- 모델링 : 표현될 물체의 3차원 그래픽 형상을 정하는 작업 -> 렌더링 하기 전 수행
- 모핑 : 두 개의 이미지를 부드럽게 연결해 변환/통합하는 것
- 필터링 : 이미 작성된 그림을 여러 가지 형태의 새로운 이미지로 바꿔주는 작업
- 리터칭 : 기존의 이미지를 다른 형태로 새롭게 변형/수정하는 작업
- 인터레이싱 : 이미지의 대략적인 모습을 먼저 보여준 다음 점차 자세한 모습을 보여주는 기법
- 안티엘리어싱 : 이미지의 경계선을 부드럽게 해주는 필터링 기술 -> 계단현상 방지

086 그래픽 데이터의 표현 방식
- 비트맵 : 점으로 이미지 표현
- 벡터 : 선으로 이미지 표현

087 그래픽 파일 형식
- BMP : 윈도우 표준 비트맵 파일 형식 -> 고해상도, 크기 큼
- JPEG(JPG) : 국제 표준 압축 방식
- GIF : 인터넷 표준 그래픽 형식
- PNG : 최상의 이미지를 표현하기 위해 제정한 그래픽 형식
- WMF : 윈도우 기본 벡터 파일 형식

088 시퀀싱/P2P
- 시퀀싱 : 컴퓨터를 이용해 음악을 제작, 녹음, 편집
- P2P : 개인 대 개인이 PC를 이용해 서로 데이터를 공유하는 방법

089 비디오 데이터
- MPEG : 동영상 압축 기술에 대한 국제 표준 규격 -> 오디오도 압축 가능
- AVI : 윈도우 표준 동영상 파일 형식
- ASF/WMV : 인터넷을 통해 오디오, 비디오 및 생방송 수신 등을 지원하는 멀티미디어 형식 -> 스트리밍을 위한 표준 기술 규격

090 MPEG 규격
- MPEG-1 : CD같은 고용량 매체에서 동영상 재생
- MPEF-2 : 1의 화질 개선 -> HDTV, 디지털 위성방송, DVD 등
- MPEF-4 : 양방향 멀티미디어 -> 화상 통신
- MPEF-7 : 멀티미디어 정보 검색 가능 -> 동영상, 데이터 검색 및 전자상거래 등
- MPEF-21 : 통합

091 멀티미디어 활용
- VOD : 사용자가 요구하는 정보를 원하는 시간에 볼 수 있도록 전송하는 멀티미디어 서비스 -> 사용자의 선택에 의해 정보를 서비스
- OTT : 영상 콘텐츠를 인터넷을 통해 제공하는 서비스 -> 스트리밍 기술 기반
- AR(증강현실) : 가상의 3차원 정보를 실시간으로 겹쳐 보여주는 새로운 멀티미디어 기술

092 저작권법

093 바이러스
- 컴퓨터의 정상적인 작동을 방해하기 위해 운영체제나 저장된 데이터에 손상을 입히는 프로그램
- 디스크의 부트 영역이나 프로그램 영역에 숨어 있음
- 소프트웨어 뿐만 아니라 하드웨어 성능에도 영향줌
- 연결형 바이러스 : 프로그램을 직접 감염시키지 않고 디렉터리 영역에 저장된 프로그램의 시작 위치를 바이러스의 시작 위치로 변경

094 정보 보안 요건
- 기밀성 : 시스템 내의 정보와 자원은 인가된 사용자에게만 접근 허용
- 무결성 : 시스템 내의 정보는 인가된 사용자만 수정 가능
- 가용성 : 인가받은 사용자는 언제라도 사용 가능
- 인증 : 정보를 보내오는 사람의 신원 확인
- 부인 방지 : 데이터 송/수신 증거 제공

095 보안 위협 유형
- 가로막기(Interruption) : 데이터의 정상적인 전달을 가로막아 흐름 방해
- 가로채기(Interception) : 송신된 데이터가 수신지까지 가는 도중 몰래 보거나 도청해 정보 유출
- 수정(Modification) : 전송된 데이터를 원래의 데이터가 아닌 다른 내용으로 바꿈
- 위조(Fabrication) : 마치 다른 송신자로부터 데이터가 송신된 것처럼 꾸밈

096 보안 위협의 구체적인 형태
- 분산 서비스 거부 공격(DDOS) : 여러 대의 장비 이용해 대량의 데이터를 특정 서버에 집중
- 스니핑 : 패킷 엿보면서 정보 가로채기
- 스푸핑 : 검증된 사람이 보낸것처럼 데이터를 변조해 접속 시도하는 침입 형태
- 피싱 : 거짓메일 발송해 가짜 웹사이트로 유인
- 키로거 : 키 입력 캐치 프로그램으로 개인정보 빼냄
- 백도어 : 시스템에 무단 접근하지 위한 비상구
- 크래킹 : 어떤 목적으로 타인 시스템에 불법 침입해 정보 파괴/변경
- 트로이 목마 : 프로그램 내 숨어있다가 프로그램 동작시 활성화
- 혹스 : 실제로는 악성코드 아니면서 겉으로는 악성코드인것처럼 가장해 행동

097 방화벽
- 보안이 필요한 네트워크의 통로를 단일화해 관리함으로써 외부의 불법 침입으로부터 내부의 정보 자산 보호 시스템
- 나가는 패킷은 그대로 통과, 들어오는 패킷은 체크 -> 외부로의 정보 유출 방지
- 역추적 기능 -> 침입자 역추적
- 내부로부터의 해킹은 못박음
- 운영시 네트워크 부하 증가, 트래픽이 게이트웨이로 집중됨

098 비밀키/공개키 암호화 기법
- 비밀키 : 동일한 키로 데이터 암호화/복호화
- 공개키 : 서로 다른 키로 데이터 암호화/복호화
