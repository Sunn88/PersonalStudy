1. 조인의 개념
- 조인 : 두 개의 테이블을 하나로 병합하는 것
- 서로 다른 테이블에 나뉘어 저장된 데이터를 하나의 기준으로 한 번에 조회하고 싶을 때
    - 데이터베이스는 나뉘어져 있는 데이터를 하나의 테이블로 재구성해야함
    - 조인 : 테이블 병합 연산
- 주로 두 테이블 간 PK와 FK의 연관성에 의해 성립함
    - 이런 연관성이 없더라도 논리적인 값들의 연관만으로 조인 성립 가능
- O(N^2)의 시간복잡도를 가짐 -> 많은 CPU 연산이 필요함
    - O(N^2) : 건수가 2배가 되면 연산량은 그 제곱인 4배가 됨
    - 조회 성능을 높이기 위해서는 조인이 덜 일어나도록 해야함 => 반정규화 수행

2. EQUI JOIN
- Equal(=) 즉 등식을 조건으로 사용할 때 발생하는 조인
- 칼럼 값이 정확하게 일치할 때 성립함
- WHERE절의 조건이 등식인 경우 발생

3. Non EQUI JOIN
- WHERE절의 조건이 등식이 아니라 부등식(BETWEEN, >, >=, <, <=)을 사용해서 범위를 나타낸 조건일 때 발생하는 조인
- 칼럼값이 일치하는 경우에 대한 조인 외에는 모두 해당됨
- 대부분의 DBMS가 수행할 수 있지만 설계상의 이유로 실행이 불가능할 떄도 있음

4. 3개 이상 TABLE JOIN
- 조회 조건이 복잡할 때, 조회하고자 하는 데이터가 3개 이상의 테이블에 나누어져 있을 때
- 두 개의 테이블에 대해 1번의 조인이 일어나므로 N개의 테이블에 대해서는 N-1만큼의 조인 발생
- 조회성능이 매우 하락할 수 있음 => 반정규화를 통해 조인이 필요한 테이블의 개수를 줄여 성능 최적화 필요

5. OUTER JOIN
- INNER JOIN : 조인 조건에 맞는 행들만 병합 -> 집합 연산에서의 일종의 교집합 개념
- OUTER JOIN : 조인 조건에 맞지 않는 행까지 좀 더 포괄적으로 병합함 -> 집합 연산에서의 합집합
    - LEFT OUTER JOIN, RIGHT OUTER JOIN, FULL OUTER JOIN
- 표준 조인에서 좀 더 명시적으로 OUTER JOIN을 지원함
    - 표준 조인 문법이 아닌 OUTER JOIN 문법은 DBMS에 따라 다름
- WHERE절에서 JOIN 칼럼 옆에 (+)를 붙이면 OUTER JOIN이 됨
    SELECT A.DNAME, B.ENAME
    FROM DEPT A, EMP B
    WHERE A.DEPTNO = B.DEPTNO(+);
    - (+)기호가 붙지 않은 쪽 테이블이 기준 테이블이 됨
    - A LEFT OUTER JOIN B와 같음
