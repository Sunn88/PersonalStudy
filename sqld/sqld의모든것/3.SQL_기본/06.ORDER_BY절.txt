1. ORDER BY
- SELECT문으로 칼럼을 조회할 때 ORDER BY절을 사용해 정렬을 수행할 수 있음
- 관계형 데이터베이스가 가지는 튜플의 무순서성에 따라 테이블에 삽입된 데이터는 특별한 순서 없이 디스크에 저장됨
    - 조회할 때도 특별히 ORDER BY절을 명시하지 않으면 임의의 순서로 출력이 됨
    - 순서가 중요하지 않을 때는 ORDER BY절을 생략해 조회성능을 높임
- SELECT문에서 가장 나중에 실행되는 절
    - 칼럼에 대한 별명이나 칼럼의 순서를 나타내는 정수를 사용할 수 없음
    - GROUP BY절보다 논리적 실행 순서가 나중이므로 집계함수를 사용할 수 있음
- ORDER BY절에 의해 정렬을 수행할 때 해당 칼럼에 NULL 값이 있을 경우, DBMS에 따라 NULL 취급하는 기본 방식이 다름
    - Oracle은 Null값을 최댓값으로 처리 -> NULLS FIRST 또는 NULLS LAST옵션을 추가해서 처리 방법 지정 가능
    - SQL Server는 최솟값으로 처리
- ORDER BY 칼럼 [ASC / DESC]
    - ASC : 오름차순 -> 기본값
    - DESC : 내림차순
