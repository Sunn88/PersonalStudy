- 데이터베이스에서 Null이라는 것은 속성값을 말할 때 값이 아직 입력되지 않은 상태에 대한 특별한 표현임
- 정의된 속성의 기본값으로 0이나 공백 등 특별한 어떤 값이 정의되지 않은 경우라면 모든 속성은 최초에 Null 값을 가짐

1. Null의 개념
- 속성값을 말할 때 Null은 아직 입력된 값이 없는 상태로서 아무런 값도 가지고 있지 않은 상태를 의미하는 특별한 표현
- 일반적으로 문자열 타입의 속성에 대해 Null 값과 'Null'이란 문자열값은 명백히 다름
- Bull은 공백과도 다름
- Number 타입 속성의 경우 값이 0인 것과 Null인 경우를 구분해야 함
- Null과의 연산
    - 단일행 연산 : Null 값과의 연산 수행결과는 Null이 됨
        - Null은 미정의된 값이므로 결과를 계산할 수 없음
        - ex) 100 + Null = Null
    - 다중행 연산 : Null 값인 행을 제외하고 연산을 수행함
        - Null은 미정의된 값으로 연산결과에 영향이 미치지 않도록 전체 연산 대상에서 제외함
        - ex) 10, 20, Null, 30의 평균 = (10+20+30)/3 = 20
