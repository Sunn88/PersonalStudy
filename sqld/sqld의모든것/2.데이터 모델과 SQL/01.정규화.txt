1. 정규화
- 이상현상(Anomaly) : 모델링된 데이터베이스의 테이블과 관계들이 데이터를 입력/수정/삭제하는 과정에서 의도치 않게 일관성이 깨지는 현상
    - 삽입 이상(Insertion Anomaly) : 테이블에 데이터를 삽입할 때 의도하지 않은 정보까지 삽입해야 하는 현상
    - 갱신 이상(Update Anomaly) : 중복 저장되어 있는 데이터 중 하나만 갱신하고 다른 하나를 갱신하지 않을 때 나타나는 데이터의 불일치 현상
    - 삭제 이상(Deletion Anomaly) : 테이블의 특정 데이터를 삭제할 때 의도하지 않은 정보까지 삭제되는 현상
- 정규화 : 이상현상을 방지하기 위해 데이터의 중복을 최소화하면서 테이블을 보다 잘 조직된 상태로 분해하는 과정
    - 데이터의 입력/수정/삭제 성능은 일반적으로 올라감
    - 조회성능은 조건에 따라 향상될 수도 있지만 대부분 많은 조인이 발생하면서 하락함
        - 조회성능을 올리기 위해 반정규화 수행
- 논리적 데이터 모델링의 엔터티 = 물리적 데이터 모델링의 테이블 = 관계형 데이터베이스의 릴레이션
- 정규화 절차
    - 제1정규화 : 릴레이션의 속성값이 모두 원자값(Atomic Value)만으로 구성되어야 함
        - 중복값 제거
        - 기본키 설정
    - 제2정규화 : 기본키가 2개 이상의 속성으로 이루어진 경우 부분 함수종속성 제거
    - 제3정규화 : 기본키를 제외한 칼럼 간의 종속성 제거, 이행 함수종속성 제거
    - BCNF : 기본키를 제외하고 후보키가 있는 경우 후보키가 기본키를 종속시키면 분해함

2. 제1정규형
- 모든 속성이 하나의 속성값만을 가지고 있으면서 유사한 속성이 반복되지 않는 상태
    -> 속성의 원자성(Atomicity)이 확보된 상태
- 제1정규형으로 만드는 것을 제1정규화 또는 1차 정규화라고 함
- 하나의 속성이 여러 속성값을 가질 때나 하나의 테이블에 유사한 속성이 반복될 때 이를 별도의 테이블로 분리함
    - 하나의 엔터티 내에서 주식별자에 대해 일반 속성은 1:1 관계를 가져야 함
    - 만약 1:M의 관계가 성립하면 1차 정규화 대상

3. 제2정규형
- 부분 함수종속성 : 주식별자가 두 개 이상으로 구성된 복합식별자인 경우, 일반 속성이 주식별자의 일부에만 종속성을 가질 때가 있음
    - 함수종속성 : 테이블의 속성이 함수의 대응 관계처럼 논리적 종속 관계를 가지는 성질
    - 완전 함수종속성 : 테이블의 모든 일반 속성이 주식별자에 완전하게 종속됨
- 제2정규형은 부분 함수종속성을 제거한 상태
- 제2정규형으로 만드는 것을 제2정규화 또는 2차 정규화라고 함
- 주식별자에 완전하게 종속적이지 않은 속성을 별도의 테이블로 분리함
    - 주식별자가 단일식별자읠 때는 제2정규화가 필요 없음 -> 그 자체가 이미 제2정규형

4. 제3정규형
- 이행 함수종속성 : 주식별자가 아닌 일반 속성 간에 함수종속성이 존재함
    - 집합 A, B, C가 있을 때 A->B이고 B->C이면 A->C가 성립한다는 것
- 제3정규형은 이행 함수종속성이 제거된 상태
- 제3정규형으로 만드는 것을 제3정규화 또는 3차 정규화라고 함
- 3차 정규화의 수행결과 분리된 엔터티들은 서로 비식별자 관계가 됨
