- DML(Data Manipulation Language) : 데이터 조작 언어
    - 데이터를 조작하는 작업을 수행하는 SQL 명령어
    - 이미 만들어져 있는 테이블에 레코드를 추가/변경.삭제하는 등 테이블의 데이터를 변경하는 작업 수행
    - DDL과 달리 자동커밋이 되지 않음
        - 명시적으로 커밋 실행 전 작업을 되돌리는 것이 가능(롤백)
        - 데이터베이스에 접근하는 다른 사용자가 데이터를 조회시 변경된 사항이 보이지 않음

1. INSERT
- 테이블에 레코드 추가
    INSERT INTO 테이블1 [(칼럼1 [[, 칼럼2] ... ])] VALUES (데이터1 [[, 데이터2] ... ]);

2. UPDATE
- 기존의 레코드 수정
- WHERE절을 통해 수정하고자 하는 레코드를 특정해 지정된 칼럼의 데이터 수정
    UPDATE 테이블1 SET 칼럼1 = 데이터1 [[, 칼럼2 = 데이터2] ... ][WHERE 조건식];
    - WHERE절이 없을 경우 모든 레코드에 적용됨

3. DELETE
- 기존의 레코드 삭제
- 특정 레코드를 명시하지 않으면 테이블의 모든 레코드가 삭제됨
    DELETE FROM 테이블1 [WHERE 조건식]
- DDL의 TRUNCATE는 작업취소(UNDO)를 위한 데이터를 생성하지 않아 속도가 빠르고, 디스크 공간을 릴리즈해 재사용이 가능함
- DELETE는 로그를 남기며 롤백이 가능함

4. MERGE
- 테이블 단위로 데이터 갱신
    - 두 테이블을 비교해 특정 조건에 맞는 레코드에 대해서는 UPDATE문 실행해 데이터 갱신
    - 그렇지 않은 레코드는 INSERT문 사용해 신규 추가
- 기존 테이블의 변경사항을 백업 테이블에 반영하거나 개발을 위해 변경 또는 추가된 레코드를 운영 환경에 적용할 때 사용
    MERGE INTO 테이블1
    USING 테이블2
    ON 조건식1
    [WHEN MATCHED THEN
        UPDATE SET 테이블1.칼럼1 = 테이블2.칼럼1 [[, 테이블1.칼럼2 = 테이블2.칼럼2] ... ][WHERE 조건식2]
        [DELETE [WHERE 조건식3]]]
    [WHEN NOT MATCHED THEN
        INSERT (테이블1.칼럼1 [[, 테이블1.칼럼2] ...]) VALUES(테이블2.칼럼1[[, 테이블2.칼럼2] ... ])]
        - 테이블1을 테이블2와 비교해 ON절의 조건식에 맞는 레코드의 경우 UPDATE문을 실행해 갱신
        - 그렇지 않은 레코드는 INSERT문을 실행해 신규 추가
        - 테이블2의 변경사항을 테이블1에 반영하는 것
- WHERE절을 사용해 특정 조건의 행에 대해 UPDATE를 수행하도록 할 수 있음
- DELETE절은 UPDATE가 된 행에 한해 조건에 따라 실행되므로 주의
