1. 관계의 개념
- 관계 : 엔터티와 엔터티 간에 맺고 있는 연관성
- ERD에서는 존재적 관계와 행위적 관계를 구분하지 않고 동일하게 표현함
    - 연관관계(실선) : 존재 자체로 연관성을 가짐
        - ex) Car는 Wheel을 멤버로 가지며 클래스 수준에서 사용함
    - 의존관계(점선) : 특정 행위를 할 때만 연관성을 가짐
        - ex) Car는 Driver를 특정 메소드에서만 사용함
- 존재적 관계
    - 존재 자체로 서로 연관성을 갖는 관계
    - ex) 사원과 부서와 관계
- 행위적 관계
    - 한 엔터티가 특정 행위나 이벤트를 일으킬 경우에 연관성이 발생하는 관계
    - ex) 고객과 주문의 관계

2. 표기법
- 관계명(Membership)
    - 관계의 이름을 나타냄
    - 존재적 관계와 행위적 관계를 기술함
- 관계차수(Cardinality)
    - 1:1(일대일), 1:M(일대다), M:N(다대다)과 같이 관계를 맺는 엔터티 인스턴스의 차수를 나타냄
- 관계선택사양(Optionality)
    - 필수적 관계인지 선택적 관계인지를 나타냄 -> Null 가능 여부를 의미함
    - 필수적 관계인 경우 Null 값을 가질 수 없음
    - 선택적 관계인 경우 Null 값을 가질 수 있음
        - 선택적 관계인 경우 까치발 기호에 O표시가 덧붙음
