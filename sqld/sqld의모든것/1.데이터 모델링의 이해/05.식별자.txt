1. 식별자의 개념
- 식별자 : 엔터티 인스턴스를 유일하게 구별할 수 있는 속성
- 해당 엔터티 인스턴스이 대표 속성이라고 할 수 있음
- 유사한 개념 : 키(key)

2. 주식별자
- 주식별자 : 해당 엔터티 인스턴스를 유일하게 구별해주는 식별자
- PK(Primary Key)에 해당하는 속성 -> Unique Key + Not Null
- 속성
    - 유일성 : 각 엔터티 인스턴스를 유일하게 구별할 수 있어야 함
    - 최소성 : 유일성을 보장하면서도 최소 개수의 속성이 되어야 함
    - 불변성 : 속성값이 최초 생성 시 부여된 값에서 변경되지 않고 유지되어야 함
    - 존재성 : 반드시 값을 가져야 하며 Null 값을 가질 수 없음

3. 식별자의 분류
- 대표성 여부
    - 주식별자(Primary Identifier)(Primary Key, PK)
        - 해당 엔터티 인스턴스를 유일하게 구별할 수 있는 식별자
        - 유일성, 최소성, 불변성, 존재성을 만족하는 식별자
        - 다른 엔터티와 참조관계를 연결할 수 있음
    - 보조식별자(Alternate Identifier)(Alternate Key, AK)
        - 해당 엔터티를 유일하게 구별할 수 있는 식별자이기는 하나 대표성을 가지지 못함
            -> 다른 엔터티와 참조관계를 연결할 수 없는 식별자
    - PK와 AK 모두 후보키(Candidate KEy)에 속함 : 유일성과 최소성을 만족함
- 스스로 생성 여부
    - 내부식별자(Internal Identifier) : 엔터티 내부에서 스스로 만들어지는 식별자
    - 외부식별자(Foreign Identifier) : 관계를 통해 달느 엔터티로부터 받아오는 식별자 -> FK(Foreign Key)
- 속성의 수
    - 단일식별자(Single Identifier) : 식별자를 구성하는 속성이 하나인 식별자
    - 복합식별자(Composite Identifier) : 식별자를 구성하는 속성이 둘 이상인 식별자
- 대체 여부
    - 본질식별자(Original Identifier) : 업무에 존재하는 본래의 식별자 = 원조식별자
    - 인조식별자(Surrogate Identifier) : 업무에 존재하지 않으나 원조식별자가 너무 복잡하게 구성되어 있어 인위적으로 만든 식별자 = 대리식별자

4. 식별자 관계, 비식별자 관계
- 식별자 관계
    - 엔티티 간의 강한 연결 관계 표현
    - 부모 엔터티의 식별자가 자식 엔터티의 주식별자 구성에 포함됨 -> ERD로 그릴 때 실선으로 표현
    - 부모 엔터티 인스턴스와 자식 엔터티 인스턴스가 같은 생명주기를 가질 때 식별자 관계로 표현하는 것이 적합함
- 비식별자 관계
    - 엔티티 간의 약한 연결 관계 표현
    - 부모 엔티티의 식별자가 자식 엔티티의 일반 속성이 됨 -> ERD로 그릴 때 점선으로 표현
    - 부모와 자식 엔터티 인스턴스가 서로 다른 생명주기를 가질 때 비식별자 관계가 적합함
    - 여러 개의 엔터티를 하나로 통합하면서 각각의 엔터티가 가지고 있는 여러 개별관계가 통합될 때도 비식별자 관계를 우선적으로 고려함
