1. 엔터티의 개념
- 엔터티 : 데이터베이스의 구성요소 중 독립적으로 식별 가능한 객체
- 데이터 모델링은 엔터티 정의부터 시작
- 속성 : 엔터티의 보다 상세히 자신을 표현하기 위한 하위요소

2. 엔터티의 특징
- 업무에서 필요로 하고 관리하고자 하는 정보여야 함
- 식별이 가능하도록 유일한 식별자를 가져야 함
- 영속적으로 존재하는 인스턴스가 두 개 이상인 집합을 이루어야 함
    - 인스턴스 : 엔터티는 구조를 정의하는데, 이 구조에 맞게 실제로 디스크에 저장된 데이터 1건을 인스턴스라 함
- 하위요소로 반드시 속성을 가져야 함
- 엔터티는 다른 엔티티와 한 개 이상의 관계를 가져야 함
    - 통계성 엔터티나 코드성 엔터티의 경우 관계 생략 가능
    
3. 엔터티의 분류
- 발생 시점/상속 관계에 따른 분류
    - 기본 엔터티 : 자신의 고유한 주식별자를 가지는 독립적으로 생성되는 엔터티
        - ex) 사원, 부서, 고객, 상품, 자재
    - 중심 엔터티 : 기본 엔터티로부터 주식별자를 상속받아 생성되며 업무의 중심 역할을 하는 엔터티
        - ex) 급여, 주문
    - 행위 엔터티 : 두 개 이상의 엔터티를 상속받아 생성되는 엔터티 -> 내용이 자주 변경되거나 데이터양이 계속 증가됨
        - ex) 급여내역, 주문내역
- 물리적 형태의 존재 여부에 따른 분류
    - 유형 엔터티 : 물리적 형태가 존재하는 엔터티
        - ex) 사원, 상품, 학생
    - 개념 엔터티 : 물리적 형태 없이 개념적으로 정의되는 엔터티
        - ex) 부서, 상품분류, 강의
    - 사건 엔터티 : 업무를 수행하면서 발생하는 행위나 이벤트를 나타내는 엔터티
        - ex) 주문, 대여, 수강
* 교차 엔터티 : M:N 관계를 해소하기 위해 만들어진 엔터티
* 엔터티 명명법
    - 엔터티 생성 의미대로 이름을 부여해 간결하고 명확하게 표현함
    - 가능하면 약어를 사용하지 않음
    - 영문의 경우 대문자를 사용, 단수명사를 사용
    - 협업의 업무 용어를 사용해 업무상의 의미를 분명히 함
    - 모든 엔터티의 이름은 서로 구별될 수 있도록 유일해야 함
    - 의미상 중복이 없도록 함
