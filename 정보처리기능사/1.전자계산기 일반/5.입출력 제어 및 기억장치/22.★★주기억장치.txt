* ROM과 RAM의 특징 비교
* SRAM과 DRAM 각각의 특징과 용도를 구분하기
    - SRAM은 재충전 시간이 필요 없으므로 비교적 빠르고, 캐시 메모리로 사용함
* 각 메모리의 특징과 용도를 서로 구분하기

1) 개요
- 주기억장치 : CPU가 직접 접근해 데이터를 처리할 수 있는 기억장치(Memory)
- 현재 수행되는 프로그램과 데이터를 저장하고 주소(Address)에 의해 저장된 위치를 구분함

2) ROM(Read Only Memory, 롬)
- 기억된 내용을 읽을 수만 있는 기억장치
- 일반적으로 쓰기는 불가능
- 전원이 꺼져도 기억된 내용이 지워지지 않는 비휘발성 메모리
- 주로 기본 입출력 시스템(BIOS), 글자 폰트, 자가진단 프로그램(POST, Power On Self Test) 등이 저장됨
- 종류와 특징 : 기억된 내용의 수정 가능 여부 및 데이터 기록 방법에 따라
    - Mask ROM : 제조 과정에서 미리 내용을 기억시킨 ROM -> 사용자가 임의로 수정할 수 없음
    - PROM(Programmablr ROM) : 특수 프로그램을 이용해 한 번만 기록할 수 있으며, 이후엔 읽기만 가능
    - EPROM(Erasable PROM) : 자외선을 이용해 기록된 내용을 여러 번 수정하거나 새로운 내용을 기록할 수 있음
    - EEPROM(Electrically EPROM) : 전기적인 방법을 이용해 기록된 내용을 여러 번 수정하거나 새로운 내용을 기록할 수 있음

3) RAM(Random Access Memory, 램)
- 자유롭게 읽고 쓸 수 있는 기억장치
- RAM에는 현재 사용중인 프로그램이나 데이터가 저장되어 있음
- 전원이 꺼지면 기억된 내용이 모두 사라지는 휘발성 메모리
- 일반적인 '주기억장치'를 의미함
- SRAM(Static RAM)/DRAM(Dynamic RAM)의 특징 : 재충전 여부에 따라
    구분            동적 램(DRAM)           정적 램(SRAM)
    구성소자        콘댄서                  플립플롭
    특징과          주기적인 재충전 필요    전원이 공급되는 동안엔 기억 내용 유지됨
    전력소모        적음                    많음
    접근속도        느림                    빠름
    집적도(밀도)    높음                    낮음
    가격            저가                    고가
    용도            일반적인 주기억장치     캐시 메모리

4) 기타 메모리
- 캐시 메모리(Cache Memory)
    - CPU와 주기억장치 사이에 위치
    - 컴퓨터 처리 속도 향상
    - 접근 속도가 빠른 SRAM 사용
    - 일반 메모리보다 비쌈
    - 캐시 적중률(Cache Hit Ratio)이 높을수록 시스템의 전체적인 속도가 향상됨
- 가상 메모리(Virtual Memory)
    - 보조기억장치(하드디스크)의 일부를 주기억장치처럼 사용하는 메모리 기법
    - 주기억장치보다 큰 프로그램을 불러와 실행해야 할 때 사용
- 버퍼 메모리(Buffer Memory)
    - 두 개의 장치가 데이터를 주고받을 때 두 장치 간의 속도 차이를 해결
    - 중간에 데이터를 임시로 저장해 두는 공간
    - 키보드 버퍼, 프린터 버퍼 등
    - 캐시 메모리도 일종의 버퍼
- 연관 메모리(Associative Memory)
    - 주소를 참조해 데이터를 읽어오는 방식이 아닌, 저장된 내용의 일부를 이용해 기억장치에 접근해 데이터를 읽어옴
    - 가상 메모리나 캐시 메모리의 주소 변환 테이블에서 사용됨
    - 구성 요소 : 검색 자료(데이터), 레지스터, 키(마스크) 레지스터, 매치 레지스터(일치 지시기)
- 플래시 메모리(Flash Memory)
    - EEPROM의 일종 -> 비휘발성 메모리
    - MP3 플레이어, 개인용 정보단말기, 휴대전화, 디지털 카메라 등에 널리 사용됨
