* 10진수를 BCD 코드로 변환하는 방법 알아두기
* 3초과 코드는 BCD 코드에 3을 초과(더해)해 만든코드
    - 자기 보수 코드이며 비가중치 코드
* 2진 코드와 그레이 코드간 변경 방법, 그레이 코드의 사용 용도
* 패리키 검사 코드는 1Bit를 추가해 1의 개수를 인위적으로 홀수 혹은 짝수로 만들어 데이터를 전송하는 것
    - 보내는 쪽에서 홀수개 보냈는데 받은 쪽이 짝수면 에러 발생한 것
    - 개념만 간단히
* 해밍 코드는 오류 검출과 교정이 가능한 유일한 코드임을 기억하기
* 가중치 코드와 비가중치 코드의 종류를 구분해 정리하기
* 오류 검출 코드에는 해밍 코드와 패리티 검사 코드가 있음 기억하기

1) BCD 코드
- 10진수 1자리의 수를 2진수 4Bit로 표현함
- 4Bit의 2진수 각 Bit가 8, 4, 2, 1의 자릿값을 가짐 -> 8421코드
- 대표적인 가중치 코드
    - 가중치 코드 : 2진수 각 자리가 고유한 값을 갖는 코드
    - 비가중치 코드 : 각 자리가 고유한 값을 갖지 않는 코드
- 문자 코드인 BCD에서 Zone 부분을 생략한 형태
- 10진수 입출력이 간편함

2) Excess-3 코드(3 초과 코드)
- BCD + 3 : BCD에 3(0011)을 더해 만든 코드
- 대표적인 자기 보수 코드, 비가중치 코드
    - 자기 보수 코드 : 2진수로 된 코드의 1을 0으로, 0을 1로 모두 바꿈
        - 해당 코드의 10진수 값에 대해 9의 보수를 얻는 코드

3) Gray 코드
- BCD 코드의 인접하는 비트를 XOR 연산해 만든 코드
- 입출력장치 코드로 유용하게 사용됨
- 1Bit만 변화시켜 다음 수치로 증가시키기 때문에 하드웨어적 오류가 적어 하드웨어 동작 제어에 적합함
- 연속되는 2개의 숫자를 표현한 코드에서 한 비트를 변경하면 새로운 코드가 됨
    - 아날로그-디지털(A/D) 변환, 데이터 전송에 주로 사용됨
- 2진수를 Gray Code로 변환하는 방법 : '그'자 생각하기
    - 첫 번째 그레이 비트는 2진수 비트를 그대로 내려씀
    - 2번째 Gray Bit부터는 변겨할 2진수의 해당 번째 비트와 그 왼쪽의 비트를 XOR 연산해 씀
- Gray Code를 2진수로 변환하는 방법 : '2'자 생각하기
    - 첫 번쨰 2진수 비트는 그레이 비트를 그대로 내려씀
    - 2번쨰 2진수 비트부터는 왼쪽에 구해 놓은 2진수 비트와 변경할 그레이 코드의 해당 번째 비트를 XOR 연산해 씀

4) 패리티 검사 코드
- 코드의 오류를 검사하기 위해 데이터 비트 외에 1Bit의 패리티 체크 비트를 추가하는 것
    - 1Bit의 오류만 검출할 수 있음
- 1의 개수에 따라 짝수(Even, 우수) 패리티와 홀수(Odd, 기수) 패리티 방법이 있음
- Odd Parity : 코드에서 1인 비트의 수가 홀수가 되도록 0이나 1을 추가함
- Even Parity : 코드에서 1인 비트의 수가 짝수가 되도록 0이나 1을 추가함

5) 해밍 코드
- 오류를 스스로 검출해 교정이 가능한 코드
- 2Bit의 오류를 검출할 수 있고, 1Bit의 오류를 교정할 수 있음
- 해밍 코드 중 1, 2, 4, 8, ... 2^n 번째는 오류 검출을 위한 패리티 비트임

6) 코드의 분류
- 가중치 코드(Weight Code) : BCD(8421), 2421, 84-2-1, Biquinary(5043210), 51111, Ring-Counter(9876543210)
- 비가중치 코드(Non-Weight Code) : 3초과(Excess-3), Gray, Jonson, 2-out-of-5, 3-out-of-5
- 자기 보수 코드(Self-Complement Code) : Excess03, 2421, 51111, 84-2-1
- 오류 검출용 코드 : 해밍 코드, 패리티 검사 코드, Biquinary, Ring-Counter, 2-out-of-5, 3-out-of-5
