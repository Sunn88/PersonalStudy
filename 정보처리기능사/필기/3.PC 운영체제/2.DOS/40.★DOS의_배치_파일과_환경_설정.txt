* ★환경 설정 파일의 특징 기억하기
    - 환경 설정 파일에서 사용되는 명령어 중 LASTDRIVE, FILES, BREAK 등 중심으로 명령어 기능 파악하기

1) 배치 파일(Batch File)
- 자주 사용되는 일련의 명령어들을 한 그룹으로 묶은 것 -> 일괄 처리 파일
- 일반적으로 확장자는 'BAT'
- 배치 파일은 편집기를 사용해 작성할 수 있음
- 부팅 시 자동 실행되는 Autoexec.bat와 사용자가 작성해 실행할 수 있는 일반 배치 파일로 나눌 수 있음
    - Autoexec.bat : 배치 파일의 특수한 형태 -> 컴퓨터 부팅 시 자동으로 실행되는 파일
        - 일반 배치 파일에서 사용하는 각종 명령어를 모두 사용할 수 있음
        - Autoexec.bat는 루트 디렉터리에 존재해야만 부팅 시 자동으로 주기억장치에 적재됨
            - 루트 디렉터리 : 가장 위쪽에 있는 디렉터리 -> 최상위 디렉터리 -> 일반적으로 'C:\'로 표시
        - COPY CON 명령으로 파일을 작성 및 수정할 수 있음
- 배치 파일 전용 명령어
    - ECHO [ON/OFF] : 명령어의 화면 표시 여부 설정
    - PAUSE : 배치 파일의 실행을 일시 정지시킴
    - GOTO : 특정 레이블로 이동
    - CALL : 다른 배치 파일 호출
    - REM : 주석(설명문)을 달고자 할 때 사용
    - FOR : 실행 반복
    - IF : 조건에 따라 수행이 다를 떄 사용

2) 환경 설정 파일
- DOS로 부팅될 떄 자신에게 필요한 시스템 환경을 설정해 주는 파일 -> CONFIG.SYS
- Config.sys 파일은 반드시 루트 디렉터리에 존재해야 실행됨
- 디스크의 동작 속도를 향상시켜 주는 버퍼/캐시를 설정할 수 있음
- 키보드, 마우스, 기타 주변장치 활용 방법을 설정할 수 있음
- COPY CON 명령으로 파일을 작성할 수 있음
- 주요 환경 설정 명령어
    - LASTDRIVE : 드라이브의 최대 개수 지정
        - 마지막 드라이브가 무엇인지 지정하는 것 -> 드라이브의 최대 개수가 지정됨
    - FILES : 동시에 열 수 있는 파일의 수 지정
    - BREAK : 프로그램 실행을 중지하는 Ctrl+C(Ctrl+Break)의 작동 여부 설정
    - PAUSE :  : CONFIG.SYS 파일의 실행을 일시 중지
    - DEVICE : 마우스, 스캐너와 같은 장치 사용 시 장치 드라이버 연결
    - BUFFERS : 시스템이 사용 가능한 버퍼 수 지정
    - LOAD HIGH : 램 상주 명령을 UMB 영역에 상주시켜 기본 메모리를 절약할 수 있게 함
        - UMB(Uper Memory Byte, 상위 메모리 영역) : 메모리 영역 중 640~1024KB 영역
            - 램 상주 프로그램이나 장치 구동 프로그램을 적재시키는 곳
    - SHELL : COMMAND.COM 이외의 명령 처리기를 사용할 떄 지정
    - INSTALL : 램 상주용 프로그램을 메모리에 적재
    - RAMDRIVE : 램의 일부를 드라이브처럼 할당해 고속 처리가 가능하도록 해줌
    - MEM : 시스템이 현재 사용하고 있는 메모리와 사용 가능한 메모리의 크기 표시
    - HIMEM : DOS가 연속 확장 메모리를 사용할 수 있도록 해줌
    - EMM386 : 연장 메모리(EMS)를 사용할 수 있도록 해줌
        - 연장 메모리 : 메모리 영역 중 1088KB 이상의 영역 -> 램 드라이브나 디스크 캐시 등에 사용
