* ★운영체제의 정의를 묻는 문제가 자주 출제됨
    - 운영체제의 정의를 제시하고 답을 찾는 문제
    - 그림에서 운영체제가 위치할 곳
    - 운영체제의 정의 기억하기, 컴퓨터 시스템 계층 구조에서 운영체제의 위치 파악
* ★운영체제의 성능 평가요인에 대한 문제가 자주 출제됨
    - 운영체제의 성능 평가 요인과 반환 시간의 의미 파악
* 운영체제의 종류 구분하기

1) 정의
- 운영체제(OS; Operating System)
    - 컴퓨터 하드웨어와 일반 컴퓨터 사용자 또는 컴퓨터에서 실행되는 응용 프로그램의 중간에 위치
    - 사용자들이 보다 쉽고 간편하게 컴퓨터 시스템을 이용할 수 있도록 제어하는 시스템 소프트웨어의 일종
    - 하드웨어 -> 운영체제 -> 유틸리티 -> 응용 프로그램 -> 사용자

2) 목적
- 처리 능력(Throughput) 향상 : 일정 기간 내에 시스템이 처리하는 일의 양
- 반환 시간(Turn Around Time) 단축 : 시스템에 작업을 제시하고 나서부터 결과를 받을 때까지의 경과 시간
- 사용 가능도(Availability) 향상 : 시스템을 사용할 필요가 있을 때 즉시 사용 가능한 정도
- 신뢰도(Reliability) 향상 : 시스템이 주어진 문제를 정확하게 해결하는 정도

3) 기능
- 사용자와 컴퓨터 간의 인터페이스 제공
- 시스템의 효율적인 운영 및 관리
- 자원 스케줄링 및 주변장치 관리
    - 자원 : 시스템에서 사용할 수 있는 CPU, 주기억장치, 보조기억장치, 프린터, 파일 및 정보 등
    - 스케줄링 : 어떤 자원을 누가, 언제, 어떤 방식으로 사용할지 결정
- 사용자 간의 데이터 호환
- 각종 하드웨어와 네트워크 관리
- 데이터 공유 및 관리
- 초기 설정 기능 및 이식성 기능
- 시스템의 오류 검사 및 복구

4) 종류
- 단일 작업 처리 시스템(Single Tasking System)
    - 컴퓨터 시스템을 한 개의 작업이 독점해 사용하는 방식
    - ex) DOS
- 다중 작업 처리 시스템(Multi-Tasking System)
    - 여러 개의 프로그램을 열어두고 다양한 작업을 동시에 진행하는 방식
    - ex) Windows, UNIX, LINUX
