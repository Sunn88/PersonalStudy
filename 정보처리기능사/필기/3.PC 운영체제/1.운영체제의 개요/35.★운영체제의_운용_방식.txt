* 운영체제를 운용하는 방식의 종류와 함께 각각 구분할 수 있도록 특징 파악
    - 일괄 처리 시스템, 시분할 시스템, 실시간 처리 시스템은 영어로 묻기도

1) 일괄 처리 시스템(Batch Processing System)
- 초기의 컴퓨터 시스템에서 사용된 형태
- 일정량 또는 일정 기간 동안 데이터를 모아 한꺼번에 처리하는 방식
- 컴퓨터 시스템을 효율적으로 사용할 수 있음
- 급여 계산, 지불 계산, 연말 결산 등의 업무에 사용됨

2) 다중 프로그래밍 시스템(Multi-Programming System)
- 하나의 CPU와 주기억장치를 이용해 여러 개의 프로그램을 동시에 처리하는 방식
    - 프로그램이 여러 개(다중)
- 하나의 주기억장치에 두 개 이상의 프로그램을 기억시켜 놓고, 하나의 CPU와 대화하면서 동시에 처리함
- CPU의 사용률과 처리량이 증가함
    - 유휴 시간(Idle Time) : 컴퓨터 시스템이 정상적으로 사용 가능한 상태이나 계산 처리 시간과 데이터 입출력 처리 시간 등의 차이로 실제적인 작업이 없는 시간

3) 시분할 시스템(Time Sharing System)
- 단말장치 사용자가 일정한 시간 간격(Time Slice)동안 CPU 를 사용함으로 단독으로CPU를 사용하는 것과 같은 효과를 가지는 시스팀
    - 라운드 로빈(Round Robin) 방식
- 한 시스템을 여러 명의 사용자가 공유해 동시에 작업 수행
- 시스템의 전체 효율은 좋아지나 개인별 사용자 입장에서는 반응 속도가 느려질 수 있음
- 목적 : 각 작업에 대한 응답 시간을 최소한으로 줄이는 것

4) 다중 처리 시스템(Multi-Processing System)
- 컴퓨터 한 대에 두 개 이상의 CPU를 설치해 병렬 처리하는 것
- 한 시스템에서 여러 개의 처리 과정을 동시에 수행하는 것
- 하나의 CPU가 고장나더라도 다른 CPU를 이용해 업무를 처리할 수 있으므로 시스템의 신뢰성과 안저성이 높음
- 여러 개의 프로그램을 동시에 실행해 CPU와 입출력장치를 최대한 활용하게 됨

5) 실시간 처리 시스템(Real Time Processing System)
- 데이터 발생 즉시, 또는 데이터 처리 요구가 있는 즉시 처리해 결과를 산출하는 방식
- 처리 시간이 단축되고, 처리 비용이 절감됨
- 우주선 운행, 레이더 추적기, 핵물리학 실험 및 데이터 수집, 전화 교환장치 제어, 은행 온라인 업무 등
- 시간에 제한을 두고 신속하게 수행되어야 하는 작업에 사용됨

6) 분산 처리 시스템(Distributed Processing System)
- 지역적으로 분산된 여러 개의 컴퓨터(프로세서)를 통신 회선으로 연결해 하나의 작업을 처리하는 방식
- 각 단말장치나 컴퓨터 시스템은 고유의 운영체제와 CPU, 메모리를 가짐

7) 운영체제의 발달 과정
- 일괄->다중프로그램->시분할->다중처리->실시간->범용->분산
