1-1장
운영체제 운용 기법 발달 과정 : 일처 다프 시분 다처 실처 범용 분산
일괄 처리 -> 다중 프로그래밍 -> 시분할 -> 다중 처리 -> 실시간 처리 -> 범용 -> 분산 처리

운영체제의 목적
처리 능력/사용 가능도/ 신뢰도 향상, 반환 시간 단축

기억장치 관리 전략
반입(언제), 배치(위치), 교체

교체 전략
OPT(오랫동안 안 쓸 것), FIFO(가장 오래 있던 것), LRU(가장 덜 참조)
LFU(가장 적게 사용), NUR(최근에 사용 안 함), MRU(가장 많이 사용)

비선점 스케줄링
FIFO, SJF, HRN, 우선 순위

선점 스케줄링
SRT, RR, 다단계 큐, 다단계 피드백 큐

데이터베이스 설계 순서
요구 조건 분석 - 개념적 설계 - 논리적 설계 - 물리적 설계 - 구현

스키마
외부 스키마(사용자 입장 논리적 구조)
개념 스키마(전체적인 논리적 구조)
내부 스키마(저장장치 입장 논리적 구조)

데이터베이스 사용자
데이터베이스 관리자(DBA), 응용 프로그래머, 일반 사용자(End User)

DBMS
필수 기능 : 정의, 조작, 제어
종류 : 계층형(H), 망형(N), 관계형(R), 비관계형

데이터 모델 구성 요소 : 개체(사각형), 속성(타원), 관계(다이아몬드)

관계형 데이터베이스
구성 : 릴레이션 스키마(구조), 릴레이션 인스턴스(실제 값들)
튜플 : 행, 레코드, 카디널리티, 기수
속성 : 열, 데이터 필드, 디그리, 차수
도메인 : 속성 범위

무결성
개체 무결성(기본키), 도메인 무결성(속성 값), 참조 무결성(외래키)
사용자 정의 무결성, NULl 무결성, 고유 무결성, 키 무결성, 관계 무결성

정규화 과정 : 도부이결다조
도메인이 원자값, 부분적 함수 종속 제거, 이행적 함수 종속 제거,
결정자이면서 후보키가 아닌 것 제거, 다치 종속 제거, 조인 종속성 이용
비정규 릴레이션-1NF-2NF-3NF-BCNF-4NF-5NF

이상의 종류 : 삽입, 삭제, 갱신

고급 데이터베이스 기능 : 뷰, 인덱스, 시스템 카탈로그(자료 사전, 메타 데이터), 
트랜잭션(원자성, 일관성, 독립성, 영속성)

OSI 참조모델 : 물데네 전세표응

전송(세그먼트, TCP, UDP, DCCP), 세션(메시지), 표현(메시지), 응용(메시지, HTTP, SMTP, FTP 등)

네트워크 관련 장비

브리지(LAN 연결, 데이터링크 계층), 스위치(더 큰 LAN, L2, L3, L4, L7)
라우터(데이터 전송 최적 경로), 게이트웨이(전 계층 연결)

프로토콜 : 컴퓨터 간 정보 교환 통신 규약
구문(데이터 규정), 의미(제어 정보), 시간
HTTP(웹 HTML), TELNET(멀리 떨어진 컴퓨터), FTP(원격 파일 전송)
SMTP(전자 우편 전송), IMAP(전자 우편 액세스)
SNMP(네트워크 관리), SSH(원격 접속, 파일 복사),

TCP : 신뢰성 있는 연결형 서비스

3-1장
테스트 : 확인, 검증
애플리케이션 테스트의 기본 원리
완벽한 테스트 불가능, 결함 집중(파레토 법칙), 살충제 패러독스, 테스팅 정황 의존, 
오류-부재의 궤변, 테스트와 위험은 반비례, 테스트의 점진적 확대, 테스트의 별도 팀 수행

정적 테스트 : 워크스루, 인스펙션, 코드 검사 등
동적 테스트 : 블랙박스/화이트백스 테스트
테스트 기반 테스트 : 명세(동등 분할, 경계값 분석)/구조(구문,결정,조건)/경험(에러, 체크리스트) 기반
시각에 따른 테스트 : 검증(개발자 시각), 확인(사용자 시각)
목적에 따른 테스트 : 회복(실패 후 복구), 안전(시스템 보호), 강도(과부하),
                    성능(응답시간, 처리량, 효율성), 구조(논리 경로, 복잡도),
                    회귀(새로운 결함), 병행(기존과 동일 데이터)

화이트박스 : 기초 경로 검사, 제어 구조 검사(조건, 루프, 데이터 흐름 검사)
기준 : 문장(모든 구문), 분기(모든 조건문), 조건(조건식 결과), 분기/조건 검증 기준

블랙박스 : 동치 분할(입력 자료), 경계값, 원인-효과 그래프(효용성 높은 테스트 케이스),
            오류 예측(과거 경험), 비교(여러 버전 프로그램) 검사

개발 단계 테스트 : 단통시인
단위 테스트 : 구조 기반(제어 흐름, 조건 결정), 명세 기반(동등 분할, 경계값 분석)
통합 테스트 : 비점진적(빅뱅 통합 테스트), 점진적(하향식 통합(스텁), 상향식 통합(클러스터, 드라이버), 혼합식 통합 테스트)
시스템 테스트 : 기능적(명세서 기반의 블랙박스), 비기능적(화이트박스)
인수 테스트 : 사용자 인수, 운영상 인수, 계약 인수, 규정 인수, 알파, 베타 테스트

테스트 프로세스 : 계획 -> 분석/디자인 -> 케이스/시나리오 작성 -> 수행 -> 평가/리포팅 -> 결함 추적 관리
테스트 계획서(수행 계획 문서), 테스트 케이스(테스트 항목 명세서)
테스트 시나리오(케이스 동작 순서), 테스트 결과서(결과 비교분석)
테스트 오라클(결과 판단 위해 참 값 대입) : 참, 샘플링, 추정, 일관성 검사 오라클

결함 관리 프로세스 : 발견 -> 등록 -> 분석 -> 확정 -> 할당 -> 조치 -> 조치 검토 및 승인

휴먼 에러 : 사람 실수로 발생
테스트 자동화 도구 : 정적 분석 도구, 테스트 실행 도구, 성능 테스트 도구, 테스트 통제 도구, 테스트 하네스 도구가
