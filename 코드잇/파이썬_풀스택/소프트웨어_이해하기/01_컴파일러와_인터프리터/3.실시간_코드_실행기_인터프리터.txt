컴파일러는 프로그램 전체를 한 번에 번역한 후 완성된 컴퓨터 언어로 만들어줌
- 완성된 머신코드를 사용자 컴퓨터로 전송

인터프리터는 한 줄씩 즉시 실행해줌
- 컴파일 과정이 필요없음
- 개발 속도 상승
- 프로그래밍 코드 자체를 전송
    - 사용자 컴퓨터에서 인터프리터가 필요함
    - 컴파일러에 비해 실행 속도 감소
    - 코드가 유출 될 위험성 상승
- 코드가 더 짧은 고수준 언어에 주로 사용
    - 프로그램 용량이 더 작음


비교
- 컴파일러
    - 개발 편의성 : 코드 수정 후 실행시 컴파일 다시 해야함
    - 실행 속도 : 빠름
    - 보안 : 프로그램의 코드가 유출되지 않음
    - 파일 용량 : 프로그램의 실행 파일 전체를 전송해야 하므로 용량이 큼
    - 프로그래밍 언어 : C, C++처럼 비교적 저수준에 가까운 언어
        - 갸발 편의성은 떨어지지만 실행 속도는 빠름
- 인터프리터
    - 개발 편의성 : 코드를 수정하고 즉시 실행 가능
    - 실행 속도 : 느림
    - 보안 : 코드가 유출될 수 있음
    - 파일 용량 : 코드만 전송하면 실행되므로 용량이 작음
    - 프로그래밍 언어 : Python, Ruby처럼 비교적 고수준에 가까운 언어
        - 개발 편의성이 높지만 실행 속도는 느림
