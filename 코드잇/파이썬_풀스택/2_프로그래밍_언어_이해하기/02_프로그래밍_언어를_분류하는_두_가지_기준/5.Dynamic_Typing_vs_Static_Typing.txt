Dynamic typing : 자료형을 미리 정해두지 않고 실행될 떄 정함
- 간결하고 편리함
- 어려운 내용이 적어 코드의 내용, 로직이 잘 이해됨
- 작은 사이즈의 프로그램에 적합
- 컴퓨터적 구조가 생략되어 실행속도가 다소 느림
- ex) 파이썬, 루비, JS, php 등

Static typing : 자료형을 실행 전 미리 정해둠
- 명확하고 정교함
- 컴퓨터적 구조가 눈에 잘 들어옴
- 비교적 제약과 규칙이 많음 -> 실수 방지
- 많은 사람이 협업해야 하는 프로젝트에 적합
- ex) C, C++, Java, Kotlin 등

장단점
- Dynamic typing	Static typing
  코딩 속도가 빠르다.	코딩 속도가 느리다.
  실행 속도가 느리다.	실행 속도가 빠르다.
  코드의 내용이 눈에 잘 들어온다.	코드의 구조가 눈에 잘 들어온다.
  배우기 쉽다.	배우기 어렵다.
  혼자서 빠르게 하는, 작고 단순한 프로젝트에 적합	여러 사람이 협업하는, 크고 복잡한 프로젝트에 적합
  
총정리
Dynamic typing 언어는 간편함과 편리함을 추구하며, 컴퓨터적 구조를 생략하기 때문에 코드의 내용(로직)이 눈에 잘 들어오고 사람이 코드를 이해하기 편해서 학습하기가 쉬움
다만 컴퓨터적 구조를 생략하기 때문에 실행 속도에 민감한 곳이나 성능이 좋지 않은 환경에서는 활약하기 힘듦
Static typing 언어는 안정성과 정교함을 추구하며, 컴퓨터적 구조를 코드에 적어주기 때문에 코드의 구조가 눈에 잘 들어오지만 코드가 복잡해져서 배우기 어려움
다만 코드의 컴퓨터적 구조 덕분에 컴퓨터가 고민을 덜 하게 되고, 실행 속도가 중요한 곳에서 활약할 수 있음
이런 특징을 살려 Static typing 언어는 임베디드, 실시간 투자, 인공 지능 분야에서 활약하고 있음
