1.웹 프로그래밍의 기본
- JSP 등장 이전 : 아파치 같은 웹 서버를 이용해 웹 페이지들을 브라우저에 표시 -> 정적(static) 웹 프로그래밍
    - 웹 서버에 미리 보여줄 HTML 페이지, CSS, 이미지, 자바스크립트 파일을 저장해 놓고 요청시 그대로 전달하는 방식
    - 페이지가 변경되지 않는 한 고정된 웹 페이지를 보게 됨
    - 요즘은 잘 사용되지 않음
    - 정적 웹 프로그래밍 구성 요소는 JSP에서도 여전히 화면 디자인과 기능 처리를 담당함
- 정적 웹 프로그래밍 구성 요소
    - 웹 서버 : 각 클라이언트에게 서비스를 제공하는 컴퓨터
    - 클라이언트 : 네트워크로 서버에 접속한 수 서버로부터 서비스를 제공받는 컴퓨터
    - HTTP 프로토콜
        - Hyper Text Transfer Protocol
        - www 서비스를 제공하는 통신 규약
        - 웹 서버와 클라이언트는 이 프로토콜을 이용해 정보를 주고받음
    - HTML
        - Hyper Text Markup Language
        - www 서비스를 제공하기 위한 표준 언어
    - 자바스크립트 : HTML 웹 페이지의 여러 동적인 기능을 제 공하는 스크립트 언어
    - CSS(Style Sheet) : HTML 문서에서 세부적인 페이지의 디자인에 관련된 여러 가지 기능 제공

2. 정적 웹 프로그래밍
- 사용자에게 화면 디자인 같은 고정된 정보만 제공함
- 정보 수정 시 관리자가 직접 HTML 소스를 수정해 사용자에게 정보를 제공함
- 초기에는 가능했으나 사용자들에게 제공하는 정보의 변경 속도가 급속도로 빨라지는 현실에서는 사실상 불가능

3. 동적 웹 프로그래밍
- CGI : Common Gate Interface, 공용 게이트웨이 인터페이스
    - 처음 동적인 방식으로 프로그램 제공하는 기능
    - 특징
        - 초기 웹 프로그램에서 사용하는 방식
        - 프로세스 방식으로 실행됨
        - 서버의 부하가 심함
    - 웹 애플리케이션 서버에서 실행되는 기능들이 모두 프로세스 방식으로 실행됨
    - 같은 기능을 수행하더라도 각 경우에 대해 처음부터 메모리에 기능을 로드해 수행해야 함
    - 인터넷 환경이 발전하고 사용자 수가 급격히 늘어나 동시에 처리해야 할 일이 많은 상황에서는 메모리 과부하 발생
- CGI의 문제점을 개선해 나온 것이 JSP, ASP, PHP와 같은 동적 웹 프로그래밍 기술

4. JSP 프로그램의 특징
- 기본적으로 스레드 방식으로 실행 -> CGI 방식보다 효율적으로 기능 수행
- 프로그램의 수행 속도를 떨어뜨리는 요인 중 하나가 프로그램을 외부 장치에서 메모리로 로드하는 것
- 스레드로 수행하면 이 과정을 거치지 않아도 되니 기존 CGI 프로그램보다 훨씬 빠르게 동작함
- 다수의 사용자가 접속해서 기능을 수행해야 하는 환경에서는 스레드 방식 프로그램이 유리함
- 특징
    - 프로세스 방식이 아닌 스레드 방식으로 실행
    - 클라이언트의 요구를 처리하는 기능은 최초 한 번만 메모리에 로드됨
    - 클라이언트가 동일한 기능을 요구하면 기존에 사용한 기능을 재사용함
