1) IPv4(Internet Protocol version 4)
- IPv4의 개념
    - 32비트 길이의 IP 주소
    - 주소의 각 부분을 8비트씩 4개로 나눠서 10진수로 표현함
    - IP 주소=네트워크 주소(Netid)+호스트 주소(Hostid)
- IPv4 주소 체계
    - 클래스 A : 국가나 대형 통신망에서 사용
    - 클래스 B : 중대형 통신망에서 사용
    - 클래스 C : 소규모 통신망에서 사용
    - 클래스 D : 멀티캐스트용으로 사용
    - 클래스 E : 실험용으로 사용
- 서브넷 마스크(Subnet Mask)
    - 네트워크를 작은 내부 네트워크로 분리해 효율적으로 네트워크를 관리하기 위한 수단
    - 서브넷 마스크는 32bit의 값으로 IP 주소를 네트워크와 호스트 IP주소로 구분하는 역할을 함
    - 네트워크 ID에 해당하는 모든 비트를 1ㄹ로 설정해 호스트 ID에 해당하는 모든 비트를 0으로 설정함
    - CIDR 표기 형식 : 10진수의 IP/네트워크 ID의 1비트의 개수
2) IPv6
- IPv6의 개념
    - IPv4의 주소 부족 문제를 해결하기 위해 개발됨
    - 128비트 길이의 IP 주소
    - 16비트씩 8개의 필드로 분리 표기됨
    - 패킷 헤더는 40바이트의 고정된 길이를 가짐
- IPv6의 장점
    - 인증 및 보안 기능을 포함하고 있어 IPv4보다 보안성이 강화됨
    - IPv6 확장 헤더를 통해 네트워크 기능 확장이 용이함
    - 임의의 크기의 패킷을 주고받을 수 있도록 패킷 크기 제한이 없음
    - 멀티미디어의 실시간 처리가 가능함
    - 자동으로 네트워크 환경 구성이 가능함
    - 주소 체계는 유니캐스트(Unicast), 애니캐스트(Anycast), 멀티캐스트(Multicast)로 나뉨
- IPv6 통신 방식
    - 유니캐스트 : 하나의 호스트에서 다른 하나의 호스트에게 전달하는 1:1 통신 방식
    - 애니캐스트 : 하나의 호스트에서 그룹 내의 가장 가까운 곳에 있는 수신자에게 전달하는 '가장 가까운 1:1' 통신 방식
    - 멀티캐스트 : 하나의 호스트에서 네트워크상의 특정 그룹 호스트들에게 전달하는 1:n 통신 방식
- IPv4에서 IPv6로의 천이 전략
    - Dual Stack(듀얼 스텍) : 호스트가 IPv4와 IPv6를 모두 처리할 수 있도록 2개의 스택을 구성하는 전략
    - Tunneling(터널링)
        - IPv6를 사용하는 두 컴퓨터가 서로 통신하기 위해 IPv4를 사용하는 네트워크 영역을 통과해야 할 때 사용되는 전략
        - Tunneling을 통과하기 위해 패킷은 IPv4 주소를 가져야만 함
        - IPv6 패킷은 그 영역에 들어갈 때 IPv4 패킷 내에 캡슐화되고, 터널링을 나올 떄 역캡슐화됨
    - Header Translation(헤더 변환) : IPv4 패킷 헤더를 IPv6 패킷 헤더로 변환하거나 또는 그 반대의 동작을 수행하는 전력
- CSMA/CA
    - 무선 랜에서 데이터 전송 시, 매체가 비어있음을 확인한 뒤 충돌을 회피하기 위해 임의 시간을 기다린 후 데이터를 전송하는 방법
    - 네트워크에 디어트이 전송이 없는 경우라도 동시 전송에 의한 충돌에 대비하여 확인 신호를 전송함
- CSMA/CD
    - 전송 중에 충돌이 감지되면 패킷의 전송을 즉시 중단하고 충돌이 발생한 사실을 모든 스테이션들이 알 수 있도록 간단한 통보 신호를 송신함
    - 스테이션의 수가 많아지면 충돌이 많아져서 효울이 떨어짐
    - 어느 한 기기에 고장이 발생해도 다른 기기의 통신에 전혀 미치지 않음
3) 경로 제어 및 트래픽 제어
- 경로 제어(Routing) : 각 메시지에서 목적지가지 갈 수 있는 여러 경로 중 한 가지 경로를 설정해 주는 과정
- 경로 설정 프로토콜
    - 각 메시지에서 목적지까지 갈 수 있는 여러 경로 중 한 가지 경로를 설정해 주는 과정
    - IGP(Interior Gateway Protocol)
        - RIP(Routing Information Protocol)
            - 경유하는 라우터의 대수(hop의 수량)에 따라 최단 경로를 동적으로 결정하는 거리 벡터 알고리즘을 사용함
            - 홉 카운트가 15로 제한되어있지만 OSPF는 이런 제한이 없음
        - OSPF(Open Shortest Path First protocol) : 링크 상태 라우팅 프로토콜로 IP 패킷에서 프로토콜 번호 89번을 사용해 라우팅 정보를 전송해 안정되고 다양한 기능을 제공함
    - EGP(Exterior Gateway Protocol)
    - BGP(Border Gateway Protocol)
