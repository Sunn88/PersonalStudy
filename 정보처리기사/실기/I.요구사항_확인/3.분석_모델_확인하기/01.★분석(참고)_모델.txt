출제빈도 중

1) 분석(참고) 모델의 개념
- 요구사항을 분석하기 위한 잘 정의된 수행 방법, 절차
- 종류 : 구조적 분석 모델, 객체지향 분석 모델, 정보공학 분석 모델, 정형화 분석 모델
- 분석 모델 비교
    - 구조적 분석
        - 관점 : 자료 + 함수
        - 내용 : 자료보다는 함수에 중점을 두며 프로세스를 먼저 정하고 프로세스에 대한 입출력을 나중에 정함
        - 도구 : 순서도, DFD
    - 객체지향 분석
        - 관점 : 객체 + 객체
        - 내용 : 객체 자체와 객체 사이의 관계 파악이 중요하며 객체가 가지는 자료와 오퍼레이션의 정의
        - 도구 : UML 다이어그램
    - 정보공학 분석
        - 관점 : 자료 + 프로세스
        - 내용 : 자료 및 자료들 사이의 관계를 우선 파악하고 자료에 대한 오퍼레이션 패턴으로 프로세스를 그룹화
        - 도구 : 정보공학 관점의 UMl 다이어그램
    - 정형화 분석
        - 관점 : 시스템의 상태와 상태 변화
        - 내용 : 상태에 대한 논리적 표현이 중요함
        - 도구 : Petri-Net, 상태 전이도
            - Petri-Net : 프로세스 마이닝의 가장 기본이 되는 프로세스 모델 => 가장 간단한 형태로 프로세스를 나타낼 수 있음

2) 구조적 분석 모델
- 사용자의 요구분석 사항을 파악하기 위해 자료의 흐름과 가공 절차를 그림 중심으로 표현하는 방법
- 하향식(Top-Down Partitioning) 원리를 적용함
- 사용자의 업무 요구사항을 쉽게 문서화하고 사용자와 분석자 간의 의사소통을 위한 공용어이며 실체의 모형(추상적 표현)을 추출함
- 절차 : 배경도 작성 -> 상위 자료 흐름도 작성 -> 하위 자료 흐름도 작성 -> 자료 사전 작성 -> 소단위 명세서 작성
- 자료 흐름도(DFD, Data Flow, Diagram)
    - 시스템 내의 모든 자료 흐름을 4가지의 기본 기호(처리, 자료 흐름, 자료 저장소, 단말)로 기술하는 방법
    - 그림 중심의 표현이고 하향식 분할 원리를 적용함
    - 다차원적이며 자료 흐름을 중심으로 한 분석용 도구
    - 시스템이나 프로그램 간의 총체적인 데이터 흐름을 표시할 수 있으며, 기본적인 데이터 요소와 그들 사이의 데이터 흐름 형태로 기술됨
- 자료 흐름도와 작성 원칙
    - 출력 자료 흐름은 입력 자료 흐름을 이용해 생성해야 함
    - 입력, 출력 자료 자체에 대해서만 인지하고 자료의 위치나 방향은 알 필요가 없음
    - 자료 흐름 변환의 형태에는 본질의 변환, 합성의 변환, 관점의 변환, 구성의 변환 등이 있음
- 자료 흐름도의 구성 요소 - 67p 그림
    - 처리 공정(Process) : 자료를 변환시키는 과정을 나타냄
    - 자료 흐름(Data Flow) : 자료의 흐름을 나타냄
    - 자료 저장소(Data Store) : 파일, 데이터베이스를 나타냄
    - 단말(Terminator) : 자료의 출처와 도착지를 나타냄
- 자료 사전(DD, Data Dictionary)
    - 자료 흐름도에 있는 자료를 상세하게 기록한 것
    - 자료 사전의 표기법
        - = : 자료의 정의(Is composed of)
        - + : 자료의 연결(And)
        - () : 자료의 생략(Optional)
        - [|] : 자료의 선택(Or)
        - {} : 자료의 반복(Iteration of)
        - ** : 자료의 설명(Comment)
- 소단위 명세서(Mini-Specification)
    - 자료 흐름도의 처리 공정의 절차를 기술한 것
    - 프로세스 명세서라고도 함
- 개체 관계도(ERD, Entity-Relationship Diagram)
    - 데이터 구조들과 그들 간의 관계를 표현하는 방법
    - 구성 : 개체(Entity), 속성(Attribute), 관계(Relationship)
- 상태 전이도
    - 시스템 이벤트 발생 시 시스템의 상태 변화를 모델링하는 것
    - 시스템 상태는 직사각형으로 표현
    - 시스템 상태는 시스템이 활동 중인 것을 의미함

3) 객체지향 분석 모델
- 객체는 일의 단위를 의미함 -> 객체지향 분석 모델은 객체와 객체의 관계를 파악하는 방법으로 분석함
- 객체 자체와 객체 사이의 관계 파악이 중요함 -> 객체가 가지는 자료와 오퍼레이션의 정의가 중점사항
- 객체지향 분석을 기록하는 방법 : UML(Unified Modeling Language)을 가장 많이 사용함
