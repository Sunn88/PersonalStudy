출제빈도 상
- 공격 유형의 정의, 공격 방법

1) 서비스 공격 유형의 종류
- DoS(Denila of Service, 서비스 거부)
    - 시스템의 자원을 부족하게 해 원래 의도된 용도로 사용하지 못하게 하는 공격 방법
    - 정보 보안의 3대 요소 중 사용성(Availability)을 위협하는 행위
    - 공격자가 임의로 자신의 IP 주소를 속여서 다량으로 서버에 보냄
    - 헤더가 조작된 일련의 IP 패킷 조각들을 전송함
    - 라우터, 웹, 전자우편, DNS 서버 등 모든 네트워크 장비를 대상으로 이루어질 수 있음
    - 스머프(Smurf)
        - 공격 대상의 IP 주소를 근원지로 대량의 ICMP 응답 패킷을 전송해, 서비스 거부를 유발하는 공격 방법
        - IP 도는 ICMP의 특성을 악용해 특정 사이트에 집중적으로 데이터를 보내 네트워크 또는 시스템의 상태를 불능으로 만드는 공격 방법
    - TCP SYN flooding
        - TCP Connection은 반드시 30Way handshaking 과정을 거친 후 Session을 형성하고 통신이 시작됨
        - 이러한 TCP 연결 설정 과정의 취약점을 악용한 서비스 거부 공격
        - TCP 3-Way Handshaking 과정에서 Half-Open 연결 시도가 가능하다는 취약성을 이용한 공격 방법
    - UDP 플러딩(UDP flooding)
        - 대량의 UDP 패킷을 만들어 보내 정상적인 서비스를 하지 못하도록 하는 공격 방법
    - Ping 플러딩(Ping flooding)
        - 네트워크의 정상 작동 여부를 확인하기 위해 사용하는 Ping 테스트를 공격자가 공격 대상의 컴퓨터를 확인하려는 방법으로 사용하는 공격 방법
    - Ping of Death
        - 비정상적인 ICMP 패킷을 전송해, 시스템의 성능을 저하하는 공격 방법
    - 티어드랍(Teardrop)
        - 패킷 재조합의 문제를 악용해 오프셋이나 순서가 조작된 일련의 패킷 조각들을 보냄으로써 자원을 고갈시키는 공격 방법
    - LAND Attack(Local Area Network Denial Attack)
        - IP 스푸핑을 이용한 SYN 공격
        - 공격자가 패킷의 출발지 주소(Address)나 포트(Port)를 임의로 변경해 출발지와 목적지 주소(또는 포트)를 동일하게 함으로써, 공격 대상 컴퓨터의 실행 속도가 느려지거나 동작이 마비되어 서비스 거부 상태에 빠지도록 하는 공격 방법
        - 패킷의 출발지와 목적지의 IP 주소를 희생자 측으로 동일하게 변조한다는 것이 핵심
- DDos(Distributed Denial of Service, 분산 서비스 거부)
    - 여러 대의 공격자를 분산 배치해 동시에 서비스 거부 공격을 함으로써 공격 대상이 되는 시스템이 정상적인 서비스를 할 수 없도록 방해하는 공격 방법
    - DDoS 공격에 사용된 좀비 PC는 악성코드의 흔적을 지우기 위해 스스로 하드디스크를 손상시킬 수도 있으며, 이러한 좀비 PC끼리 형성된 네트워크를 봇넷(Botnet)이라 함
    - 공격용 도구 : Trinoo, TFN(Tribe Flood Network), TFN2K, Stacheldraft 등
- 피싱(Phishing)
    - 진짜 웹 사이트와 거의 동일하게 꾸며진 가짜 웹 사이트를 통해 개인정보를 탈취하는 수법
    - 금융기관 등의 웹 사이트에서 보내온 메일로 위장해 개인의 인증번호나 신용카드 번호, 계좌 정보 등을 빼내 이를 불법적으로 이용함
- 파밍(Pharming)
    - 도메인을 탈취하거나 악성코드를 통해 DNS의 이름을 속여 사용자가 진짜 웹 사이트로 오인하게 만들어 개인정보를 탈취하는 수법
    - 공격자는 사용자의 합법적 도메인을 탈취하거나 도메인 네임 시스템(DNS) 또는 프락시 서버의 주소를 변조해, 사용자가 진짜 사이트로 오인해 접속하도록 유도한 후 개인정보를 훔침
- 스니핑(Sniffing)
    - 네트워크상에서 자신이 아닌 다른 상대방들의 패킷 교환을 엿듣는 것
    - 네트워크 트래픽을 도청하는 과정
    - 네트워크상에서 전달되는 모든 패킷을 분석해 사용자의 계정과 암호 등을 알아내는 것
- 스미싱(Smishing)
    - 수신한 메시지에 있는 인터넷 주소를 클릭하면 악성코드를 설치해 개인 금융 정보를 빼내는 수법
    - 초대장 등의 내용을 담은 문자 메시지 내에 링크된 인터넷 주소를 클릭하면 악성코드가 설치되어 사용자의 정보를 빼가거나 소액결제를 진행함
- 큐싱(Qshing)
    - 사용자 인증 등이 필요한 것처럼 속여 QR코드(Quick Response Code)를 통해 악성 앱을 내려받도록 유도하는 수법
    - 스미싱에서 한 단계 더 진화된 금융 사기 수법
- 랜섬웨어(Ransomware)
    - 인터넷 사용자의 컴퓨터에 침입해 내부 문서 파일 등을 암호화해 사용자가 열지 못하게 하는 공격
    - 암호 해독용 프로그램의 전달을 조건으로 사용자에게 돈을 요구하기도 함
- 키 로거(Key Logger)
    - 컴퓨터 사용자의 키보드 움직임을 탐지해 ID, 패스워드 등 개인의 중요한 정보를 몰래 빼가는 공격 방법
- SQL 삽입(SQL Injection)
    - 공격자가 악의적으로 만든 SQL 명령을 응용 프로그램이 수행하도록 하는 공격 방법
    - 공격자가 입력 폼 및 URL 입력란에 SQL문을 삽입해 정보를 열람하거나 조작함
- XSS(Cross Site Scripting)
    - 게시판의 글에 원본과 함께 악성코드를 삽입해 글을 읽으며 악성코드가 실행되도록 해 클라이언트의 정보를 유출하는 공격 방법
    - 웹 페이지가 사용자로부터 입력받은 데이터를 필터링하지 않고 그대로 동적으로 생성된 웹 페이지에 포함해 사용자에게 재전송할 때 발생함
- 무작위 대입 공격(Brute-Force Attack)
    - 패스워드에 사용될 수 있는 문자열의 범위를 정하고, 그 범위 내에서 생성 가능한 패스워드를 활용하는 공격 방법
- 스피어 피싱(Spear Phishing)
    - 공격 대상이 방문할 가능성이 있는 합법적인 웹 사이트를 미리 감염시킨 뒤, 잠복하고 있다가 공격 대상이 방문하면 대상의 컴퓨터에 악성코드를 설치하는 공격 방법
- APT(Advanced Persistent Threat, 지능적 지속 위협)
    - 다양한 보안 위협을 만들어 침해에 성공해 정보를 유출하거나 장기간의 접속 권한을 획득하기 위해 또는 장기간의 접근을 위해 지속해서 수행되는 공격 방법
    - 개인 단체, 정치 단체, 국가, 산업체 등이 조직을 타깃으로 함
- 제로데이(Zero-day) 공격
    - 조사된 정보를 바탕으로 정보 시스템, 웹 애플리케이션 등의 알려지지 않은 취약점 및 보안 시스템에서 탐지되지 않는 악성코드 등을 감염시키는 것
- 백도어(Back Door)
    - 프로그램이나 손상된 시스템에 허가되지 않는 접근을 할 수 있도록 정상적인 보안 절차를 우회하는 악성 프로그램
    - 트랩 도어(Trap Door)라고도 함
    - 백도어 공격 도구 : NetBus, Back Orifice, RootKit 등
    - 백도어 탐지 방법 : 무결성 검사, 열린 포트 확인, 로그 분석, SetUID 파일 검사 등
    - tripwire : 크래커가 침입해 백도어를 만들어 놓거나, 설정 파일을 변경했을 때 분석하는 도구
- Rainbow Table Attack
    - 최근 패스워크 크래킹 기법으로 패스워드별로 해시값을 미리 생성해 놓은 테이블을 사용해 Reduction 함수의 반복 수행을 통해 일치하는 해시값으로 패스워드를 탈취하는 기법
- CSRF(Cross Site Request Forgery)
    - 특정 사용자를 대상으로 하지 않고 불특정 다수를 대상으로 로그인된 사용자가 자신의 의지와 상관없이 공격자의 의도에 따라 행위를 하게 만드는 공격
    - XSS 공격과 유사함
    - 유명 인터넷 쇼핑몰인 옥션에서 발생한 개인정보 유출 사건에 사용된 공격 기법
- TOCTOU(Time Of Check To Time Of Use)
    - 병렬 시스템을 사용할 때 두 지점 사이의 타이밍을 노리는 공격 또는 그런 공격을 가능하게 하는 버그 유형
    - 하나의 자원에 대해 동시에 검사 시점과 사용 시점이 달라 생기는 보안 약점으로 인해 동기화 오류뿐 아니라 교착 상태 등과 같은 문제점이 발생할 수 있음
