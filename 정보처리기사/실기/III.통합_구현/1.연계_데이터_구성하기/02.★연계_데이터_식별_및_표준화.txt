출제빈도 중

1) 연계 데이터 식별
- 연계 데이터 구성을 위해 연계 데이터를 구분해 식별하고, 이렇게 식별된 연계 데이터를 표준화해 
    연계(인터페이스) 정의서를 작성함
- 대내외 시스템 연계를 위해 송신 시스템과 수신 시스템 데이터에 대한 상세 식별 정보를 확인함

2) 연계 데이터 표준화
- 정의
    - 연계 데이터 중 코드화된 정보는 송신 시스템과 수신 시스템에서 상호 교환 가능하도록 
        코드 매핑 정보를 제공하거나 송/수신 시스템 간 다른 코드 정보를 표준화함
    - 인터페이스 설계 단계에서 송/수신 시스템 사이에 전송되는 표준 항목과 업무 처리 데이터, 
        공통 코드 정보 등을 누락 없이 식별하고 인터페이스 명세서를 작성함
        - 인터페이스 명세서 : 공통부, 개별부, 종료부로 구성됨
- 절차
    - 범위 및 항복 정의 -> 연계 코드 변환 및 매핑 -> 변경된 데이터 구분 방식 정의
        -> 데이터 표현 방법 정의 -> 정의서 및 명세서 작성
        - 범위 및 항목 정의 : 송/수신 시스템 연계 항목이 서로 다를 경우 일반적으로 수신 시스템 
            기준으로 정의함
    - 데이터 형식은 크게 데이터베이스 테이블과 데이터베이스 파일로 구분되며 데이터베이스 파일은 
        다시 태그, 콤마 등 구분자에 의해 구분함

3) 연계 데이터 식별 및 표준화 절차
- 정의
    - 연계 데이터 구성을 위해 연계 데이터를 분류 식별하고, 식별된 연계 데이터를 표준화하는 과정
        -> 이 수행 결과로 연계(인터페이스) 정의서를 작성함
    - 연계 범위 및 항목 정의 -> 연계 데이터 코드 변환 및 매핑 -> 연계 데이터 식별자와 변경
        구분 추가 -> 연계 데이터 표현 방법 정의 -> 연계(인터페이스) 정의서 및 명세서
- 절차
    - 연계 범위 및 항목 정의
        - 시스템 간에 연계하려는 정보를 상세화하며 범위와 항목을 정의함
        - 상세화하는 방법은 연계 필요 정보를 정보 그룹에서 그룹을 구성하는 단위 항목으로 확인함
        - 송신/수신 시스템에서 연계대상 항목의 데이터 타입 및 길이, 코드와 여부 등을 확인함
            - 연계대상 확인 :  송신 시스템의 '테이블 정의서', 수신 시스템의 '테이블 정의서'를 
                참고함
        - 송신/수신 시스템의 연계 항목이 다를 경우, 수신 시스템 기준으로 적용 및 변환함
    - 연계 데이터 코드 변환 및 매핑
        - 연계대상 정보에서 코드로 관리되는 항목을 변환함
        - 송신 시스템 코드를 수신 시스템 코드로 매핑해주는 방법과, 송수신 시스템에서 사용되는 
            코드를 통합해 표준화한 후 전환하는 방법이 있음
    - 연계 데이터 식별자(PK, Primary Key)와 변경 구분 추가
        - 송신 시스템의 식별키 항목을 추가해 송신된 정보가 수신 시스템의 어떤 데이터에 해당하는지
            구분함
        - 송신 정보를 수신 시스템의 테이블에 추가, 수정, 삭제할 데이터인지 식별해주는
            구분 정보를 추가함
            - 구분 정보 추가 : 변경 구분 송신 데이터를 수신 시스템에 반영하기 위해
                구분 정보를 추가함
        - 인터페이스 테이블 또는 파일에 관리 정보를 추가해 연계되는 정보의 송수신 여부,
            송수신 일시, 오류 코드 등을 확인하고 감시함
    - 연계 데이터 표현 방법 정의
        - 연계하고자 하는 연계대상 범위와 항목, 연계 데이터 코드 매핑 방식 등을 정의해
            연계 데이터를 테이블이나 파일 등의 형식으로 구성함
        - 구성된 연계 데이터는 연계 데이터를 생성하는 시점, 연계 주기, 적용되는 연계 솔루션의 
            지원 기능 등에 다르게 표현될 수 있음
        - 연계 데이터 표현 방법을 상세화함
    - DB : 테이블, 파일 형식
    - 파일 : 파일 형식에 따라 태그(Tag), 항목 분리자(델리 미터, Delimiter) 사용
    - 연계(인터페이스) 정의서 및 명세서
        - 연계 항목, 연계 데이터 타입, 길이 등을 구성하고 형식을 정의해 결과물인 
            연계(인터페이스) 정의서를 작성함
            - 연계(인터페이스) 정의서 : 송신 시스템과 수신 시스템 간의 인터페이스 현황을 작성함
        - 연계(인터페이스) 정의서에는 송/수신 시스템 간의 인터페이스 현황을 작성함
        - 연계(인터페이스) 정의서에 작성한 인터페이스 ID별로 송수신하는 데이터 타입, 길이 등 
            인터페이스 항목을 상세하게 작성함
