출제빈도 중

1) 통합 구현의 정의
- 단위 모듈 간 연계와 통합을 통해 사용자의 요구사항을 수용하고, 새로운 서비스 추가를 위한 절차
- 시스템 아키텍처 구성, 송수신 방식과 송수신 모듈 구현 방법 등에 따라 다양함
- 구축하고자 하는 실제 환경과 사용자 요구사항에 따라 알맞은 통합 구현 방법을 설계하도록 함

2) 연계 시스템 구성
- 송신 시스템
    - 전송하고자 하는 데이터를 생성해 필요에 따라 변환 후 송신하는 송신 모듈과 데이터 생성 및 송신 상태를 모니터링하는 기능으로 구성됨
    - 송신 모듈 : 내부 연계 시에도 필수 요소에 해당하며 일반적으로 연계 솔루션이 적용될 때는 데이터 생성 처리만 구현함
    - 모니터링 : 송신 시스템 내의 처리 과정 및 송신 상태 등을 확인할 수 있도록 구현함
- 중계 시스템
    - 송/수신 시스템 간의 연계 시에 적용되는 아키텍처
    - 송/수신 구간을 분리해 보안성이 강화됨
    - 인터넷 망(Internet)과 인트라넷 망(Intranet)을 연결할 수도 있음
    - 중계 모둘 : 송신된 데이터의 오류 처리 및 수신 시스템의 데이터 형식으로 변환 또는 매핑 등을 수행함
- 수신 시스템
    - 수신 모듈과 모니터링으로 구성됨
    - 수신 모듈 : 수신받은 데이터를 정제(Cleansing)하고, 응용 애플리케이션이나 데이터베이스의 테이블에 적합하도록 변환해 반영하는 기능 수행
    - 모니터링 : 연계 데이터의 수신 현황 및 오류 처리, 데이터 반영을 모니터링하는 기능 수행
- 연계 데이터
    - 의미를 내포한 속성, 길이, 타입 등 실제 송수신되는 데이터
    - 송수신되는 연계 데이터 형식 : 데이터베이스의 테이블, 컬럼, 파일
    - 파일 형식 : TXT, XML, CSV 등
        - CSV(Comman Separated Value)
            - 쉼표를 기준으로 항목을 구분해 저장한 데이터
            - 데이터베이스나 스프레드시트 데이터를 보존하거나 다른 소프트웨어에 데이터를 주도받을 때 사용하는 파일 형식
- 네트워크
    - 각 시스템을 연결해 주는 통신망
    - 유선/무선, ISP(Internet Service Provider) 사업자의 공증망, 사설망 등의 물리적인 망과 송수신 규약을 위한 프로토콜(Protocol)을 의미함

3) 연계 요구사항 분석
- 정의
    - 통합 구현을 위해 연계 시스템과 관련된 요구사항을 분석하는 과정
    - 연계에 관한 요구사항은 연계 데이터 및 환경 구성을 위해 성능과 데이터 발생빈도 및 유형과 보안 등의 다양한 관점으로 분석하도록 함
- 절차 : HW/SW/네트워크 확인 -> 코드/테이블 정의서 확인 -> 점검표 작성 -> 인터뷰/설문 조사 시행 -> 요구사항 분석서 작성
- 기법 : 사용자 인터뷰, 핵심 그룹 면담, 점검표, 설문지 및 설문 조사, 체크리스트, 델파이 기법, 연계 솔루션 비교
    - 델파이 기법 : 전문가의 경험적 지식을 통한 분석 기법 -> 업무 전문가 등 각 분야 전문가로 구성됨
    - 연계 솔루션 비교 : 다양한 연계 방식과 연계 솔루션별(EAI, ESB, Open API 등) 연계시의 성능, 보안, 데이터 처리, 모니터링 등의 장단점을 비교
- 분석 시 참고 문서
    - 시스템 구성도 : 송/수신 시스템의 네트웤, 하드웨어 및 시스템 소프트웨어의 구성
    - 응용 애플리케이션 구성 : 애플리케이션의 메뉴 구성, 화면 설계서, 사용자 인터페이스 정의서, 데이터 발생 시점, 발생 주기, 발생 패턴
    - 엔티티 관계도, 공통코드 정의서 : 데이터 모델링 기술서, 테이블 간의 연관도, 정의된 테이블 정의서, 공통 코드와 공통 코드에 관한 설명서

4) 연계 모듈 구현 시 환경 구축 절차
- 연예 메커니즘 정의서 또는 연계 아키텍처 설계서를 확인 및 점검함
- 송신 시스템의 연꼐 모듈 구현 환경을 구축함
- 수신 시스템의 연계 모듈 구현 환경을 구축함
- 송/수신 시스템의 연계 응용 프로그램을 구현함
