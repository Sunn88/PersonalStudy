출제빈도 상

1) 일반적인 시스템 개발 절차
- 비즈니스 요구사항
- 전략 수립 : 개념 모델링
- 분석 : 논리 데이터 모델링, 분석 모델링
- 설계 : 물리 데이터 모델링, 설계 모델링
- 개발 : 데이터베이스 구축, 애플리케이션 개발
- 운영 시스템 : 데이터베이스 <-> 애플리케이션

2) 데이터의 세계
- 현실 세계를 데이터로 표현하는 데이터베이스 설계 과정에서 컴퓨터에 저장할 데이터의 구조를 논리적으로 표현하기 위해 사용하는 지능적 도구
- 데이터는 기본적으로 세 가지의 다른 세계
    - 인간의 오감을 통해 감지할 수 있는 개체(Entity)로 되어 있는 현실 세계
    - 현실 세계에 존재하는 실체의 의미로부터 얻은 개념으로 표현한 개념 세계
    - 게념 세계를 컴퓨터가 처리할 수 있는 데이터로 표현한 물리적 세계
* 데이터 모델
    - 현실 세계의 정보를 컴퓨터 세계의 환경에 맞게 표현하기 위해 단순화/추상화하여 체계적으로 표현한 개념적인 도구
    - 현실 세계를 데이터베이스에 표현하는 중간 과정, 즉 데이터베이스 설계 과정에서 데이터의 구조를 표현하기 위해 사용되는 도구
    - 데이터 모델에서 표현해야 할 요소
        - 논리적으로 표현된 데이터 구조
        - 구성요소의 연산
        - 구성요소의 제약 조건

3) 데이터 모델링의 개요
- 현실 세계의 정보 구조를 실체(Enntity)와 관계(Relation)를 중심으로 명확하고 체계적으로 표현해 문서로 만드는 기법
- 데이터 모델링의 목적
    - 개발 대상과 연관 조직의 정보 요구에 대해 정확한 이해를 할 수 있음
    - 사용자, 설계자, 개발자 간에 효율적인 의사소통 수단을 제공함
    - 데이터 체계 구축을 통한 고품질 S/W와 유지보수 비용의 감소 효과를 기대할 수 있음
    - 신규 또는 개선 시스템의 개발 기초를 제공함
- 데이터 모델링의 특성
    - 데이터 중심 분석을 통한 업무 흐름 파악이 쉬움
    - 데이터 무결성을 보장할 수 있음
    - 데이터의 공유를 통한 중복을 제거하고 일관성 있는 정보를 받을 수 있음

4) 데이터 모델의 분류
- 데이터베이스 모델
    - 개념적 모델
        - E-R 다이어그램
    - 논리적 모델
        - 계층형
        - 네트워크(망)형
        - 관계형
        - 객체지향형

5) 데이터베이스 설계 순서
- 데이터베이스 계획
- 요구사항 분석 : 데이터베이스에 저장된 내용을 정하기 위해 사용자의 요구사항 분석
- 개념적 데이터 설계 : DBMS 독립적 개념 스키마 설계, 트랜잭션 모델링 및 정의
- 논리적 데이터 설계 : 목표 DBMS에 맞는 스키마 설계, 트랜잭션 인터페이스 설계
- 물리적 데이터 설계 : 목표 DBMS에 맞는 물리적 구조 설계, 트랜잭션 세부 설계
- 구현

6) 데이터 모델링 절차
- 데이터 모델링은 개념 모델링, 논리 모델링, 물리 모델링을 통해 데이터베이스를 구축하는 일련의 절차를 거쳐 진행됨
- 요구사항 분석
    - 데이터베이스를 사용할 사람들이 필요로 하는 요구를 분석하고 명세서를 작성하는 단계
- 개념 데이터 모델링(Conceptual Data Model)
    - 현실 세계에 있는 그대로 사람이 이해할 수 있는 형태의 정보 구조(Information Structure)로 만들어 가는 과정을 의미 -> 정보 모델이라고도 함
    - 속성들로 기술된 개체 타입과 이 개체 타입 간의 관계를 이용해 현실 세계를 표현하는 방법
    - 대표적 개념적 데이터 모델 : 개체-관계(E-R) 모델
- 논리 데이터 모델링(Logical Data Model)
    - 개념적 데이터 모델링 과정에서 추출된 엔티티와 속성들의 관계를 구조적으로 정의하는 단계
    - 개념적 구조를 컴퓨터가 이해하고 처리할 수 있도록 변환하는 과정
    - 데이터베이스 개발 과정의 첫 단계 -> 전략 수립 및 분석 단계에서 실시함
    - 이해당사자들과 의사소통의 보조자료로서 산출된 E-R 모델을 대상으로 해 구축 대상 DBMS에 맞게 스키마를 설계함
    - 데이터 구조에 대한 논리적 정의 단계로서 정확한 업무 분석을 통한 자료의 흐름을 분석해 현재 사용 중인 양식, 문서, 장표를 중심으로 자료 항목을 추출
        -> 필드로 기술된 데이터 타입과 이 데이터 타입 간의 관계를 이용해 현실 세계를 표현함
    - 데이터를 정규화(Normalization)해 모델링하고 규칙과 관계를 완전하고 정확하게 표현함
    - 성능 혹은 기타 제약 사항과는 독립적인 모델 -< 논리적 모델은 H/W나 S/W에 독립적임
    - 종류
        - 계층형 데이터베이스 모델
        - 네트워크(망)형 데이터베이스 모델
        - 관계형 데이터베이스 모델
        - 객체지향형 모델
- 물리 데이터 모델링
    - 논리적 데이터 설계에서 구조화된 데이터베이스를 실제 저장장치에 어떻게 저장할지를 설계하는 단계
    - 주어진 응용 프로그램에 대한 성능 향상을 위해 데이터베이스 파일에 특정한 저장 구조로 접근 경로를 결정
        - 레코드 양식의 순서, 저장 공간, 액세스 경로 인덱싱, 클러스터링, 해싱 등의 설계가 포함됨
    - 설계 단계에서 시스템의 설계적 및 정보 요건을 정확하고 완전하게 표현한 모델 -> 데이터베이스 생성을 위한 물리 구조로 변환함
    - 시스템 설꼐 요건 반영을 위한 설계용 엔티티 타입, 설계용 속성 같은 객체를 추가함
    - 적용 DBMS 특성과 엔티티 타입의 분리 또는 통합을 검토함
    - 반정규화(Renormalization)와 관계의 해제를 통해 설계와 성능을 고려한 조정을 수행함
    - 인덱스 추가 및 조정, 테이블 스페이스 조정, 인덱스 스페이스 조정을 통해 적용 DBMS에 적합한 성능조정을 수행함
- 구현
    - 데이터베이스를 실제로 구축하는 과정 -> 목표 DBMS의 데이터 정의어(DDL)로 스키마를 생성함

7) 개체-관계 모델의 정의
- 대표적인 개념적 모델링 기법으로 현실 세계의 데이터베이스 모델을 시각적으로 표현함
- 데이터베이스 설계 과정 중 개념적 설계 과정에 사용되는 하나의 모델로 개체 타입과 이들 간 관계 타입을 이용해 현실 세계를 개념적으로 표현함
- 특정 DBMS에 종속되지 않으며, 일대일(1:1), 일대다(!:N), 다대다(N:M) 관계를 모두 나타낼 수 있음

8) 개체-관꼐 다이어그램(E-R 다이어그램, ERD)
- 개체-관계 다이어그램은 여러 가지 기호를 이용해 그래프 형태로 표현한 것
- 기호의 종류와 의미 : 100p 표
