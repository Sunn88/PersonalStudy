빈출태그 : UNIX의 커널 / UNIX의 쉘 / UNIX의 주요 명령어

1) 운영체제의 종류
- Windows의 특징
    - 마이크로소프트사에서 발표하고 있는 컴퓨터 운영체제
    - GUI(Graphic User Interface)이며, 선점형 멀티태스킹으로 수행됨
    - PnP(Plug and Play)가 지원됨
    - OLE(Object Linking and Embedding) 지원으로 응용 프로그램 간의 자료를 공유함
    - Single-User System : 개인용 윈도우의 경우에는 사용자 1인만 사용 가능
- MS-SOX(Microsoft Disk Operating System)
    - CUI(Character User Interface), Single-User, Single-Tasking의 특징을 가짐
    - 트리 구조 파일 시스템
    - 시스템 부팅 시 필수인 파일 : MSDOS.SYS, IO.SYS, COMMAND.COM
    - MS-DOS 명령어
        - 내부 명령어 : 명령어 처리 루틴이 메모리에 상주하는 명령어
            - ex) DIR, COPY, DEL, TYPE, CLS 등
        - 외부 명령어 : 디스크에 파일로 저장된 명령어
            - ex) FORMAT, DISKCOPY, DISKCOMP 등
- 유닉스(UNIX) 시스템의 특징
    - 이식서이 높으며 대화식 운영체제
    - C언어라는 고급 프로그래밍 언어로 커널까지 작성된 운영체제
    - 서버 운영에 필수적인 CLI 인터페이스가 강력함
    - 파일 생성, 삭제, 보호 기능을 가지며, 디렉터리 구조는 트리 구조 형태임
    - 멀티태스킹과 멀티유저 운영체제
    - 파일 소유자, 그룹 및 그 외 다른 사람들로부터 사용자를 구분해 파일을 보호함
    - 주변장치를 파일과 동일하게 취급함
    - UNIX 계열의 운영체제 : 리눅스(LINUX), MAC OS X 등
- 리눅스(LINUX)의 특징
    - UNIX와의 호환이 완벽함
    - 허가권과 소유권의 권한을 가짐
    - 멀티태스킹과 멀티유저를 지원함
    - 대소문자를 구분함
    - Windows의 파일 확장자라는 개념이 없음
    - 종류 : RHEL, CentOS, Fedora, Ubuntu, Raspbian, Kali

2) UNIX의 개요
- 유닉스의 특징
    - 다중 사용자 시스템 -> 대화식 운영체제
    - 높은 이식성과 확장성을 가짐
    - 네트워킹 시스템
    - 계층적 파일 시스템
- 유닉스의 구성 : 하드웨어-커널-쉘-유틸리티-사용자
- 커널(Kernel)
    - UNIX 시스템의 중심부에 해당함
    - 주기억장치에 적재된 후 상주하면서 실행됨
    - 기능 : 프로세스 관리, 기억장치 관리, 입/출력 관리, 파일 관리, 시스템 호출 인터페이스 등
    - 하드웨어를 캡슐화함
    - 대부분이 C언어로 개발되어 이상성과 확장성이 뛰어남 -> C언어 90% + 어셈블리어 10%
- 쉘(Shell)
    - 명령어 해석기(Interpreter)
    - 사용자의 명령어를 인식해 필요한 프로그램을 호출하고 그 명령을 수행하는 기능 담당
    - 사용자와 시스템 간의 인터페이스를 제공하며 사용자의 명령어들을 처리하는 스크립트 기능 지원
    - DOS의 COMMAND.COM과 같은 역할
- 유틸리티(Utility)
    - 사용자의 편의를 위한 프로그램 제공
    - 문서 편집기, 컴파일러, 정렬 기능 제공

3) UNIX의 파일 시스템
- 디렉터리 구조 : 트리 구조
- 구조
    - 부트 블록(Boot Block)
        - 부팅에 필요한 코드를 저장하고 있는 블록
    - 슈퍼 블록(Super Block)
        - 전체 파일 시스템에 대한 정보를 저장하고 있는 블록
    - i-node 블록(Index Node Block)
        - 각 파일에 대한 정보를 저장하고 있는 블록
        - 파일 소유자의 식별 번호, 파일 크기, 파일의 생성 시간, 파일의 최종 수정 시간,
            파일 링크 수 등이 기록됨
    - 데이터 블록(Data Block)
        - 실제 데이터를 저장하고 있는 블록

4) UNIX의 주요 병령어
- UNIx는 쉘에 CLI 방법으로 운영체제를 제어함
- 명령어
    - fork : 프로세스 생성, 복제
    - exec : 새로운 프로세스 수행
    - exit : 프로세스 수행 종료
    - wait : 자식 프로세스의 하나가 종료될 때까지 부모 프로세스를 임시 중지시킴
    - getpid : 자신의 프로세스 아이디를 얻음
    - getppid : 부모의 프로세스 아이디를 얻음
    - mount : 기존 파일 시스템에 새로운 파일 시스템을 서브 디렉터리에 연결
    - umount : 마운팅된 파일 시스템에서 해제
    - chdir : 디렉터리의 위치 변경
        - DOS의 유사 명령어 : cd
    - cp : 파일 복사
        - DOS의 유사 명령어 : copy
    - mv : 파일 이동
        - DOS의 유사 명령어 : move
    - rm : 파일 삭제
        - DOS의 유사 명령어 : del
    - cat : 파일 내용 화면에 표시
        - DOS의 유사 명령어 : type
    - chmod : 파일의 사용 허가 지정
        - DOS의 유사 명령어 : attrib
    - ls : 현재 디렉터리 내의 파일 목록 확인
        - DOS의 유사 명령어 : dir
    - chown : 파일 소유자를 변경

5) Shell script
- 쉘 : 커널과 유저를 이어주는 명령어 해석기
- 쉘의 종류 : BASH Shell, Bourne Shell, C Chell, Korn Shell 등
- 쉘 스크립트 : 쉘에서 사용할 수 있는 명령어들로 작성된 쉘 프로그램
    - 쉘 스크립트를 통해 쉘 명령어들을 순서대로 실행 가능함
    - 응용 SW 개발의 첫 단계에서 전체 동작 상태를 점검해 볼 수 있기 때문에 전체 구조상의
        중요한 결함을 발견하는 것이 가능함
    - 애플리케이션의 프로토타입으로 쉘 스크립트를 유용하게 사용함
- 쉘 스크립트 구성 : 명령어, 변수, 제어문, 조건식, 메타 문자(meta character) 등
- C언어와 유사하며, 스크립트 언어이기 때문에 컴파일이 필요하지 않음
- 쉘 스크립트 프로그램은 명령어들을 나열하 놓은 배치(batch) 파일의 형태
- 쉘 스크립트 작성 도구 : vi 편집기, gedit
- 쉘 스크립트 파일 확장자 : *.sh
