출제빈도 상
빈출태그 : 프로토콜 / TDP/IP / X.25

!) 프로토콜의 개념
- 둘 이상의 컴퓨터 사이에 데이터 전송을 할 수 있도록 미리 정보의 송수신측에서 정해 둔 통신 규칙
- 다른 기종의 장비는 각기 다른 통신 규약을 사용하는데, 프로토콜을 사용하면 다른 기기 간 정보의 전달을
    표준화할 수 있음

2) 프로토콜의 기본 요소
- 구문(Syntax) : 전송 데이터의 형식, 부호화, 신호 레벨 등을 규정함
- 의미(Semantic) : 전송 제어와 오류 관리를 위한 제어 정보를 포함함
- 타이밍(Timing) : 두 개체 간에 통신 속도를 조정하거나 메시지의 전송 및 순서도에 대한 특성을 가리킴

3) 통신 프로토콜의 기능
- 흐름 제어(Flow Control)
    - 네트워크 내의 원활한 흐름을 위해 송수신측 간에 전송되는 패킷의 흐름 및 속도를 규제하는 것
    - 정지 및 대기(Stop and Wait), 슬라이딩 윈도우(Sliding Windows) 방식을 이용함
- 연결 제어(Connection control)
    - 통신을 위해 송신측과 수신측 사이의 연결을 설정하고, 유지하고, 종료하는 등의 작업을 제어하는 것
- 오류 제어(Error control)
    - 데이터 전송 중에 발생하는 메세지나 제어 정보의 파손에 대비해 오류를 검출하고 수정하는 것
- 순서 제어(Sequencing control)
    - 데이터 전송 시 송신측이 전송하는 데이터들을 분할한 PDU의 순서대로 수신측에 전달해 순서를
        결정하는 것
- 동기화(Synchronization)
    - 통신하는 송신측과 수신측 간의 초기화 상태, 검사 전 상태, 종료 상태 등과 같이 명확히 정의된
        상태를 서로 맞춰줌으로써 통신의 진행을 원활히 할 수 있는 것
- 다중화(Multiplexing)
    - 하나의 회선(혹은 전송로)을 다수의 주파수 대역이나 시간 간격으로 분할해 각각의 영역에서
        독립된 신호를 동시에 송수신 가능하도록 빠르고 정확하게 통신로를 지원하는 것
- 주소 지정(Addressing)
    - 송신측과 수신측의 주소를 설정하고 주소에 따른 메시지의 형태를 구분할 수 있는 것

4) X.25
- X.25의 특징
    - 패킷 교환망에 대한 ITU-T의 권고안으로 DTE와 DCE의 인터페이스를 규정함
    - 흐름 및 오류 제어 기능을 제공하며 패킷형 단말기를 패킷 교환망 접속을 위한 인터페이스 프로토콜
    - 물리 계층, 프레임 계층, 패킷 계층 3개의 계층으로 구성됨
    - OSI 계층의 하위 계층인 물리, 데이터링크, 네트워크 계층까지를 규정함
    - 가상회선을 영구 가상회선과 교환 가상회선으로 구분함
    - 헤더는 GFI(general format identifier) 4bit, LCI(logical channel identifier) 12bit로 구성됨
- X.25의 계층 구조
    - 물리 계층 : OSI 7계층의 물리 계층에 해당함
    - 프레임 계층(링크 계층) : OSI 7계층의 데이터 링크 계층에 해당함
    - 패킷 계층 : OSI 7계층의 네트워크 계층에 해당함

5) TCP/IP
- IP(Internet Protocol)
    - OSI 7계층의 네트워크 계층에 해당함 -> 비신뢰성 서비스를 제공함
    - 데이터그램이라는 데이터 전송 형식을 가짐
    - 신뢰성이 부족한 비연결형 서비스를 제공하기 때문에 상위 프로토콜에서 이러한 단점을 보완해야 함
    - 비연결성이기 때문에 송신지가 여러 개인 데이터그램을 보내면서 순서가 뒤바뀌어 도달할 수 있음
    - 기능 : 패킷 분해/조립, 호스트 주소 지정, 경로 선택
    - 직접 전달(direct delivery) : 송신자는 목적지 IP 주소를 이용해 목적지 물리 주소를 찾아서
        (ARP 이용) 데이터 링크 계층으로 보내어 패킷을 전달함
    - 간접 전달(indirect delivery) : 최종 목적지와 같은 네트워크에 연결된 라우터에 도달할 때 까지
        여러 라우터를 경유해 전달함
- TCP/IP의 구조(=인터넷 모델 계층)
    - 링크 계층(Link Layer)
        - 프레임(실제 데이터) 송/수신
        - Ethernet, IEEE 802, HDLC, RS-232C
        - OSI 7계층의 물리 계층, 데이터링크 계층
    - 인터넷 계층(Intenet Layer)
        - 주소 지정, 경로 배정, 폭주 제어 기능을 수행하는 비연결형 패킷 전달 서비스 제공
        - IP(Internet Protocol)
            - 신뢰성 없는 비연결형 서비스로 데이터가 순서를 무시하고, 손실 또는 중복될 수 있음
            - 오류 검사 및 추적을 제공하지 않음
        - ARP(Address Resolution Protocol)
            - IP Address를 물리적 하드웨어 주소(MAC Address)로 변환하는 프로토콜
        - RARP(Reverse Address Resolution Protocol)
            - 호스트의 물리 주소를 통해 논리 주소인 IP 주소를 얻어 오기 위해 사용되는 프로토콜
        - ICMP(Internet Control Message Protocol)
            - IP 프로토콜에서 오류 보고와 오류 수정 기능, 호스트와 관리 질의를 위한 제어 메시지를
                관리하는 네트워크 계층 프로토콜
            - 메시지는 하위 계층으로 가기 전에 IP 프로토콜 데이터그램 내에 캡슐화됨
            - 메시지는 4바이트의 헤더와 가변 길이의 데이터 영역으로 나뉨
                - type(8)
                - code(8)
                - checksum(16)
                - icmp 메시지(가변)
        - IGMP, X.25
        - OSI 7계층의 네트워크 계층
    - 전송 계층(Transport Layer)
        - 호스트 간 통신 제공
        - TCP, UDP
        - OSI 7계층의 전송 계층
    - 응용 계층(Application Layer)
        - 응용 프로그램 간의 데이터 송수신
        - FTP, SMTP, SNMP, Telnet
        - OSI 7계층의 세션 계층, 표현 계층, 응용 계층
