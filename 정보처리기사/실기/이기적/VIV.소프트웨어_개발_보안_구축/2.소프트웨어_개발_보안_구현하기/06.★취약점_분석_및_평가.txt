출제빈도 중
빈출태그 : 취약점 분석 및 평가 / 로그 분석

1) 취약점 분석 및 평가
- 개념
    - 악성코드 유포, 해킹 등의 사이버 위협에 대한 시스템의 취약점을 종합적으로 분석, 평가,
        개선하는 일련의 과정
    - 시스템의 안정적인 운영을 위협하는 사이버 보안 점검 항목과 각 항목별 세부 점검 항목을
        도출해 취약점 분석을 실시함
    - 발견된 취약점에 대해 위험 등급을 부여하고, 개선 방향을 수립하는 등 유기적인 취약점 평가를
        수행함
- 수행 주체 및 주기
    - 수행 주체
        - 관리 기관이 직접 수행할 경우 자체 전담반을 구성해 운영함
        - 관리 기관이 외부 기관에게 위탁할 경우 한국인터넷진흥원, 정보공유/분석센터,
            한국전자통신연구원, 지식정보보안 컨설팅전문업체 등 전문기관에 위탁 수행함
    - 수행 주기
        - 매년 정기적으로 취약점 분석/평가를 실기함
        - 가용 자원과 대상 시설 식별, 자산 중요도 산정 및 해당 시스템 대한 정밀 분석을 실시함
        - 대상 시스템에 중대한 변화가 있거나, 관리 기관에서 필요하다고 판단할 경우 수시로
            취약점 분석/평가를 실시할 수 있음
- 범위 및 항목
    - 취약점 분석/평가의 범위
        - 취약점 분석/평가의 범위 : 정보 시스템 자산, 제어 시스템 자산, 의료 시스템 자산 등
        - 정보 시스템 자산에 직간접적으로 관여하는 물리적 분야, 관리적 분야, 기술적 분야를 포함함
        - 타 시스템과 연계된 경우 연계 시스템이 미치는 영향을 포함함
    - 취약점 분석/평가의 기본 항목
        - 기본 항목은 관리적 분야, 물리적 분야, 기술적 분야로 구분함
        - "상/중/하" 3단계로 중요도를 분리함
        - 중요도 "상"인 점검 항목은 필수적으로 점검함
        - "중", "하" 항목은 기관의 사정에 따라 선택적으로 점검함
- 수행 절차 및 방법
    - 1단계 : 취약점 분석/평가 계획 수립
        - ex) 수행 주체, 수행 절차, 소요 예산, 산출물 등
    - 2단계 : 취약점 분석/평가 대상 선별
        - 자산(IT 자산, 제어 시스템 자산, 의료 장비 등)을 식별함
        - 네트워크 장비, 보안 장비, 시스템 장비 등 자산 유형별로 그룹화함
        - 취약점 분석/평가 대상 목록을 작성함
        - 식별된 대상 목록의 각 자산에 대해 중요도를 산정함
    - 3단계 : 취약점 분석
        - 취약점 분석/평가를 위한 관리적분야, 물리적분야, 기술적분야별 세부 점검 항목표를 수립함
        - 취약점 분석 요령은 관리적 분야, 물리적 분야, 기술적 분야로 구분해 확인함
    - 4단계 : 취약점 수행
        - 취약점 분석 결과에 대한 세부 내용 서술
        - 발견된 취약점별로 위험 등급을 표시하고 개선 방향을 수립함
        - 위험 등급 "상"은 조기 개선, "중"과 "하"는 중기 또는 장기 개선으로 구분해 개선 방향 수립

2) 로그 분석
- 로그(Log)분석의 개념
    - 로그는 데이터에 대한 기록이 파일 또는 인쇄물 형태로 저장된 것
    - 로그는 시스템 내부에서 영향을 미치거나 네트워크를 통한 외부에서 시스템에 영향을 미칠 때
        해당 데이터를 기록해 문제를 해결하거나 예방하는데 활용됨
- 로그 관리
    - 사용자가 시스템에 로그인해 명령하는 과정에 대한 시스템의 동작에 대해 인증(authentication),
        인가(Authorization), 계정(Accounting)에 대한 로그를 기록함
    - 로그는 중앙 시스템에 기록하거나 분산 기록함
- 로그 관리 구현
    - 로그를 정기적으로 분석하고 그 결과를 보고함
    - 정기적인 로그 분석을 통해 시스템 침입 흔적과 취약점을 확인할 수 있음
    - 로그의 정기적 검토 및 보고를 위해 로그의 정기적 검토 및 보고 체계를 수립하고,
        정책에 따른 시스템 로깅을 설정함
- 로그 관리 구현 여부 테스트
    - 로그의 정기적 검토 및 보고 체계를 수립했는지 테스트함
    - 정기적인 로그 분석을 통해 시스템 침입 흔적과 취약점을 확인할 수 있도록 로그의 정기적 검토
        및 보고 체계를 수립했는지 테스트함

3) AAA(인증, 권한 부여, 계정 관리)
- 개념
    - 일관된 방식을 통해 3개의 독립적인 보안 기능의 세트를 구성하기 위한 프레임워크
    - 서버/클라이언트 구조를 가짐
    - 인증(Authentication) : 망, 시스템 접근을 허용하기 전에 사용자의 신원 검증을 제공함
    - 권한 부여(Authorization) : 신원이 검증된 사용자에게 허용할 권한을 제공함
    - 계정 관리(Accounting) : 권한이 부여된 사용자의 행위를 기록하고 사용자 자원에 대한 정보를
        취합해 과금, 감사, 증설, 리포팅 등을 제공함
- AAA 기능 구현을 위한 인증 프로토콜
    - RADIUS
        - AAA : Combine
        - 암호화 : 패스워드만
        - Protocol : 서버 to 클라이언트(단방향)
        - Layer 3 Protocol : UDP
        - 보안 기능 : 대칭키 암호화 방식
    - DIAMETER Protocol
        - AAA : Separated
        - 암호화 : 패킷 페이로드 전체 암호화
        - Protocol : Peer to Peer(양방향)
        - Layer 3 Protocol : TCP/SCTP
        - 보안 기능 : End-To-End(TLS), IPSec/TLS
    - TACACS+(Terminal Access Controller Access Control System+)
        - AAA : Separated
        - 암호화 : 패킷 페이로드 전체 암호화
        - Protocol : 서버 to 클라이언트(단방향)
        - Layer 3 Protocol : TCP
        - 보안 기능 : 대칭키 암호화 방식
    - Kerberos
        - AAA : Separated
        - 암호화 : DES 암호화 사용
        - Protocol : AS(Authentication Server), TGS(Ticket Granting Service), SS(Service Server)
        - Layer 3 Protocol : -
        - 보안 기능 : 대칭키 암호화 방식(LADAP)
