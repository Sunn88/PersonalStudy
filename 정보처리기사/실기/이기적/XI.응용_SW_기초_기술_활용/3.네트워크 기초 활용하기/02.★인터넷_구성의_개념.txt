출제빈도 중
빈출태그 : IP주소 / IPv6 / 도메인 네임 / DNS / 서브넷

1) 인터네트워킹
- 각각 운영 중인 네트워크를 상호 연결해 데이터 통신이 이루어지도록 하는 것
- 일반적으로 근거리 통신망(LAN)과 광대역 통신망(WAN)을 연결하는 것
- 인터네트워크 관련 장비 : 리피터, 브리지, 라우터, 게이트웨이

2) 인터넷
- TCP/IP 프로토콜을 기반으로 전 세계 수많은 컴퓨터와 네트워크가 연결된 광범위한 통신망
- 네트워크들의 집합체
- 인터넷으로 연결된 각 컴퓨터들은 IP 주소라는 고유의 컴퓨터 주소를 가짐

3) 인터넷 주요 서비스(TCP/IP상에서 운용됨)
- WWW(World Wide Web) : 하이퍼텍스트 기반으로 되어 있는HTTP 프로토콜을 사용해 웹페이지는 서버에서
    정보를 제공해 주고 클라이언트에서는 웹 브라우저에 의해 정보를 검색하고 제공받는 서비스
- FTP : 파일 전송 프로토콜의 약자로 인터넷에서 파일을 송수신할 때 사용되는 서비스
- 텔넷(Telnet) : 컴퓨터 통신망상의 다른 컴퓨터에 로그인하기 위해 사용하는 프로토콜 혹은 서비스
- 유스넷(Usenet) : 분야별로 공통된 관심사를 가진 인터넷 사용자들이 서로의 의견을 주고받을 수 있게
    하는 서비스
- 아키(Archie) : 익명의 FTP 사이트에 있는 파일 정보를 검색할 수 있도록 하는 서비스
- 전자우편(E-Mail) : 컴퓨터 통신망을 이용해 사용자 간에 편지나 여러 정보를 주고받을 수 있는 서비스

4) 인터넷 주소 체계
- 인터넷 프로토콜(IP, Internet Protocol)
    - 패킷 교환 네트워크에서 정보를 주고받기 위해 사용하는 프로토콜
    - OSI 7계층 중 3계층인 네트워크 계층 혹은 인터넷 모델의 인터넷 계층에 속함
    - 패킷 헤더(Packet Header)에 포함된 IP 주소를 기반으로 패킷을 전달해 호스트의 주소 지정과
        패킷 분할 및 조립 기능을 담당함
    - 패킷 전송과 정확한 순서를 보장하려면 TCP 프로토콜과 같은 IP의 상위 프로토콜을 이용해야 함
- IP 주소
    - 컴퓨터 네트워크에서 컴퓨터 및 통신장비에 부여하는 고유한 주소
    - 인터넷에 연결된 기기들 간의 통신을 위해 사용하며 IP 주소를 이용해 송신 호스트에서
        수신 호스트로 패킷을 전달함
- IPv4
    - IP 주소는 컴퓨터 네트워크의 각 장치들에게 할당하는 논리적인 주소로 8비트 크기의 필드 네 개를
        모아 구성한 32비트(4바이트) 논리 주소임
    - IP 주소는 .(점)으로 구분한 10진수 형태 4개로 구성됨
    - IPv4의 IP 주소는 32비트로 네트워크 부분과 호스트 부분으로 구성되며, 효율적인 관리를 위해
        클래스의 개념을 도입하며 라우팅 시 네트워크 부분만 참조함
    - 전송 방식으로는 멀티캐스트, 유니캐스트, 브로드캐스트 방식이 있음
- IPv4 주소 체계
    - IPv4는 클래스 기반 주소 지정 방식으로 5개의 클래스로 구성됨
    - 각 클래스의 네트워크 주소 : 네트워크 자체의 주소
    - 호스트 주소 : 각 네트워크에 속한 호스트 주소
    - A class
        - 대형 기관 및 기업에서 사용
        - 2^24 중 16,777,214개의 호스트 사용 가능
        - Net id 1개, Host id 3개
        - 첫 번째 바이트 범위 : 0~127
    - B class
        - 중형 기관 및 기업에서 사용
        - 2^16 중 65,543개의 호스트 사용 가능
        - Net id 2개, Host id 2개
        - 첫 번째 바이트 범위 : 128~191
    - C class
        - 소형 기관 및 기업에서 사용
        - 2^8 중 254개의 호스트 사용 가능
        - Net id 3개, Host id 1개
        - 첫 번째 바이트 범위 : 192~223
    - D class
        - 멀티캐스트용
        - 첫 번째 바이트 범위 : 224~239
    - E class
        - 실험용
        - 첫 번째 바이트 범위 : 240~254
    - 주소 127.0.0.1번 : 루프백 인터페이스 -> 로컬 시스템의 내부 테스트를 위해 예약되어 있음
    - 사설 네트워크 주소 : 10.0.0.0, 172.16.0.0, 192.168.0.0
- 도메인 이름
    - 숫자 IP 주소를 사람들이 이해하기 쉬운 단어로 표현한 것
        - IP 주소 : 62.168.1.21
        - 도메인 네임 : www.daum.Net
    - DNA(Domain NAme System) : 문자로 된 도메인 네임을 숫자로 된 IP 주소로 변환하는 시스템
- 서브넷(Subnet)
    - 하나의 네트워크 주소를 이용해 여러 개의 네트워크로 구성하는 것
    - 호스트 주소 비트의 일부를 네트워크 주소로 사용함
- 서브넷 마스크
    - 컴퓨터가 속한 네트워크를 나타내는 네트워크 식별자를 추출하는 것
    - IP 주소를 네트워크 주소 부분과 호스트 주소 부분으로 구분하기 위해 사용함
    - 네트워크 부분과 호스트 부분을 명시하는 역할
        - 네트워크 부분 : 1
        - 호스트 부분 : 0
    - 기본 서브넷 마스크
        - A class : 255.0.0.0
        - B class : 255.255.0.0
        - C class : 255.255.255.0

5) IPv6
- IPv6는 IPv4 주소 자원의 부족과 인터넷 보안의 강화를 위해 제시된 인터넷 프로토콜 6번째 버전
- IPv6 주소는 기존의 IPv4 주소 체계를 128비트 크기로 확장한 차세대 인터넷 프로토콜 주소
- IPv6와 IPv4의 차이점
    - IPv4
        - 주소를 나타내기 위해 32비트 사용
        - 32비트를 8비트 단위로 구분해 10진수로 표현
        - ex) 201.10.34.130
    - IPv6
        - 주소를 나타내기 위해 128비트 사용
        - 사용 가능 주소 : 2^128
        - ex) 2010:0DAC:0000:0000:0000:0000:14C0:75AB
- IPv6의 주소 유형
    - 유니캐스트 : 1대1(특정 단일)
    - 멀티캐스트 : 1대다(특정 집단)
    - 애니캐스트 : 기타 방식 1대1(근접 누구나)
- IPv4/IPv6 전환 기술
    - IPv6를 이용해 구축한 네트워크와 기존 네트워크(IPv4)간의 호환성이 지원되어야 함
    - IPv6 패킷이 IPv4 망을 통해 전달될 수 있어야 함
    - 듀얼 스택(Dual Stack)
        - IPv4/IPv6를 동시에 지원
        - IPv4/IPv6 패킷을 주고받을 수 있음
    - 터널링(Tunneling)
        - 두 IPv6 네트워크 간에 터널을 이용하는 기술
        - IPv4/IPv6 호스트와 라우터에서 IPv6 패킷을 IPv4 패킷에 캡슐화해 전송
    - 주소 변환(Address Translation)
        - IPv4와 IPv6 간에 주소를 변환해 두 버전을 연동
