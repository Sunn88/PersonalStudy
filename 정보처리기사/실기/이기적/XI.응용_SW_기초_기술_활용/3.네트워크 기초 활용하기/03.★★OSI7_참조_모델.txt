출제빈도 상
빈출태그 : OSI 7 Layer / PDU

1) 계층화 구조
- 통신 기능의 확장 및 통신기술의 도입을 쉽게 지원하는 다양한 방법 중 프로토콜을 몇 개의 계층으로
    나누는 계층화와 관련된 참조 모델에는 대표적으로 OSI 7 Layer가 있음
- 구성요소 : 개체(entity), 데이터 단위(data unit), 접속(connection), 프로토콜(protocol),
    서비스 접근점(service access point), 서비스(service), 식별자(identifier)로 구성됨
- OSI 모델
    - 통신 과정을 7계층으로 나누어 각 계층마다 통신 기능을 정의하고 표준화한 모델을 제시함
    - 표준 프로토콜을 사용함으로써 다른 통신 시스템들 간의 상호 호환성을 구현함
    - 두 컴퓨터의 응용 프로그램 간의 데이터 전달 과정을 규정한 모델
- OSI 참고 모델에서 계층을 나누는 목적
    - 시스템 간의 통신을 위한 표준을 제공하는 데 있음
    - 시스템 간의 정보 교환을 하기 위한 상호 접속점을 정의함
    - 관련 규격의 적합성을 조성하기 위한 공통적인 기반 조성을 할 수 있음
    - 네트워크 표준안을 따르고 있어 상호 호환성 문제가 발생하지 않음

2) PDU(Protocol Data Unit)
- 동일 계층 간 데이터를 전송하는 기본 단위
- 계층별 단위
    - 물리 계층 : 비트(Bit)
    - 데이터 링크 계층 : 프레임(Frame)
    - 네트워크 계층 : 패킷(Packet)
    - 전송 계층 : 세그먼트(Segment), 데이터그램(Datagram)
    - 세션/표현/응용 계층 : 메시지(Message)

3) OSI(Open Systems Interconnection) 7 계층의 주요 기능
- 하위 계층
    - 물리 계층(Physical Layer)
        - 전기적, 기계적, 기능적, 절차적 기능 정의
        - 허브, 네트워크 카드, 케이블 등 전송 매체를 이용해 비트를 전송
        - 표준 : RS-232C, X.21
        - 주요 장비 : 리피터
    - 데이터 링크 계층(Data Link Layer)
        - 내부 네트워크상에서의 흐름 제어, 에러 제어
        - 현재 노드와 다음에 접근할 노드의 물리적 주소를 포함해 프레임을 구성
        - 표준 : HDLC, LLC, LAPB, LAPD, ADCCP
        - 주요 장비 : 스위치
    - 네트워크 계층(Network Layer)
        - 논리 주소 지정, 패킷의 최적의 경로를 설정 및 네트워크 연결 관리
        - 표준 : X.25, IP
        - 주요 장비 : 라우터
    - 전송 계층(Transport Layer)
        - 외부 네트워크 통신 종단 간의 에러 제어 및 흐름 제어
        - 표준 : TCP, UDP
        - 주요 장비 : 게이트웨이
- 상위 계층
    - 세션 계층(Session Layer)
        - 회화 구성(반이중, 전이중), 동기 제어, 데이터 교환 관리, 프로세스 간의 연결을 확립,
            관리, 단절시키는 수단을 관장
        - 주요 장비 : 게이트웨이
    - 표현 계층(Presentation Layer)
        - 코드 변환, 암호화 및 복호화, 압축, 구문 검색
        - 주요 장비 : 게이트웨이
    - 응용 계층(Application Layer)
        - 응용 프로그램 간의 네트워크 서비스
        - 프로토콜의 종류 : HTTP, SNMP, FTP, TELNET 등
        - 주요 장비 : 게이트웨이
