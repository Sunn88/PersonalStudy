출제빈도 중
빈출태그 : LAN / WAN / LAN의 토폴로지 / 네트워크 장비 / 패킷 교환망

1) 컴퓨터 네트워크(Computer Network)
- 네트워크(통신망, Network) : 원하는 정보를 원하는 수신자 또는 거기에 정확하게 전송하기 위한
    기반 인프라
- 컴퓨터 네트워크 : 유/무선 매체를 이용해 통신 설비를 갖춘 장비를 연결하는 통신망

2) 네트워크의 장점
- 네트워크를 구축해 서버를 통해 구성원들 간에 데이터 공유를 편리하게 할 수 있음
- 주변 장치를 공유해 공간 및 비용을 절약할 수 있음
- 저장 서버를 지정하면 중복 백업을 방지해 백업이 용이함
- 거리과 공간의 제약을 극복함으로써 다양한 응용 프로그램과 서비스 지원이 용이함
- 사용자에게 편리성과 효율성을 제공할 수 있음

3) 거리(규모)에 따른 네트워크 분류
- LAN(Local Area Network) : 근거리 네트워크 : 한정된 지역에서 장치들을 연결해 구축한 네트워크
- MAN(Metropolitan Area Network) : 도시권 네트워크 : 하나의 도시와 같이 LAN보다 더 큰 규모
- WN(Wide Area Network) : 광대역 네트워크 : 멀리 떨어진 지역을 네트워크로 구성

4) LAN의 이해
- 동일 빌딩 내 또는 수백 m ~ 수 Km 이내의 한정된 지역 내 등 비교적 좁은 지역에 분산 배치된
    컴퓨터와 프린터 등의 단말기를 통신 회선으로 연결해 각종 정보를 교환할 수 있는 통신 네트워크
- 특징
    - 단일 기관의 소유 및 제한된 지역 내의 네트워크
    - 어떤 종류의 통신 시스템 기기와도 연결이 가능함
    - 광대역 전송 매체의 사용으로 고속 통신이 가능함
    - 오류 발생률이 낮으며. 전송 지연을 최소화할 수 있음
    - 공유 매체 사용으로 경로 선택 없이 매체에 연결된 모든 장치로 데이터 전송이 가능함
    - 통신 기기의 재배치와 확장성이 좋음
    - Wi-Fi 기술을 이용한 무선 LAN의 비중이 높아지고 있음
- 기본 형태(위상, 토폴로지, topoloty)
    - 토폴로지 : 컴퓨터 네트워크의 요소들을 물리적으로 연결한 배치의 형태
    - 성형(Star, 스타형)
        - 중앙에 호스트 컴퓨터가 있고 이를 중심으로 터미널들이 연결되는 중앙 집중식이 네트워크
            구성 형태
        - 중앙 컴퓨터와 직접 연결되어 응답이 빠르고 통신 비용이 적게 소요되지만, 중앙 컴퓨터에
            장애가 발생하면 전체 시스템이 마비됨
    - 링형(Ring)
        - 데이터는 한쪽 방향으로만 흐르고 병목 현상이 드물지만, 두 노드 사이의 채널이 고장나면
            전체 네트워크가 손상될 수 있음
        - 한 노드가 절단되어도 우회로를 구성해 통신이 가능함
    - 버스형(Bus)
        - 한 개의 통신 회선에 여러 개의 사이트가 연결된 형태
        - 한 사이트의 고장은 나머지 사이트들 간의 통신에 아무런 영향을 주지 않음
    - 계층형(Tree)
        - 트리 형태
        - 분산 처리 시스템을 구성하는 방식
    - 망형(Mesh)
        - 각 사이트는 시스템 내의 모든 사이트들과 직접 연결괸 형태
        - 통신 회선의 총 경로가 다른 네트워크 형태에 비해 가장 길게 소요됨
        - 많은 단말기로부터 많은 양의 통신을 필요로 하는 경우에 유리함
        - n개의 구간을 망형으로 연결하면 n(n-1)/2개의 회선이 필요
- LAN의 전송 방식에 의한 분류
    - 베이스 밴드(Baseband) 방식
        - 신호 변조 없이 고유 주파수 영역을 사용하는 방식
        - 시분할 다중화 방식(TDM)을 사용하고 통신 방식이 쉽고 경제적임
            - 다중화
                - 여러 개의 채널들이 하나의 통신 회선을 통해 결합된 신호의 형태로 전송되고 수신
                    측에서 다시 이를 여러 개의 채널 신호를 분리하는 것
                - 통신 회선을 다중화해 선로의 공동 이용이 가능해 전송 효율을 높일 수 있음
                - 시분할 다중화(TDM : Time Division Multiplexing)
                - 주파수분할 다중화(FDM : Frequency Division Multiplexing)
    - 브로드 맨드(Broadband) 방식
        - 디지털 신호를 아날로그 신호로 광대역 변조하는 방식
        - 주파수분할 다중화 방식(FDM)을 사용
- LAN의 표준안(계층 구조)
    데이터 링크 계층        LLC(논리 링크 제어)
                            MAC(매체 접근 제어)
    물리 계층               물리 계층
    OSI 7 계층형            LAN의 계층
- IEEE 802의 표준 규격
    - 802.1 : 상위 계층 인터페이스
    - 802.2 : 논리 링크 제어(LLC)
    - 802.3 : CSMA/CD
    - 802.4 : 토큰 버스(Token Bus)
    - 802.5 : 토큰 링(Token Ring)
    - 802.6 : MAN
    - 802.8 : 고속 이더넷(Fast Ethernet)
    - 802.11 : 무선 LAN
    - 802.15 : 블루투스
- 전송 매체 접근 제어(MAC, Media Access Control)
    - 하나의 통신 회선에 여러 대의 컴퓨터를 연결해 통신이 가능하도록 함
    - 연결된 컴퓨터들이 일정한 규칙 없이 데이터를 전송할 경우 통신 회선을 공유하기 때문에
        데이터가 충돌하게 됨
    - MAC 방식
        - CSMA(Carrier Sense Multiple Access)
        - CSMA/CD(Carrier Sense Multiple Access/Collision Detection)
            - 통신 채널 상태를 파악해 통신 채널이 데이터 전송을 하지 않을 때 정보를 전송하는 방식
            - 전송 후 충돌이 발생하면 즉시 전송을 멈추고 다른 노드들에게 충돌 신호를 보낸 다음
                기다렸다가 재전송 하는 기법
        - 토큰 버스(Token Bus)
        - 토큰 링(Token Ring)
        - 토큰 패싱(Token Passing)
- LAN의 표준안 동향
    - 이더넷(Ethernet)
        - 가장 많이 사용하는 LAN 구축 방식
        - 제록스사에서 개발 후 DEC와 인텔사가 연합해 확장한 LAN의 표준안
        - 1985년 IEEE에 의해 802.3이 표준안으로 채택된 후 대부분 버스형에 많이 사용됨
    - 고속 이더넷(Fast Ethernet)
        - 100 BASE T라고도 불리는 이더넷 고속버전
        - 100Mbps의 전송 속도를 지원하는 CSMA/CD 방식 기반의 LAN의 표준안
    - 기가비트 이더넷(Gigabit Ethernet)
        - 1Gbps의 속도를 제공하며 기존 이더넷 방식을 그대로 채택하고 있으므로 호환성이 높아
            효율적임
    - FDDI(Fiber Distributed Data Interface)
        - LAN간의 트래픽 폭증 문제를 해결할 수 있는 고속 LAN으로 대표적인 표준임
        - 미국 표준협회(ANSI)와 ITU-T에 의해 표준화됨
        - 100Mbps의 속도를 갖는 두 개의 링으로 구성됨
- 네트워크 관련 장비
    - 리피터(Repeater)
        - 디지털 신호를 증폭시켜 주는 역할
        - 신호가 약해지지 않고 컴퓨터로 수신되도록 함
        - OSI 7 참조 모델의 1계층(물리 계층)에서 동작
    - 허브(Hub)
        - 네트워크에 연결된 각 회선이 모이는 접선 장치
        - 각 회선을 통합적으로 관리하는 장비
    - 브리지(Bridge), 스위치(Switch)
        - 두 개의 LAN이 데이터 링크 계층에서 서로 결합되어 있는 경우에 이들을 연결하는 장비
        - 스위치는 처리 방식이 하드웨어로 이루어지기 때문에 소프트웨어적으로 프레임을 처리하는
            브리지에 비해 훨씬 빠름
        - 브리지는 포트들이 같은 속도를 지원하는 반면, 스위치는 서로 다른 속도를 연결 기능을 제공함
        - 스위치는 브리지에 비해 제공하는 포트 수가 훨씬 많음
        - OSI 7 참조 모델의 2계층(데이터링크 계층)에서 동작
    - 라우터(Router)
        - 서로 다른 형태의 네트워크를 상호 접속하는 3계층(네트워크 계층) 장비
        - 적절한 전송 경로를 선택하고 이 경로로 데이터를 전달함
        - OSI 7 참조 모델의 3계층(네트워크 계층)에서 동작
    - 게이트웨이(Gateway)
        - 서로 다른 프로토콜을 사용하는 망 연결
        - OSI 7 참조 모델의 전(1~7) 계층에서 동작

* L2, L3, L4 스위치
    - 네트워크 장비인 스위치는 OSI중 어떤 계층에서 수행되는지에 따라 구분됨
        - L2 스위치 : 가장 원초적인 스위치 -> 상위 레이어에서 동작하는 IP 이해가 불가능함
        - L3 스위치 : Ip 레이에서의 스위칭을 수행해 외부로 전송하며 라우터와의 경계가 모호함
        - L4 스위치 : TCP/UDP 등 스위칭을 수행하며 응용 계층에서 파악이 가능한 이메일 내용 등
            정교한 로드 밸런싱 수행이 불가능함

5) WAN(Wide Area Network)의 이해
- 국가, 대륙과 같이 광범위한 지역을 연결하는 네트워크
- 거리에 제약이 없으나 다양한 경로를 지나 정보가 전달되므로 LAN보다 속도가 느리고 에러율도 높음
- 전용 회선 방식 : 통신 사업자가 사전에 계약을 체결한 송신자와 수신자끼리만 데이터를 교환하는 방식
    - 전용 회선(Leased Line)
        - 회선이 단말기 상호 간에 항상 고정되어 있는 방식
        - 전송 속도가 빠르며, 전송 오류가 적음
- 교환 회선 방식 : 공중망을 활용해 다수의 사용자가 선로를 공유하는 방식
    - 교환 회선(Switched Line)
        - 교환기에 의해 단말기 상호 간에 연결되는 방식
        - 전용 회선에 비해 속도가 느림
- 회선 구성 방식
    - 점-대-점 방식
        - Point-to-Point(=PEER To PEER)
        - 중앙 컴퓨터와 단말기를 일대일로 연결하는 방식
        - 통신망을 성형(Star)으로 구성 시 사용함
        - 회선제어 : 경쟁 방식 사용
    - 다중 점(Multi-Point) 방식
        - 한 개의 통신 회선에 여러 개의 단말기를 연결하는 방식
        - 멀티 드롭(Multi-Drop) 방식이라고도 함
        - 통신망을 버스형(Bus)으로 구성 시 사용함
        - 회선제어 : 폴링과 셀렉션 사용
    - 회선다중(Line Multiplexing) 방식
        - 여러 개의 단말기를 다중화기를 이용해 중앙 컴퓨터와 연결하는 방식
- 교환 회선 방식
    - 교환 회선
        - 회선교환
        - 축적 교환
            - 메시지교환
            - 패킷 교환
                - 가상회선
                - 데이터그램
- 회선 교환 방식(Circuir Switching)
    - 물리적 전용선을 활용해 데이터 전달 경로가 정해진 후 동일 경로로만 전달이 됨
    - 데이터를 동시에 전송할 수 있는 양을 의미하는 대역폭이 고정되고 안정적인 전송률을 확보할 수 있음
    - 회선 교환 방식이 특징
        - 메시지가 전송되기 전에 발생지에서 목적지까지의 물리적 통신 회선 연결이 선행되어야 함
        - 기억 장치를 사용하지 않음
        - 데이터 전송 전에 먼저 통신망을 통한 연결이 필요함
        - 코드와 속도가 다른 단말기 간에는 통신이 불가능함
        - 전체 경로가 미리 확보되어야 함
        - 일정한 데이터 전송률을 제공하므로 동일한 전송 속도가 유지됨
        - 연결만 되면 실시간 통신이 가능함
        - 연결되면 통신 회선은 공유되지 않고 일대일 방식으로 데이터를 송수신하게 됨
        - 통신 회선이 독점되므로 통신 비용이 비쌈
        - 고정 대역폭(Band Witdh)을 사용함
        - 종류 : 공간 분할 교환 방식(SDS), 시분할 교환 방식(TDS)
- 축적 교환 방식
    - 메시지 교환 방식(Message Switching)
        - 하나의 메시지 단위로 저장-전달(Store-and-Forward) 반식에 의해 데이터를 교환하는 방식
        - 각 메시지마다 수신 주소를 붙여서 전송하므로 메시지마다 전송 경로가 다르게 전달됨
        - 네트워크에서 속도나 코드 변환이 가능함
    - 패킷 교환 방식(Packet Switching)
        - 메시지를 일정한 길이의 전송 단위인 패킷으로 나누어 전송하는 방식
        - 다수의 사용자 간에 비대칭적 데이터 전송을 원활하게 하므로 모든 사용자 간에
            빠른 응답 시간 제공이 가능함
        - 전송에 실패한 패킷의 경우 재전송이 가능함
        - 패킷 단위로 헤더를 추가하므로 패킷별 오버헤드가 발생함
        - 현재 컴퓨터 네트워크에서 주로 사용하는 방식
        - 패킷교환공중데이터통신망(PSDN)이라고도 함
        - 패킷 교환 방식은 축적 후 전달(Store-and-Forward) 방식
        - 메시지를 작은 데이터 조각인 패킷으로 블록화함
        - 종류 : 가상 회선 방식, 데이터그램 방식
