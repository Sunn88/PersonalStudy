1) 관계 대수(Relational Algebra) - 321p 그림
- 릴레이션에서 사용자가 원하는 결과를 얻기 위해 연산자를 표현하는 방법
- 결과를 얻기 위한 절차를 표현 -> 절차적 언어
- 순수 관계 연산자
    - SELECT
        - 릴레이션에서 조건을 만족하는 수평적 부분 집합(튜플)을 구하기 위한 연산
        - 결과는 조건을 만족하는 튜플들로 테이블이 만들어짐
        - 표기 형식 : 
    - PROJECT
        - 릴레이션에서 수직적 부분 집합(속성의 값)을 구하는 연산
        - 원하는 속성만 추출하기 위한 연산
        - 표기 형식 : 
    - JOIN
        - 두 테이블에서 조건에 맞는 관련된 튜플들을 하나의 튜플로 결합해 하나의 테이블로 만드는 연산
        - 표기 형식 : 
        - 동일 조인(Equi Join)
            - 관계 연산자 중 '=' 연산자만을 사용해 조건 표현
            - 가장 기본이 되는 조인
            - 두 테이블의 모든 속성을 합한 하나의 구조로 만들어짐(중속도 모두 표현)
        - 자연 조인(Natural Join)
            - 동일 조인한 결과에서 중복되는 속성을 제거해 표현함
        - 외부 조인(Outer Join)
            - 조인 시 조건에 맞지 않는 튜플도 결과 테이블에 포함시켜 조인함
            - 해당 자료가 없는 부분은 NULL 값이 됨
    - DIVISION
        - B 테이블의 조건을 만족하는 튜플들을 테이블 A에서 추출하는 연산
        - 결과는 연산에 사용된 속성은 제외됨
        - 표기 형식
- 일반 집합 연산자 : 합집합, 교집합, 차집합, 카티션 프로덕트
