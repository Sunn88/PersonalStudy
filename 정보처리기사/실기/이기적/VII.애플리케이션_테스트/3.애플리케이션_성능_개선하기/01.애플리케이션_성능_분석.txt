    / 공개 소스 성능 테스트 도구

1) 애플리케이션 성능을 측정하기 위한 지표
- 처리량(Throughput)
    - 애플리케이션이 제한된 시간에 처리할 수 있는 처리량, 트랜잭션 수
    - 웹 애플리케이션의 처리량은 시간당 처리 가능한 페이지 수를 의미
- 자원 사용률(Resource Usage)
    - 처리하는 동안 사용하는 CPU 사용량, 메모리 사용량, 네트워크 사용량
- 응답 시간(Response Time)
    - 요구자의 작업 요청 후 애플리케이션의 응답 출력이 개시될 때까지의 시간
    - 웹 애플리케이션의 응답 시간은 메뉴 클릭 시 해당 메뉴가 나타나기까지 걸리는 시간을 의미
- 경과 시간(Turaround Time)
    - 요구를 입력한 시점부터 트랜잭션 처리 후 그 결과의 출력이 완료될 때까지 걸리는 시간

2) 유형별 성능 분석 도구
- 성능/부하/스트레스(Performance/Load/Stress) 점검 도구
    - 애플리케이션의 성능 분석을 위한 가상의 사용자를 인위적으로 생성해 시스템의
        부하, 스트레스를 가한 후 처리량, 응답 시간, 경과 시간 등을 점검하기 위한 도구
- 모니터링(Monitoring) 도구
    - 애플리케이션 실행 시 자원 사용량을 확인하고 분석 가능한 도구
    - 모니터링 도구는 시스템의 안정적 운영을 지원하는 도구
    - 성능 모니터링, 성능 저하 원인 분석, 시스템 부하량 분석, 장애 진단, 사용자 분석,
        용량 산정 등의 기능 제공

3) 애플리케이션 성능 저하 원인 분석
- DB 연결 및 쿼리 실행 시 발견되는 성능 저하 원인
    - 성능 저하 원인 : 일반적으로 DB에 연결하기 위해 Connection 객체를 생성하거나 쿼리를
        실행하는 애플리케이션 로직에서 성능 저하 또는 장애가 많이 발견됨
    - DB Lock
        - 대량의 데이터 조회, 과도한 업데이트, 인덱스 생성 시 발생하는 현상
        - 요청한 작업이 Lock의 해제 시까지 대기하거나 타임아웃될 때 성능 저하 현상이
            발생할 수 있음
    - 불필요한 DB Fetch
        - 실제 필요한 데이터보다 많은 대량의 데이터 요청이 들어올 때 발생함
        - 결과 세트에서 마지막 위치로 커서를 옮기는 작업이 빈번한 경우 응답 시간 저하 현상이
            발생할 수 있음
    - 연결 누수와 부적절한 커넥션 풀 크기
        - 연결 누수(Connection Leak) : DB 연결과 관련한 JDBC 객체를 사용 후 종료하지 않을 때
            발생함
        - 부적절한 커넥션 풀 크기 : 커넥션 풀을 너무 작거나 크게 설정할 경우 성능 저하 현상이
            발생할 가능성이 있음
    - 기타
        - 트랜잭션이 확정(Commit)되지 않고 커넥션 풀에 반환되는 경우 성능 저하 현상이
            발생할 수 있음
        - 잘못 작성된 코드로 인해 불필요한 커밋이 자주 발생하는 경우 성능 저하 현상이
            발생할 수 있음
- 내부 로직으로 인한 성능 저하 원인
    - 웹 애플리케이션의 인터넷 접속 불량으로 인해 성능이 저하될 수 있음
    - 특정 파일의 업로드, 다운로드로 인해 성능이 저하될 수 있음
    - 정상적으로 처리되지 않은 오류 처리로 인해 성능이 저하될 수 있음
- 외부 호출(HTTP, 소켓 통신)로 인한 성능 저하 원인
    - 임의 트랜잭션이 수행되는 도중 다른 외부 호출 트랜잭션이 장시간 요청되거나,
        타임아웃이 발생하는 경우 성능이 저하될 수 있음
- 잘못된 환경 설정이나 네트워크 문제로 인한 성능 저하 원인
    - 환경 설정으로 인한 성능 저하 : 스레드 풀(Thread Pool), 힙 메모리(Heap Memory)의 크기를
        너무 작게 설정하면 Heap Memory Full 현상 발생으로 성능이 저하될 가능성이 있음
    - 네트워크 장비로 인한 성능 저하 : 라우터, L4 스위치 등 네트워크 관련 장비 간 데이터
        전송 실패 또는 전송 지연에 따른 데이터 손실 발생 시 애플리케이션의 성능 저하 또는
        장애가 발생할 수 있음

4) 오픈소스 성능 테스트 도구
- JMeter
    - 설명 : HTTP, FTP, LDAP 등 다양한 프로토콜 지원으로 안전성, 확장성, 부사, 기능 테스트 도구
    - 지원 환경 : Cross Platform
- LoadUI
    - 설명 : HTTP, JDBC 등의 웹 서비스를 대상으로 부하 테스트를 수행하는 서버 모니터링에
        UI를 강화한 도구
    - 지원 환경 : Cross Platform
- OpenSTA
    - 설명 : HTTP, HTTPS 지원하는 부하 테스트 및 생산품 모니터링
    - 지원 환경 : MS Windows

5) 오픈소스 시스템 모니터링 도구
- Scouter
    - 설명 : 통합/실시간 모니터링 및 튜닝에 최적화된 통합 모니터링 툴 -> 단일 뷰 형태를 제공함
    - 지원 환경 : Cross Platform
- Zabbix
    - 설명 : 웹 기반 서버, 서비스, 애플리케이션 모니터링 툴
    - 지원 환경 : Cross Platform
