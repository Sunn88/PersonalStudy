출제빈도 중
빈출태그 : 연계 테스트 / 테스트 구간
* 연계 테스트는 연계가 정상적으로 작동하는지를 테스트하는 단계
* 큰 틀에서읙 ㅐ념과 연계 테스트 구간별 테스트 확인사항 기억하기

1) 연계 테스트
- 구축된 연계 시스템과 연계 시스템의 구성요소가 정상적으로 동작하는지 확인하는 활동
    - 연계 시스템의 구성요소 : 송신 모듈, 수신 모듈, 연계 서버와 엔진, 모니터링 현황
- 연계 테스트, 데이터가 발생하고 활용되는 응용 애플리케이션 기능과의 통합, 흐름을 테스트하는
    통합 테스트로 단계적으로 수행함

2) 연계 테스트 구간 - 196p 표
- 구간을 상세화해 송수신 시스템 간, 연계 테이블 간의 테스트를 수행하고 정상적으로 데이터가
    송수신되면 송신 시스템의 운영 DB에서 연계 데이터를 추출 및 생성해 수신 시스템의 운영 DB에
    반영하는 테스트
- 연계 테스트는 송신 시스템의 운영 DB에서 연계 데이터를 추출 및 생성해 수신 시스템의 운영 DB에
    반영하는 구간의 테스트
- 송신 시스템
    - 테스트 확인사항           위치                    테스트 구간
        데이터 발행과 DB 저장   운영 응용 애플리케이션  통합
        연계 데이터 추출 생성   연계 응용 프로그램      연계, 통합
        연계 테이블 생성        연계 응용 프로그램      연계, 통합
- 중계 서버 
    - 테스트 확인사항           위치                        테스트 구간
        연계 테이블 간 송수신   연계 서버 또는 중계 서버    단위, 연계, 통합
        연계 테이블 생성        연계 서버 또는 중계 서버    단위, 연계, 통합
- 수신 시스템
    - 테스트 확인사항                       위치                       테스트 구간
        연계 테이블에서 연계 데이터 로드    연계 응용 프로그램          연계, 통합
        운영 DB 반영                        연계 응용 프로그램          연계, 통합
        반영된 데이터 검색, 활용            운영 응용 애플리케이션      통합

3) 연계 테스트 케이스 작성 및 명세화
- 연계 테스트 구간에서의 데이터 및 프로세스 흐름에 따라 테스트 케이스(Test case)를 작성함
- 연계 테스트 케이스는 연계 테이블 단위로 작성함
- 테스트 케이스 작성 시 가장 핵심적인 사항은 테스트 항목의 도출
- 테스트할 대상 및 기능 등이 충분히(완전하게), 누락 없이, 일관성 있게, 구체적으로 식별되어야 함
- 연계 테스트 환경은 실제 운영 환경과 같거나 유사하게 구축함
- 송신/수신용 연계 응용 프로그램의 기능 위주 결함을 확인하는 단위 테스트 케이스로 작성함

4) 연계 테스트 환경 구축
- 연계 테스트 환경은 실제 운영 환경과 같게 또는 유사하게 구축함
- 연계 서버(엔진), 송/수신용 어댑터 또는 에이전트(Agent) 설치를 비롯해 송수신 운영 DB,
    송수신 운영 DB에서 연계 데이터 추출 및 반영에 필요한 테이블, 데이터와 송수신용 연계 응용
    프로그램들을 설치 및 준비
- 테스트 수행 전 연계 테스트 일정, 절차, 방법, 소요시간, 테스트 환경, 환경 구축 기간 등을
    협의해 계획을 수립하고 테스트 환경을 구축함

5) 연계 테스트 수행
- 연계 응용 프로그램을 실행해 연계 테스트 케이스의 시험 항목 및 처리 절차 등을 실제로 진행하는 것
- 송수신용 연계 응용 프로그램의 단위 테스트를 먼저 수행
    -> 단위 테스트의 수행을 완료
    -> 연계 테스트 케이스에 따라 데이터 추출, 데이터 송수신, 데이터 반영 과정 등의
        연계 테스트 수행

6) 연계 테스트 수행 결과 검증
- 연계 테스트 케이스의 시험 항목 및 처리 절차를 수행한 결과가 예상 결과와 같은지 검증 수행
- 테스트 결과를 검증하는 방법
    - 운영 DB 테이블의 건수를 카운트하는 방법
    - 실제 테이블이나 파일을 열어서 데이터를 확인하는 방법
    - 파일 생성 위치의 파일 생성 여부와 파일 크기를 확인하는 방법
    - 연계 서버(또는 연계 엔진)에서 제공하는 모니터링 화면의 내용을 확인하는 방법
    - 시스템에서 기록하는 로그를 확인하는 방법
